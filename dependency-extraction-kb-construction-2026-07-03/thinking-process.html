Thinking Process

The iterative inference chain. Each node shows what the entry rationale Expected ◈, which problems were Solved ✓, which Remain ✗, which were Added +, where a direction Pivoted ↻, and how later queries were Revised ✎.

추론 과정

반복 추론 사슬. 각 node는 진입 rationale이 기대한 것 ◈, 어떤 문제가 해결되고 ✓, 무엇이 남으며 ✗, 무엇이 추가되고 +, 방향이 어디서 전환되며 ↻, 이후 질의가 어떻게 수정되었는지 ✎를 보여준다.

Starting Problem출발 문제

Given a newly-added paper, build an LLM-agent pipeline that (1) extracts candidate dependency signals from its text and citations, (2) links them to existing KB nodes or spawns new ones, and (3) keeps the resulting dependency DAG globally consistent (acyclic, non-redundant, stale-free) as it grows incrementally — prioritizing techniques adoptable via prompting/retrieval over custom-trained models.

새로 추가된 논문이 주어질 때, (1) 그 텍스트와 인용에서 후보 의존 신호를 추출하고, (2) 이를 기존 KB 노드에 링킹하거나 신규 생성하며, (3) 증분 성장하는 의존 DAG를 전역 일관(비순환·비중복·stale 없음) 상태로 유지하는 LLM 에이전트 파이프라인을 — 맞춤 학습 모델보다 prompting/검색 채택 가능 기법을 우선하여 — 구축한다.

1

SP1: Citation Intent and Function Classification — Can citations surface directed use/extend derivation-lineage edges, training-free?

SP1: 인용 의도 및 기능 분류 — 인용이 방향성 use/extend 파생-계보 edge를 학습 없이 표면화할 수 있는가?

Papers: [3] ACL-ARC, [1] SciCite, [4] SciBERT, [9] CitePrompt

Expected ◈
  • Citation-intent classification is the primary, longest-established O1 signal, and prompting-adaptable variants should make USES/EXTENDS lineage extraction training-free.
  • 인용 의도 분류가 주요·최장수 O1 신호이며, prompting 적응 변형이 USES/EXTENDS 계보 추출을 학습-free로 만들 것이다.
Solved ✓
  • ACL-ARC 6-way schema keeps USES/EXTENDS as separate lineage classes; SciBERT reaches ~68–71 macro-F1 (the +12-F1 lever) — the O1 citation-lineage signal is identified and benchmarked.
  • ACL-ARC 6-way 스키마가 USES/EXTENDS를 별도 계보 클래스로 유지하고 SciBERT가 ~68–71 macro-F1(+12 F1 레버)을 달성 — O1 인용-계보 신호가 식별·벤치마크됨.
Remains ✗
  • Body-text concept/relation signals (the other half of O1) are not covered here → SP2.
  • Resolving which KB node a cited artifact maps to is not addressed → SP4.
  • 본문 개념/관계 신호(O1의 나머지 절반)는 여기서 다루지 않음 → SP2.
  • 피인용 산물이 어느 KB 노드에 대응하는지 해소는 미해결 → SP4.
Added +
  • Sentence-intent → paper-level dependency-edge aggregation is unmethodized — every classifier stops at sentence labels.
  • 문장 의도 → 논문 수준 의존-edge 집계가 미방법화 — 모든 분류기가 문장 라벨에서 멈춤.
Pivoted ↻
  • Assumed a citation-intent label is a usable dependency edge; inverted — the classification task and the edge-extraction task are one methodized step apart, unaddressed by the entire SP1 literature. Training-free prompting also holds only on the coarse SciCite schema, not the lineage-bearing ACL-ARC one.
  • 인용 의도 라벨이 곧 쓸 수 있는 의존 edge라 가정했으나 반전됨 — 분류 과제와 edge 추출 과제는 한 단계 떨어져 있고 SP1 문헌 전체가 미해결로 둔다. 학습-free prompting도 계보를 담은 ACL-ARC가 아닌 거친 SciCite에서만 성립한다.
Revised ✎
  • SP2: explicitly task it with the sentence-intent → directed-edge aggregation and cited-target resolution that SP1 leaves open — because both are flagged as a 'missing link' in CitePrompt/Kunnath.
  • SP4: add a query on resolving the cited artifact of a USES/EXTENDS citation to a KB node (the intent label alone does not name the target).
  • SP2: SP1이 남긴 문장 의도 → 방향 edge 집계와 피인용 대상 해소를 명시적으로 부과 — CitePrompt/Kunnath가 'missing link'로 지적했기 때문.
  • SP4: USES/EXTENDS 인용의 피인용 산물을 KB 노드로 해소하는 질의 추가(의도 라벨만으로는 대상을 명명하지 못함).
2

SP2: Scientific Concept and Relation Extraction — How to extract concepts and typed relations from a paper's body text?

SP2: 과학 개념 및 관계 추출 — 논문 본문에서 개념과 유형 관계를 어떻게 추출하는가?

Papers: [16] SciERC, [17] DyGIE++, [19] PURE, [22] SemEval-2018

Expected ◈
  • Body-text concept/relation extraction supplies the prerequisite-concept signals needed before any linking step; at least some strong methods should be prompting-adaptable.
  • 본문 개념/관계 추출이 링킹 전 필요한 선행-개념 신호를 공급하며, 강한 방법 일부는 prompting 적응 가능할 것이다.
Solved ✓
  • Span-based SciBERT extraction (SciIE→DyGIE++→PURE→PL-Marker) reaches ~67 entity / ~48 relation F1 on SciERC, with candidate relation vocabularies (SciERC 7, SemEval-2018 6) — the O1 extraction machinery is identified.
  • span 기반 SciBERT 추출(SciIE→DyGIE++→PURE→PL-Marker)이 SciERC에서 ~67 개체 / ~48 관계 F1을 달성하고 후보 관계 어휘(SciERC 7, SemEval-2018 6)를 제공 — O1 추출 기계가 식별됨.
Remains ✗
  • The sentence→edge aggregation carried from SP1 is still open; extraction outputs per-document tuples, not KB nodes → SP4.
  • SP1에서 이월된 문장→edge 집계가 여전히 미해결; 추출은 KB 노드가 아닌 문서별 튜플을 출력 → SP4.
Added +
  • No relation schema contains a requires/prerequisite type — the closest proxies are Used-for and Hyponym-of.
  • End-to-end relation extraction collapses to ~20 macro-F1 on predicted (non-gold) entities.
  • 어떤 관계 스키마도 requires/prerequisite 유형을 담지 않음 — 가장 근접한 대리는 Used-for와 Hyponym-of.
  • 종단간 관계 추출이 예측(비-gold) 개체에서 ~20 macro-F1로 붕괴.
Pivoted ↻
  • Assumed body-text extraction would directly yield prerequisite signals; found scientific IE grew out of descriptive relation typing, so the prerequisite relation is a separate task (SP3), not a downstream label of these extractors.
  • 본문 추출이 선행 신호를 직접 낼 것이라 가정했으나, 과학 IE는 서술적 관계 유형화에서 자랐기에 선행 관계는 이 추출기의 하류 라벨이 아닌 별개 과제(SP3)임이 드러남.
Revised ✎
  • SP3: emphasize that SP3 must supply the prerequisite/requires type these schemas provably lack.
  • SP4: add a query targeting linking of noisy, predicted (non-gold) mentions, since end-to-end accuracy drops sharply.
  • SP6: add a query for LLM-prompting IE benchmarked against SciERC/SemEval — SP2 found only a model-catalog survey.
  • SP3: 이 스키마들이 입증적으로 결여한 prerequisite/requires 유형을 SP3가 공급해야 함을 강조.
  • SP4: 종단간 정확도가 급락하므로 잡음·예측(비-gold) mention 링킹을 겨냥한 질의 추가.
  • SP6: SciERC/SemEval 대비 벤치마크된 LLM prompting IE 질의 추가 — SP2는 모델 카탈로그 서베이만 발견.
3

SP3: Concept Prerequisite Learning and Prediction — How is the prerequisite decision defined, and what mechanism does it require?

SP3: 개념 선행관계 학습 및 예측 — 선행 결정은 어떻게 정의되며 어떤 메커니즘을 요구하는가?

Papers: [30] LectureBank, [32] PREREQ, [34] CD-VGAE, [38] GKROM

Expected ◈
  • This field defines the prerequisite task with matching criteria; GNN/embedding methods should be usable as plug-in similarity-threshold decision criteria.
  • 이 분야가 매칭 기준과 함께 선행 과제를 정의하며, GNN/임베딩 방법이 plug-in 유사도-임계값 결정 기준으로 쓰일 것이다.
Solved ✓
  • The task is formalized (directed, irreflexive, transitive edge) and benchmarked (LectureBank/CD, MOOC); the decision must be asymmetric — symmetric similarity gives P0.98/R0.16 — which supplies O2's directional design constraint and SOTA ceiling (~0.82).
  • 과제가 형식화되고(방향·비반사·이행 edge) 벤치마크됨(LectureBank/CD, MOOC); 결정은 비대칭이어야 하며 — 대칭 유사도는 P0.98/R0.16 — 이는 O2의 방향 설계 제약과 SOTA 상한(~0.82)을 공급한다.
Remains ✗
  • The agent-usable reuse-vs-new-node similarity threshold is not supplied here → SP4.
  • 에이전트가 쓸 재사용-대-신규 유사도 임계값은 여기서 미공급 → SP4.
Added +
  • No training-free / prompting-based prerequisite decision criterion exists — every method is a supervised classifier on a fixed vocabulary; extraction-error propagation is unquantified.
  • 학습-free / prompting 기반 선행 결정 기준이 존재하지 않음 — 모든 방법이 고정 어휘의 지도 분류기; 추출-오류 전파가 미정량화.
Pivoted ↻
  • Assumed GNN/embedding methods double as plug-in thresholds; inverted — they are end-to-end supervised trained classifiers, the threshold learned jointly and inseparable from the model. The field has also plateaued (a plain SVM beat the 2019 VGAE; SOTA margins 0.3–1.2%).
  • GNN/임베딩 방법이 plug-in 임계값을 겸한다 가정했으나 반전됨 — 종단간 지도 학습 분류기로 임계값이 모델과 함께 학습되어 분리 불가. 분야도 정체됨(단순 SVM이 2019 VGAE를 능가; SOTA 마진 0.3–1.2%).
Revised ✎
  • SP4: state that prerequisite methods do NOT supply an agent threshold; SP4 must own the reuse-vs-new decision, composable with an asymmetric directional check.
  • SP6: probe whether LLM prompting can match supervised prerequisite SOTA (~0.82) training-free.
  • SP8/SP9: carry the transitive-closure consistency and cycle-handling requirement (gold LectureBank has 12 cyclic pairs).
  • SP4: 선행 방법이 에이전트 임계값을 공급하지 않음을 명시; SP4가 비대칭 방향 검사와 합성 가능한 재사용-대-신규 결정을 소유해야 함.
  • SP6: LLM prompting이 학습-free로 지도 선행 SOTA(~0.82)에 필적하는지 탐색.
  • SP8/SP9: 이행폐포 일관성과 cycle 처리 요구 이월(gold LectureBank에 12개 순환 쌍).
4

SP4: Scientific Entity and Concept Linking — How to match an extracted mention to an existing node, or decide to spawn a new one?

SP4: 과학 개체 및 개념 링킹 — 추출 mention을 기존 노드에 매칭하거나 신규 생성을 결정하는가?

Papers: [45] BLINK, [47] SapBERT, [46] GENRE, [52] S2abEL

Expected ◈
  • Entity linking provides the matching mechanism AND decision criteria (embedding-similarity thresholds, novelty checks) usable by an agent. (Revised after SP3: SP4 owns the reuse-vs-new decision with an asymmetric check.)
  • 개체 링킹이 매칭 메커니즘과 에이전트가 쓸 결정 기준(임베딩 유사도 임계값, 신규성 검사)을 제공한다. (SP3 이후 수정: SP4가 비대칭 검사와 함께 재사용-대-신규 결정을 소유.)
Solved ✓
  • Zero-shot dense linking (BLINK retrieve-then-rerank, SapBERT cosine, GENRE name generation) links unseen nodes without retraining — the O2 matching mechanism is solved and agent-adoptable.
  • zero-shot 밀집 링킹(BLINK retrieve-then-rerank, SapBERT cosine, GENRE 이름 생성)이 재학습 없이 unseen 노드를 링킹 — O2 매칭 메커니즘이 해결·에이전트 채택 가능.
Remains ✗
  • The operational reuse-vs-spawn threshold is uncalibrated; linking on terse, canonical-name-poor nodes is untested — carried to SP6/SP7.
  • 운용 재사용-대-생성 임계값이 미보정; 간결·정본 이름 빈약 노드 링킹이 미검증 — SP6/SP7로 이월.
Added +
  • The reuse-vs-new threshold is the least-benchmarked quantity — four of six method papers assume closed-world and defer abstention; only S2abEL makes outKB first-class (42.8%, IAA 60–66%).
  • 재사용-대-신규 임계값이 가장 덜 벤치마크됨 — 6개 중 4개가 closed-world를 가정하고 기각을 미룸; S2abEL만 outKB를 일급으로(42.8%, IAA 60–66%).
Pivoted ↻
  • Assumed decision criteria (thresholds, novelty checks) would be available; inverted — mainstream EL benchmarks (TACKBP, Zeshel, AIDA) are built under the in-KB guarantee, so the field optimized disambiguation, not novelty calibration. The exact quantity the agent needs is the least-benchmarked one.
  • 결정 기준(임계값, 신규성 검사)이 가용하리라 가정했으나 반전됨 — 주류 EL 벤치마크(TACKBP, Zeshel, AIDA)가 in-KB 보장 하에 구축되어 분야는 신규성 보정이 아닌 명확화를 최적화. 에이전트가 필요로 하는 정확한 값이 가장 덜 벤치마크됨.
Revised ✎
  • SP5: frame paper-level novelty detection as the coarse partner of entity-level outKB/NIL, targeting the operational-threshold gap jointly.
  • SP7: add a query for node-equivalence when nodes lack canonical names or rich descriptions.
  • SP5: 논문 수준 신규성 검출을 개체 수준 outKB/NIL의 거친 짝으로 프레이밍하여 운용-임계값 gap을 공동 겨냥.
  • SP7: 노드가 정본 이름이나 풍부한 설명을 결여할 때의 노드-동등성 질의 추가.
5

SP5: Related Work Generation and Extension/Novelty Detection — Do related-work / novelty methods emit lineage edges and new-node decisions?

SP5: 관련연구 생성 및 확장/신규성 검출 — 관련연구/신규성 방법이 계보 edge와 신규-노드 결정을 방출하는가?

Papers: [61] ScisummNet, [64] CitationSum, [72] GraphMind, [67] SelectReadWrite

Expected ◈
  • Methods that generate related-work or detect extension directly identify derivation-lineage edges and inform new-node-vs-extension decisions — the coarse partner of SP4's entity-level outKB/NIL.
  • 관련연구를 생성하거나 확장을 검출하는 방법이 파생-계보 edge를 직접 식별하고 신규-노드-대-확장 결정에 정보를 준다 — SP4 개체 수준 outKB/NIL의 거친 짝.
Solved ✓
  • Concrete prompting/retrieval-adoptable primitives exist: citation-count authority weighting + span localization, ROUGE-weighted ρ=0.7 pruning, contrastive link-prediction relatedness, stance typing, and grounded novelty scoring (F1 0.75) — O1/O2 primitives are well-stocked.
  • 구체적 prompting/검색 채택 primitive가 존재: 인용 수 authority 가중 + span 국소화, ρ=0.7 ROUGE 가중 pruning, contrastive link-prediction 관련도, stance 유형화, 근거 신규성 채점(F1 0.75) — O1/O2 primitive가 풍부.
Remains ✗
  • Turning these primitives into typed edges and a graph-insertion policy is design work no paper does → SP6.
  • 이 primitive를 유형 edge와 그래프 삽입 정책으로 바꾸는 것은 어떤 논문도 안 한 설계 작업 → SP6.
Added +
  • No method emits typed dependency edges as output; retrieval of the cited set is skipped (assumed given); novelty is scored against an external proxy, never a DAG position.
  • 어떤 방법도 유형 의존 edge를 출력으로 내지 않음; 피인용 집합 검색이 생략됨(given 가정); 신규성이 DAG 위치가 아닌 외부 대리에 대해 채점됨.
Pivoted ↻
  • Assumed RWG/extension methods would produce lineage edges; inverted — they uniformly consume citation structure as an input to prose generation. The field's objective is readable text, so the graph is always an input feature, never an output.
  • RWG/확장 방법이 계보 edge를 낼 것이라 가정했으나 반전됨 — 모두 인용 구조를 산문 생성의 입력으로 소비. 분야 목표가 가독 텍스트라 그래프는 항상 입력 자질일 뿐 출력이 아님.
Revised ✎
  • SP6: target agent pipelines that OUTPUT typed edges and insert them into a persistent graph, not just generate text.
  • SP4: add emphasis on retrieving candidate prior art (the step RWG skips); consider background/target decomposition for retrieval precision.
  • SP6: 텍스트 생성만이 아니라 유형 edge를 출력하고 영속 그래프에 삽입하는 에이전트 파이프라인을 겨냥.
  • SP4: 후보 선행 검색(RWG가 생략한 단계)을 강조; 검색 정밀도를 위한 background/target 분해 고려.
6

SP6: LLM-Agent and RAG Pipelines for KG Construction — Is there a working end-to-end LLM-agent blueprint spanning O1–O3?

SP6: 지식그래프 구성을 위한 LLM 에이전트·RAG 파이프라인 — O1–O3에 걸친 작동하는 종단간 LLM 에이전트 청사진이 있는가?

Papers: [77] EDC, [83] KARMA, [135] iText2KG, [131] Graphiti, [130] GraphRAG

Expected ◈
  • This field demonstrates end-to-end agent pipelines combining extraction, linking, and graph updates across O1–O3. (Revised after SP2/3/5: probe prompting vs. supervised prerequisites, typed-edge output, and note the toolchain provides zero citation-lineage.)
  • 이 분야가 O1–O3에 걸쳐 추출·링킹·그래프 갱신을 결합한 종단간 에이전트 파이프라인을 시연한다. (SP2/3/5 이후 수정: prompting 대 지도 선행, 유형-edge 출력을 탐색하고 툴체인이 인용-계보를 전혀 제공 안 함을 명시.)
Solved ✓
  • The load-bearing spine is confirmed: open-schema extraction (O1) + embedding-threshold link-or-spawn (O2, converged across KARMA/iText2KG/Neo4j/EDC) + incremental append (O3 half). This lifts O1 and O2 to full coverage and is directly agent-adoptable today.
  • 핵심 척추 확인: open-schema 추출(O1) + 임베딩 임계 링크-대-생성(O2, KARMA/iText2KG/Neo4j/EDC 수렴) + 증분 추가(O3 절반). 이는 O1·O2를 완전 충족으로 끌어올리며 오늘 바로 에이전트 채택 가능.
Remains ✗
  • DAG structural consistency (cycle/redundancy/staleness) and citation-lineage extraction are absent from the whole toolchain → SP8/SP9 and SP1/SP5.
  • DAG 구조 일관성(cycle/중복/staleness)과 인용-계보 추출이 전체 툴체인에 부재 → SP8/SP9 및 SP1/SP5.
Added +
  • No tool enforces DAG invariants (it will create cycles and redundant edges), and no gold-standard evaluation of dependency-KB construction accuracy exists — correctness is LLM-self-judged.
  • 어떤 도구도 DAG 불변식을 강제하지 않고(cycle·중복 edge를 생성), 의존-KB 구성 정확도의 gold 평가가 없음 — 정확도는 LLM 자체 판정.
Pivoted ↻
  • Assumed the 'graph updates' half would cover O3 DAG consistency; inverted — it universally means incremental append/merge. These are property graphs built for retrieval, where cycles and redundant edges are harmless, so no one built the consistency layer.
  • '그래프 갱신' 절반이 O3 DAG 일관성을 다룰 것이라 가정했으나 반전됨 — 보편적으로 증분 추가/병합을 뜻함. cycle·중복 edge가 무해한 검색용 property graph라 아무도 일관성 층을 만들지 않음.
Revised ✎
  • SP7: compare LLM-verified similarity alignment (EDC/KARMA) against classical structural matchers, not only classical surveys.
  • SP8: add queries for structural DAG consistency and bi-temporal invalidate-not-delete stale-fact handling (Graphiti as prior art).
  • SP9: prioritize LLM-augmented/generative KG-completion (SimKGC, KG-BERT), since SP6 tools provide none.
  • SP1/SP5: emphasize they are the SOLE source of citation-lineage — SP6's toolchain provides none.
  • SP7: LLM 검증 유사도 정렬(EDC/KARMA)을 고전 서베이만이 아니라 고전 구조 매처와 비교.
  • SP8: 구조 DAG 일관성과 bi-temporal 무효화-비삭제 stale-fact 처리 질의 추가(Graphiti를 선행 사례로).
  • SP9: SP6 도구가 전혀 제공하지 않으므로 LLM-증강/생성 KG 완성(SimKGC, KG-BERT)을 우선.
  • SP1/SP5: 이들이 인용-계보의 유일한 원천임을 강조 — SP6 툴체인은 전혀 제공 안 함.
7

SP7: Ontology and Taxonomy Alignment — How to decide node equivalence vs. new-node creation at a structural level?

SP7: 온톨로지 및 분류체계 정렬 — 구조 수준에서 노드 동등성 대 신규 노드 생성을 어떻게 결정하는가?

Papers: [96] Shvaiko–Euzenat, [98] Agent-OM, [101] OAEI-LLM, [91] TaxoExpan

Expected ◈
  • Ontology/taxonomy alignment provides established matching algorithms and confidence criteria for equivalence vs. new-node above single-entity linking. (Revised after SP4/SP6: compare LLM-verified alignment vs. classical matchers; handle canonical-name-poor nodes.)
  • 온톨로지/분류체계 정렬이 단일 개체 링킹 위에서 동등성 대 신규 노드를 위한 확립된 매칭 알고리즘과 신뢰도 기준을 제공한다. (SP4/SP6 이후 수정: LLM 검증 정렬 대 고전 매처 비교; 정본 이름 빈약 노드 처리.)
Solved ✓
  • The scored typed-relation 5-tuple ⟨n,R⟩ + threshold is the equivalence-vs-new-node primitive; Agent-OM proves the pipeline runs on prompting/retrieval alone; OAEI-LLM's 'disputed mapping' is the new-node signal — the O2 structural decision primitive is supplied.
  • 채점된 유형-관계 5-tuple ⟨n,R⟩ + 임계값이 동등성-대-신규 노드 primitive; Agent-OM이 파이프라인이 prompting/검색만으로 작동함을 입증; OAEI-LLM의 'disputed mapping'이 신규 노드 신호 — O2 구조 결정 primitive가 공급됨.
Remains ✗
  • Threshold calibration for autonomous merge is uncalibrated; subsumption/typed dependency edges beyond 1:1 equivalence are deferred → SP9.
  • 자율 병합 임계값 보정이 미보정; 1:1 동등성 너머의 포함/유형 의존 edge가 미룸 → SP9.
Added +
  • No single method makes the three-way reuse-vs-insert-vs-merge decision: OM matchers only find equivalences, taxonomy expansion always inserts — the composition is unevaluated end-to-end.
  • 어떤 단일 방법도 재사용-삽입-병합 3-way 결정을 안 함: OM 매처는 동등성만 찾고 taxonomy 확장은 항상 삽입 — 합성이 종단간 미평가.
Pivoted ↻
  • Assumed an alignment method decides equivalence OR new-node; inverted — OM systems only emit equivalences (no new-node branch) while taxonomy expansion always inserts (no reuse branch). The two subfields grew from opposite starting points, so the composite three-way decision was never anyone's target.
  • 정렬 방법이 동등성 또는 신규 노드를 결정한다 가정했으나 반전됨 — OM 시스템은 동등성만 방출(신규 노드 분기 없음)하고 taxonomy 확장은 항상 삽입(재사용 분기 없음). 두 하위 분야가 반대 출발점에서 자라 3-way 합성 결정은 누구의 목표도 아니었음.
Revised ✎
  • SP8: add a query for the autonomous three-way reuse/insert/merge decision and surface the disputed-mapping signal as the new-node trigger during incremental merge.
  • SP9: add subsumption/typed-relation link prediction (BERTSubs) — SP7 matchers are equivalence-only.
  • SP8: 자율 3-way 재사용/삽입/병합 결정 질의를 추가하고 증분 병합 중 disputed-mapping 신호를 신규 노드 트리거로 노출.
  • SP9: 포함/유형-관계 링크 예측(BERTSubs) 추가 — SP7 매처는 동등성 전용.
8

SP8: Incremental Knowledge Graph Construction and Maintenance — How is a KG kept consistent as new nodes/edges arrive over time?

SP8: 증분 지식그래프 구성 및 유지 — 새 노드/edge가 시간에 따라 도착할 때 KG를 어떻게 일관 유지하는가?

Papers: [107] RE-GCN, [106] Know-Evolve, [108] Hogan, [109] TKGC Survey

Expected ◈
  • Incremental/temporal-KG literature provides DAG-maintenance procedures (cycle avoidance, redundant-edge pruning, stale-link detection). (Revised after SP6/SP7: add structural-consistency, bi-temporal, and three-way-decision queries.)
  • 증분/시계열-KG 문헌이 DAG 유지 절차(cycle 회피, 중복-edge pruning, stale-link 검출)를 제공한다. (SP6/SP7 이후 수정: 구조-일관성, bi-temporal, 3-way 결정 질의 추가.)
Solved ✓
  • Snapshot-based evolution (RE-GCN) is the transferable mechanism for encoding how structure changes across versions; Hogan supplies the consistency/validity vocabulary and refinement (completion+correction); Graphiti's bi-temporal model handles stale facts. Specification + partial mechanism for O3.
  • 스냅샷 기반 진화(RE-GCN)가 버전별 구조 변화 인코딩의 전이 가능 메커니즘; Hogan이 consistency/validity 어휘와 refinement(완성+수정)를 공급; Graphiti bi-temporal 모델이 stale 사실을 처리. O3의 명세 + 부분 메커니즘.
Remains ✗
  • Cycle detection, redundant-edge pruning, and stale-link detection algorithms are addressed by no paper here → SP9.
  • cycle 검출, 중복-edge pruning, stale-link 검출 알고리즘을 다루는 논문이 이곳에 없음 → SP9.
Added +
  • The 'temporal KG' literature solves future-fact forecasting on recurring event edges, not consistency on a monotone dependency graph; both surveys flag incremental maintenance as an open future direction.
  • '시계열 KG' 문헌은 단조 의존 그래프 일관성이 아닌 재발 사건 edge의 미래-사실 예측을 풂; 두 서베이 모두 증분 유지를 미해결 미래 방향으로 표시.
Pivoted ↻
  • Assumed the temporal/incremental-KG literature would provide DAG-maintenance procedures; inverted — 'temporal KG' means time-stamped event data where edges recur, not a monotonically-growing dependency DAG. They share the quadruple formalism but not the maintenance problem.
  • 시계열/증분-KG 문헌이 DAG 유지 절차를 제공하리라 가정했으나 반전됨 — '시계열 KG'는 단조 성장 의존 DAG가 아닌, edge가 재발하는 시각-표시 사건 데이터를 뜻함. quadruple 형식은 공유하나 유지 문제는 공유 안 함.
Revised ✎
  • SP9: elevate it to the PRIMARY owner of DAG-consistency maintenance; add queries for transitive reduction / redundant-edge removal, incremental cycle detection on insertion, and transfer of temporal-KG completion to non-recurring dependency graphs.
  • SP9: DAG 일관성 유지의 주 소유자로 격상; 이행 축소/중복-edge 제거, 삽입 시 증분 cycle 검출, 시계열-KG 완성의 비재발 의존 그래프 전이 질의 추가.
9

SP9: Link Prediction and Completion for Hierarchical/DAG Graphs — How to predict missing edges, prune redundant ones, and validate DAG structure without full retraining?

SP9: 계층/DAG 그래프 링크 예측 및 완성 — 전면 재학습 없이 누락 edge 예측, 중복 pruning, DAG 구조 검증을 어떻게 하는가?

Papers: [127] LORex, [121] DynTransReduction, [118] HAKE, [122] HEF

Expected ◈
  • Hierarchical link-prediction methods are the standard technique for missing edges, redundant edges, and DAG-structure validation without full retraining. (Elevated after SP8 to primary DAG-consistency owner; after SP6, prioritize LLM-augmented completion.)
  • 계층 링크 예측 방법이 전면 재학습 없이 누락 edge·중복 edge·DAG 구조 검증의 표준 기법이다. (SP8 이후 주 DAG 일관성 소유자로 격상; SP6 이후 LLM-증강 완성 우선.)
Solved ✓
  • Fine-tune-free retrieve-rank-verify placement (LORex, +12% over fine-tuned baselines) solves node placement; fully-dynamic transitive reduction (DynTransReduction, O(m) amortized) solves redundant-edge pruning exactly — the append+prune half of O3 is closed and training-free.
  • fine-tune-free retrieve-rank-verify 배치(LORex, fine-tuned 대비 +12%)가 노드 배치를 해결; 완전 동적 이행 축소(DynTransReduction, O(m) amortized)가 중복-edge pruning을 정확 해결 — O3의 추가+pruning 절반이 학습-free로 닫힘.
Remains ✗
  • Cycle detection/resolution on insertion is answered by NO paper; fine-tune-free general (non-leaf) completion does not exist; behavior on noisy, definition-free extracted graphs is untested. These are the residual cross-SP gaps.
  • 삽입 시 cycle 검출/해소를 어떤 논문도 답하지 않음; fine-tune-free 일반(비-leaf) 완성이 존재하지 않음; 잡음·정의-부재 추출 그래프 거동이 미검증. 이들이 잔존 교차-SP gap.
Added +
  • 'DAG link prediction' is not one field: missing/redundant/validation are served by three disjoint literatures (taxonomy expansion / graph algorithms / embedding KGC) with almost no cross-citation, and validation (cycle detection) is served by none.
  • 'DAG 링크 예측'은 하나의 분야가 아님: 누락/중복/검증이 상호 인용 거의 없는 세 분리 문헌(taxonomy 확장/그래프 알고리즘/임베딩 KGC)에 나뉘고, 검증(cycle 검출)은 어느 것도 담당 안 함.
Pivoted ↻
  • The rationale bundled 'missing edges, redundant edges, validating DAG structure' as one technique family; inverted — they are three disjoint literatures, and the strongest agent-fit method (prompt-only LORex) does ONLY leaf attachment, an inversion of the hoped-for single technique for all of O3.
  • rationale는 '누락 edge, 중복 edge, DAG 구조 검증'을 하나의 기술 계열로 묶었으나 반전됨 — 세 분리 문헌이며, 가장 에이전트-적합한 방법(prompt-only LORex)은 leaf 부착만 함, O3 전체를 위한 단일 기법 기대의 반전.
Revised ✎
  • Final SP — emits a follow_up_seed: a new SP candidate on cycle detection and repair in incrementally-built DAGs (algorithmic cycle-check on edge insertion), covered by neither link-prediction nor transitive-reduction literature.
  • 최종 SP — follow_up_seed 방출: 증분 구축 DAG의 cycle 검출·수리(edge 삽입 시 알고리즘 cycle 검사)에 관한 새 SP 후보로, 링크 예측·이행 축소 문헌 어느 것도 다루지 않음.

Convergence수렴

After SP9 (the final ranked SP), all nine SPs were exhausted and no new gap generated additional papers within the pre-ranked set. Objective coverage:

  • O1 (P1) — extraction, fully covered by SP1, SP2, SP5, SP6.
  • O2 (P1) — matching/linking, fully covered by SP3, SP4, SP7, and the load-bearing threshold from SP6.
  • O3 (P2) — DAG consistency, partial: append + transitive-reduction covered by SP8/SP9; cycle handling covered by no paper.

Remaining structural gaps (5): cycle detection/resolution (P1), reuse-vs-spawn threshold calibration (P1), sentence→edge aggregation (P1), typed-edge output (P2), noisy-graph transfer (P2). See Conclusion.

Testable hypothesis: the three-stage pipeline (open-schema extraction + ACL-ARC intent → embedding-threshold link-or-spawn at θ≈0.8 with asymmetric direction → transitive reduction + classical cycle check) maintains an acyclic, transitively-reduced DAG with new-node decisions above the S2abEL IAA ceiling.

SP9(최종 랭크 SP) 이후 아홉 SP가 모두 소진되었고 사전 랭크 집합 내에서 새 gap이 추가 논문을 생성하지 않았다. 목표 충족:

  • O1 (P1) — 추출, SP1·SP2·SP5·SP6에서 완전 충족.
  • O2 (P1) — 매칭/링킹, SP3·SP4·SP7와 SP6의 핵심 임계값에서 완전 충족.
  • O3 (P2) — DAG 일관성, 부분: 추가+이행 축소는 SP8/SP9에서 충족; cycle 처리는 어떤 논문도 미충족.

잔존 구조적 gap (5개): cycle 검출/해소(P1), 재사용-대-생성 임계값 보정(P1), 문장→edge 집계(P1), 유형-edge 출력(P2), 잡음-그래프 전이(P2). 결론 참조.
