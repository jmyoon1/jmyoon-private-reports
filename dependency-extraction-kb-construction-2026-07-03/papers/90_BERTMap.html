BERTMap: A BERT-based Ontology Alignment System

He, Chen, Antonyrajah, Horrocks · AAAI 2022 · 2021

arXiv:2112.02682 · PDF · SP: SP7

P0Track A

Problem
해결하려는 문제

Prior ML-based ontology matching (DeepAlignment, OntoEmma) uses non-contextual word embeddings (Word2Vec — one global embedding per word) or ad-hoc feature engineering requiring many annotated samples, and had not yet outperformed rule-based systems (LogMap, AML) especially in unsupervised settings. Rule-based lexical matching only considers surface form (overlapping sub-strings) and cannot capture word semantics. BERT's contextual embeddings were underexplored in OM.

표면 문자열·비문맥 임베딩 매칭은 이질적 온톨로지의 동일 개념 상이 명명을 다루지 못한다.

Solution / Contribution
해결책 / 기여

Four-stage BERTMap pipeline:

  1. Corpus construction: extract synonym / non-synonym label pairs from intra-ontology, cross-ontology (from a few known mappings), and complementary (auxiliary ontology) sources.
  2. Fine-tuning: fine-tune pre-trained Bio-Clinical BERT + a linear binary classifier on the [CLS] embedding with cross-entropy loss; output score $s\in[0,1]$ = synonymy degree.
  3. Mapping prediction: sub-word inverted index selects top-$k$ candidates; mapping score $S_{map}(c,c') = 1.0$ if labels exactly match else $S_{bert}$ = average BERT synonym score over all label pairs; pick argmax candidate.
  4. Mapping refinement: iterative extension via locality principle (Algorithm 1, threshold $\kappa=0.9$) + logic-based repair to remove inconsistent mappings.

온톨로지 라벨에서 자기지도 동의어 코퍼스를 만들어 문맥적 BERT를 미세조정하고 sub-word 역색인·logic repair를 결합한 BERTMap을 제안하며, 규칙 기반을 비지도에서 능가하나 FMA-NCI에서는 열세임을 보인다.

Key Results
주요 결과
  • FMA-SNOMED: unsupervised BERTMap beats AML by +1.4% F1, LogMap by +4.2%; semi-supervised +3.0% / +5.4%.
  • FMA-SNOMED+: BERTMap beats AML by +2.5%/+3.3%, LogMap by +1.8%/+2.7%.
  • FMA-NCI: BERTMap worse than AML by −2.5% (unsup) / −2.3% (semi), and worse than LogMap similarly — not a uniform win.
  • BERTMap without extension/repair consistently outperforms LogMapLt on all tasks; significantly beats ML baseline LogMap-ML on all three.
  • Complementary corpus gives ~50% F1 boost over label-only baselines when ontologies are label-deficient.
Concepts Introduced도입된 개념

Ontology Matching / Alignment (OM)

  • What it computes: two ontologies with named class sets $C$, $C'$ → identify cross-ontology equivalence (or subsumption) → set of scored mappings $(c, c', P(c\equiv c'))$ with $P\in[0,1]$.
  • Why it was created: independently developed ontologies use heterogeneous naming (e.g., "muscle layer" in SNOMED vs. "muscularis propria" in FMA); OM enables knowledge integration and quality assurance.
  • Concrete example: FMA-SNOMED task — 10,157 source classes, 13,412 target classes, 6,026 reference equivalence mappings.
  • How it is used in this paper: BERTMap is the OM system being proposed; the whole paper targets this task.

Text-semantics corpora (synonym / non-synonym label pairs)

  • What it computes: for each class, its preprocessed labels/aliases $\Lambda(c)$ → build pairs: synonyms (labels of same or equivalent classes), non-synonyms (labels from distinct classes) → training data for a binary classifier. Split into intra-ontology, cross-ontology, and complementary corpora.
  • Why it was created: no annotated mappings exist for supervised OM; this extracts weak supervision from ontology structure itself (unsupervised/semi-supervised).
  • Concrete example: "soft" non-synonyms = labels of two random classes; "hard" non-synonyms = labels of sibling classes (assumed disjoint).
  • How it is used in this paper: fine-tunes BERT as a synonym classifier (step ii).

Sub-word inverted index candidate selection

  • What it computes: BERT WordPiece sub-word tokens → inverted index mapping each sub-word → classes containing it; query returns target classes sharing sub-words, ranked by an idf-based score $S_{sel}$; top-$k$ retained.
  • Why it was created: naive pairwise scoring is $O(n^2)$, impractical for large ontologies; sub-word indexing reduces to $O(kn)$ and captures word forms without stemming.
  • Concrete example: candidate cut-off $k=200$.
  • How it is used in this paper: prunes the candidate space before BERT scoring (mapping prediction step iii).

Mapping extension (locality principle) + mapping repair

  • What it computes: extension = iteratively add mappings between super/subclasses of high-confidence mappings above threshold $\kappa=0.9$; repair = remove a near-minimal subset of mappings that cause logical inconsistency (Jiménez-Ruiz et al. 2013 approximate diagnosis).
  • Why it was created: neural predictions miss structurally-implied mappings and can produce logically incoherent alignments; repair is standard in rule-based OM but rare in ML approaches.
  • Concrete example: extension threshold fixed at $\kappa=0.9$ (results insensitive to it).
  • How it is used in this paper: mapping refinement (step iv), applied after BERT prediction.

5Cs Analysis5Cs 분석

Category: system / engineering (novel OM system combining neural prediction + rule-based refinement). Context: builds on BERT (Devlin 2019), Bio-Clinical BERT (Alsentzer 2019); succeeds classic OM systems LogMap and AML; contrasts with ML-OM DeepAlignment, OntoEmma, LogMap-ML. Correctness: reports P/R/Macro-F1 on 3 tasks; hold-out test splits (90% unsupervised, 70% semi-supervised); no CIs mentioned in first 5 pages. Contributions: (1) BERT fine-tuned on ontology-derived synonym corpora for OM; (2) sub-word inverted index candidate selection; (3) neural prediction + graph extension + logic repair combined; head-to-head neural-vs-rule comparison. Clarity: method well-specified with equations and Algorithm 1; code available.

Contribution vs. Claim기여 대 주장

Abstract claims BERTMap "can often perform better than LogMap and AML" — the qualifier "often" is honest: it wins on FMA-SNOMED(+) but loses on FMA-NCI. Full claim/scope gap (generality across ontology domains; only biomedical ontologies tested) requires reading the experiments section in deep mode.

Implementation Feasibility구현 실현가능성
  • Needs: OWL ontologies with labels; owlready2 (ontology processing), HuggingFace transformers, Bio-Clinical BERT; single GTX 1080Ti GPU; 3 epochs fine-tuning, batch 32.
  • Code: available at https://github.com/KRR-Oxford/BERTMap.
  • Modifications: for general/non-biomedical KB use a general BERT; need entities to carry labels/aliases; repair step needs logical axioms which a plain KB may lack.
  • Unknowns: missing link: whether extension+repair transfers to non-equivalence dependency relations; performance without curated OWL disjointness axioms.

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = two OWL biomedical ontologies (FMA, SNOMED, NCI) with rdfs:label aliases; outputs = scored + repaired equivalence mappings between named classes; conditions = unsupervised or semi-supervised (small seed mappings), class-to-class equivalence only, large ontologies (thousands of classes). Our setting (SP7, dependency extraction / KB construction): inputs = extracted entities/concepts and their dependency relations across heterogeneous sources; outputs = a merged/linked knowledge base; conditions = need to align/deduplicate entities across sources, likely more relation types than equivalence. Differences:

  • Task scope (equivalence classes only vs. dependency/subsumption relations): missing link — BERTMap focuses on equivalence; our KB needs richer relations .
  • Domain (biomedical OWL ontologies vs. general dependency KB): reference only — method is domain-agnostic but tuned with Bio-Clinical BERT.
  • Weak supervision from label synonyms: resolved / reusable — corpus-construction trick transfers if entities carry aliases/labels.
  • Logic-based repair requires OWL logical axioms: missing link — our KB may lack formal disjointness/subsumption axioms needed for repair .
