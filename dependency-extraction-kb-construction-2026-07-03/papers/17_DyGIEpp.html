Entity, Relation, and Event Extraction with Contextualized Span Representations

Wadden et al. · EMNLP 2019 · 2019 · 682 citations인용 682회

arXiv:1909.03546 · PDF · SP: SP2

P0Track A

Problem
해결하려는 문제

Prior IE systems either (a) used pipelines with hand-engineered syntactic/discourse features feeding structured predictors, or (b) modeled global context but relied on ELMo-based span graphs (DyGIE) without strong contextual encoders. Neither cleanly combined a powerful contextual encoder (BERT) with explicit cross-sentence dependency modeling in one unified multi-task framework spanning entity, relation, AND event extraction. Long-range cross-sentence relationships (e.g., coreference chains that disambiguate entity types) remained hard to capture.

개체·관계·이벤트 추출을 문장을 넘는 문맥에서 통합적으로 처리하기 어렵다.

Solution / Contribution
해결책 / 기여

A four-stage span-based architecture (DyGIE++):

  1. Token encoding: BERT with a sliding window feeding each sentence plus an L-sentence neighborhood.
  2. Span enumeration: build span vectors from endpoint tokens + width embedding.
  3. Span graph propagation: dynamically build a graph over spans; update each span via task-specific message passing (coref / relation / event propagation), combining old and new reps through a gate. Update rule: $u_x^t(i)=\sum_{j\in B_x(i)} V_x^t(i,j)\odot g_j^t$ (ReLU applied for relation extraction to enforce sparsity).
  4. Multi-task scoring: two-layer FFNNs on single spans (entity/trigger) or span pairs (relation/argument role). Trained end-to-end; coreference is an auxiliary task.

span을 열거·정제·채점하고 dynamic span graph로 문장 내·간 문맥을 전파하는 DyGIE++를 제안하여 SciERC에서 개체 67.5 / 관계 48.4 F1의 강한 SOTA를 달성한다.

Key Results
주요 결과
  • SciERC Entity F1 67.5 vs SOTA 65.2 (relative error reduction 6.6%).
  • SciERC Relation F1 48.4 vs SOTA 41.6 (+6.8 abs; 11.6% rel. error reduction) — most relevant to SP2.
  • ACE05 Entity 88.6 (vs 88.4), Relation 63.4 (vs 63.2).
  • ACE05-Event: Entity 90.7 vs 87.1 (27.9% rel. error reduction), Trigger-ID 76.5 vs 73.9.
  • GENIA Entity 77.9 (vs 76.2); WLPC Entity 79.7, Relation 65.9.
  • Ablations: CorefProp lifts SciERC NER from 69.9→72.0 (BERT+LSTM); cross-sentence propagation helps most in specialized (scientific/bio) domains.
Concepts Introduced도입된 개념

Span enumeration + span representation

  • What it computes: input = document token sequence → operation = enumerate all within-sentence spans (up to a length threshold), represent each span by concatenating its left/right endpoint token embeddings plus a learned span-width embedding → output = a fixed-dim vector per candidate span.
  • Why it was created: replaces BIO-tagging pipelines so that entities, relations, and events can all be scored over the same shared span units, enabling multi-task sharing and overlapping/nested entities.
  • Concrete example: for the span "wet lab protocol", the model concatenates the BERT vector of "wet" (left) and "protocol" (right) with a width-embedding for length 3.
  • How it is used in this paper: base unit fed into graph propagation and the multi-task scoring FFNNs.

Dynamic span graph propagation

  • What it computes: input = current span representations + model's current best-guess relations/coref/event links → operation = for each span, aggregate neighbor span vectors weighted by a task-specific similarity score $V_x^t(i,j)$, forming an update $u_x^t(i)=\sum_{j\in B_x(i)} V_x^t(i,j)\odot g_j^t$; combine with previous representation via a gating function → output = re-contextualized span vector $g^{t+1}$.
  • Why it was created: BERT captures local/adjacent-sentence context but not long-range cross-sentence dependencies; the graph propagates information along coreference/relation/event edges to cross sentence boundaries.
  • Concrete example: coreference propagation lets a span inherit information from its likely antecedent to disambiguate a hard-to-classify mention.
  • How it is used in this paper: three variants — coreference propagation, relation propagation, event propagation — inserted between BERT encoding and scoring.

Multi-task IE framework (NER + relation + event + coref-auxiliary)

  • What it computes: shared span reps → separate two-layer FFNN scoring heads: FFNN(g_i) for entity/trigger labels; FFNN([g_i, g_j]) for relation/argument-role pair labels.
  • Why it was created: prior systems handled these tasks separately with hand-engineered features; a unified span framework shares representation learning across tasks.
  • Concrete example: relation head scores the pair ("BERT", "language model") within one sentence.
  • How it is used in this paper: the core model (DyGIE++) evaluated across all four datasets.

Builds directly on DyGIE (Luan et al. 2019) — extends it by adding event extraction and multi-sentence BERT encodings.

5Cs Analysis5Cs 분석

Category: new method (system) — unified multi-task span-based IE model. Context: extends DyGIE (Luan et al. 2019, ELMo span graphs); incorporates BERT (Devlin et al. 2018); compares to Zhang et al. 2019 for events. Directly relevant to SP2 (scientific entity+relation extraction via SciERC). Correctness: reports test-set F1 across 4 datasets with a significance-test appendix (all significant except ACE05 Arg-C/Arg-ID). Event trigger detection uses a 4-model ensemble, noted transparently. Contributions: (1) unified span framework giving SOTA on all tasks / all-but-one subtask across 4 domains (relative error reductions 0.2–27.9%); (2) shows BERT captures within/adjacent-sentence context via input window size; (3) shows graph message passing adds cross-sentence signal beyond BERT, especially in specialized domains. Clarity: architecture well-described and code is public (AllenNLP); reproducible in principle.

Contribution vs. Claim기여 대 주장

Abstract claims a unified framework achieving SOTA across all three tasks and four domains, and that dynamic span graphs model long-range cross-sentence relationships. Table 1 (p.4) does show SOTA on all tasks / all-but-one subtask (ACE05 Arg-ID regresses -4.2%). The "cross-sentence" claim is empirically supported by propagation ablations but the magnitude of long-range benefit vs BERT window size needs the full analysis section to disentangle — mark for deep read.

Implementation Feasibility구현 실현가능성
  • Needs: BERT-base/large, AllenNLP, GPU for fine-tuning; gold-annotated IE corpora (SciERC available).
  • Code: available at https://github.com/dwadden/dygiepp (public, adaptable to new tasks/datasets).
  • Modifications: for SP2 KB construction, add cross-sentence/cross-document relation handling and a KB assembly/entity-linking layer; adapt label schema to target relation ontology; may need domain adaptation if no in-domain gold labels.
  • Unknowns: missing link: exact hyperparameters and appendix data splits not read in skim; degree of long-range (beyond adjacent-sentence) benefit needs the full analysis section.
Follow-up Papers관련 후속 논문

  • PURE (Zhong & Chen, 2021) — revisits pipeline vs joint span-based RE, a natural comparison baseline.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = documents (tokens); outputs = entities, relations, event triggers/arguments; conditions = supervised, gold-annotated datasets (ACE05, SciERC, GENIA, WLPC), within-sentence relations only, BERT-base/large encoder, AllenNLP. Our setting (SP2, dependency extraction for KB construction): inputs = scientific/technical text; outputs = entities + typed dependency relations forming a knowledge base; conditions = likely low/no in-domain gold annotation, need for cross-document linking. Differences:

  • Relation scope: paper handles only within-sentence relations → missing link: SP2 KB construction may need cross-sentence/cross-document relations (coref propagation partially bridges this).
  • SciERC domain overlap: resolved: SciERC is scientific-abstract entity+relation extraction, a strong template for SP2.
  • Supervision: paper is fully supervised on labeled corpora → missing link: SP2 may lack gold labels; adaptation/transfer needed.
  • Output target: paper produces per-document IE tuples, not a persistent KB → reference only: KB assembly/linking layer is out of scope here.
