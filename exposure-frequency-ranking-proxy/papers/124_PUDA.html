Prompt-Unknown Promotion Attacks against LLM-based Sequential Recommender Systems

Zhao et al. · SIGIR 2026 · 2026

arXiv:2604.23640 · PDF · SP: SP11

P0Track B

한 줄 요약

LLM 기반 순차 recommender에 대한 프롬프트 미상 승격 공격(PUDA, SIGIR 2026). 공격 문헌이 개입 레이어 자체를 겨냥한 적은 없음을 보여주는 최신 사례.

Problem해결하려는 문제

Existing promotion attacks on LLM-based sequential recommenders assume the attacker can access either the victim model's parameters or its system prompt. In real deployments both are proprietary and inaccessible, making those methods ill-posed. Additionally, text-only manipulation methods distort item semantics (making them detectable) and ignore the sequential dependencies and item co-occurrence patterns that drive interaction-level exposure.

Solution / Contribution해결책 / 기여

[SKIM - verify] Full black-box PUDA framework in two stages:

  1. Joint prompt + surrogate distillation — an LLM-driven evolutionary loop (Lamarckian init → crossover $O_C$ + mutation $O_M$ → elitism, Eqs. 1–6) infers a high-fidelity discrete prompt $P^*$; a surrogate $S^*$ is then trained via autoregressive SFT (Eq. 8) on observed data plus victim-queried synthetic sequences $\mathcal{D}_{syn}$ (perturbation operators: item swap/delete/subsequence, Eq. 7).
  2. Dual-poisoning promotion — LLM-guided similarity-gated title refinement toward anchor patterns from popular set $\mathcal{I}_{pop}$ and recommendation-frequency-derived $\mathcal{I}_{easy}$ (Eq. 9), complemented by surrogate-guided poisoning sequences reinforcing favorable co-occurrence.

Concepts Introduced도입된 개념

PUDA (Prompt-Unknown Dual-poisoning Attack) framework

  • What it computes: input = partial observable interaction data + limited query budget on a black-box LLM-SRS → operation = (1) distill the hidden system prompt and train a surrogate model, (2) dual-poison the target item via text refinement + injected sequences → output = a promotion attack that raises the exposure/ranking of an unpopular target item.
  • Why it was created: prior LLM-SRS promotion attacks (TextSimu, CheatAgent, Zhang et al.) assume access to the victim model or its system prompt — unrealistic in real deployments where both are proprietary.
  • Concrete example: attacker sees only $\mathcal{D}' \subseteq \mathcal{D}$ (e.g., a fraction of Amazon reviews) and rewrites a long-tail book title to imitate popular-item title patterns while injecting plausible fake user sequences.
  • How it is used in this paper: the whole method; two stages (§4.2 distillation, §4.3 dual-poisoning).

LLM-driven evolutionary prompt distillation

  • What it computes: input = observed demonstration pairs $\mathcal{X}=\{(s_u,y_u)\}$ → operation = Lamarckian operator seeds an initial prompt population, then crossover + mutation operators evolve it under elitism, scoring each candidate by a surrogate recommendation metric (e.g., Hit Ratio) → output = a distilled discrete instruction $P^*$ that maximally agrees with the victim.
  • Why it was created: the victim's system prompt is inaccessible; discrete prompt distillation for recommendation tasks was underexplored.
  • Concrete example: crossover "merges the strengths of two top-performing instructions"; iterates until elite score stops improving (Eq. 6).
  • How it is used in this paper: §4.2.1–4.2.2; produces $P^*$ used to train the surrogate.

Easy-to-recommend item set $\mathcal{I}_{easy}$ (recommendation-frequency anchor) [SKIM - verify in deep read]

  • What it computes: input = surrogate model $S^*$ recommendation outputs → operation = count recommendation frequency $\mathrm{Freq}_{S^*}(i)$ per item, threshold at $\gamma$ → output = set of long-tail items that the recommender nonetheless recommends frequently (Eq. 9: $\mathcal{I}_{easy}=\{i \mid \mathrm{Freq}_{S^*}(i)\ge\gamma\}$).
  • Why it was created: exposure is not just about global popularity; SRSs have inductive biases favoring certain long-tail items that align with the system, and imitating them is a cheaper route to exposure than imitating globally popular items.
  • Concrete example: an item rarely purchased but frequently surfaced in $S^*$'s recommendation lists becomes a semantic "attractor" anchor.
  • How it is used in this paper: anchor set for target-title refinement and filler items in poisoning sequences — directly relevant to this survey's exposure-frequency-as-ranking-proxy theme.

Surrogate-guided dual poisoning [SKIM - verify in deep read]

  • What it computes: input = distilled prompt $P^*$, surrogate $S^*$, target item → operation = (a) LLM three-step title refinement (anchor-pattern extraction → targeted modification → similarity-gated iteration, gated by semantic Eq.10 + lexical Eq.11 similarity) and (b) generation of behavior-preserving poisoning sequences → output = revised item text + injected fake sequences.
  • Why it was created: text-only heuristics distort item semantics and ignore sequential/co-occurrence structure; sequence-only attacks ignore item text.
  • Concrete example: refined title must keep the core named entity and pass both similarity thresholds to avoid detection.
  • How it is used in this paper: §4.3, the attack payload.

  • Domain is adversarial attack, not proxy measurement per se: reference only — but the $\mathcal{I}_{easy}$ construction (Eq. 9) operationalizes recommendation frequency as a proxy for item rankability, directly on-topic.
  • Uses surrogate model's recommendation frequency $\mathrm{Freq}_{S^*}$ as the exposure signal: missing link — need deep read to see how threshold $\gamma$ is set and how well the surrogate frequency approximates victim exposure.

Root-cause vs. Symptom근본 원인 대 증상

Direct — the paper attacks the root cause of the prior unrealistic-assumption gap by actually distilling the hidden prompt and building a surrogate, rather than assuming access; it addresses both the structural (sequence/co-occurrence) and textual channels of exposure rather than patching only the text symptom.
