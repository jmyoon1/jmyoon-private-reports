ResearchAgent: Iterative Research Idea Generation over Scientific Literature with Large Language Models

Baek et al. · NAACL 2025 · 2024 · 203 citations인용 203회

arXiv:2404.07738 · PDF · SP: SP6

P0Track A

Problem
해결하려는 문제

Automated scientific idea generation prior to this work was "localized" — hypothesis-generation methods only predict binary links between two concepts or sentence-level connections, insufficient for open-ended, multi-faceted research problems. Moreover, citation-graph-only augmentation is confined to a fixed paper set and cannot harness knowledge accumulated across the broader, disconnected literature.

과학 문헌 위에서 연구 아이디어를 반복 생성하는 에이전트가 필요하였다.

Solution / Contribution
해결책 / 기여

A knowledge-augmented LLM pipeline over scientific literature with three knowledge sources feeding a three-step idea generator $o=[p,m,d]$ (problem → method → experiment, each $f=\text{LLM}(T(\cdot))$):

  1. Citation-graph literature survey: core paper $l_0$ (selected by citation count, >20 citations) + citation-connected papers $\{l_1..l_n\}$ (narrowed by abstract similarity; avg 87 references/core paper).
  2. Entity-centric knowledge store $K$: constructed from paper titles/abstracts via BLINK entity linking + pairwise co-occurrence counting; top-$k$ external entities retrieved (Eq. 2) and injected into prompts.
  3. Iterative refinement via ReviewingAgents with human-aligned criteria.

The SP6-load-bearing contribution is source (2): a scientific KB built by extracting canonicalized entities from a paper corpus and accumulating their co-occurrence statistics into a queryable matrix.

BLINK 링킹으로 논문 유래 개체 co-occurrence 행렬을 만드는 학습-free KB 위에서 아이디어를 반복 생성하는 ResearchAgent를 제시한다.

Key Results
주요 결과
  • Full ResearchAgent outperforms Naive (core-paper-only) and w/o-Entity-Retrieval ablations on all 15 criteria across problem/method/experiment (GPT-4: 4.52 / 4.28 / 4.18 vs. Naive 4.20 / 4.03 / 3.92; Table 4).
  • vs. hypothesis-generation baselines (Table 3): ResearchAgent > SciMON and Hypothesis Proposer on Relevance (4.88), Originality (4.77), Significance (4.81).
  • Ablation (Table 2): removing entities drops scores; removing references drops more (references are the stronger source); random elements > no elements.
  • Entity gains concentrate on creativity metrics (Originality, Innovativeness).
  • Refinement saturates after ~3 iterations; smaller LLMs (Mixtral, GPT-3.5) show marginal gains from KB augmentation (Table 4).
Concepts Introduced도입된 개념

Entity-Centric Knowledge Store

  • What it computes: input = titles+abstracts of a large corpus of scientific papers (papers after May 2023, 50,091 total) → an off-the-shelf entity linker (BLINK, Wu et al. 2020) tags and canonicalizes entities per paper, then all intra-paper entity pairs are counted → output = a sparse 2D co-occurrence matrix $K \in \mathbb{R}^{m \times m}$ ($m$ = unique entities) storing pairwise co-occurrence counts plus per-entity counts.
  • Why it was created: the vanilla citation-graph-only pipeline relies exclusively on a fixed set of papers (core + citations); scientific knowledge is spread across many disconnected publications, so a paper-derived, entity-level store is needed to surface cross-domain concepts (e.g., linking "database" from medicine to hematology) not reachable via citations.
  • Concrete example: matrix entry (GPT-3, CoT) = 17,326 co-occurrences; abstract of each paper yields 2.17 entities on average (~3 per paper via BLINK).
  • How it is used in this paper: given entities extracted from the core+related papers, retrieve top-$k$ external entities via co-occurrence-based conditional probability (Eq. 2, Bayes + independence approximation), then inject them into the LLM prompt to broaden idea generation. Directly the SP6-relevant artifact: a KB constructed from papers via entity extraction + co-occurrence.

Entity co-occurrence retrieval (Eq. 1-2)

  • What it computes: $\text{Ret}(\{l_0..l_n\};K) = \arg\max_{I}\prod_i P(e_i|E_{\{l_0..l_n\}})$, approximated via Bayes as $\arg\max \prod (\prod_{e_j} P(e_j|e_i)) P(e_i)$, with $P(e_j|e_i)$ and $P(e_i)$ read off the normalized matrix $K$.
  • Why: to rank external entities by their statistical relatedness to the current paper's entities.
  • Concrete example: retrieved "Drosophila Genetic Reference Panel" + "CRISPR" enabled a novel cross-topic idea (Table 16).
  • How used: alternative embedding-based retrieval (using BLINK entity embeddings) gives comparable results (Table 5, App. B.3).

ReviewingAgents / human-induced evaluation criteria

  • What it computes: LLM reviewing agents score each generated idea (problem/method/experiment) on 5 criteria each; criteria descriptions are induced by prompting GPT-4 with 10 human-annotated idea-score pairs on a 5-point Likert scale.
  • Why: raw LLM judgments diverge from human preferences; no ground-truth references exist.
  • How used: feedback drives iterative idea refinement (saturates after ~3 iterations). Peripheral to SP6's KB-construction focus.

5Cs Analysis5Cs 분석

Category: system / new method (LLM agent + paper-derived KB) Context: builds on knowledge-augmented LLMs (RAG, entity-augmentation of Baek et al. 2023/2024), literature-based discovery (Swanson 1986), LLM self-refinement (Self-Refine, Madaan 2023), and multi-agent debate (Du 2023). High-citation (203) P0 anchor for the scientific-KB objective. Correctness: no ground-truth; relies on human + GPT-4 evaluation with reported inter-annotator agreement (Spearman 0.67-0.83; Cohen's κ 0.41-0.62) and human-model agreement. Reasonable rigor for a subjective task but no confidence intervals on main scores. Contributions: (1) open-ended research-idea-generation task formulation; (2) entity-centric paper-derived knowledge store + co-occurrence retrieval; (3) human-preference-aligned LLM reviewing agents. Clarity: KB construction (entity linker + co-occurrence matrix) is reproducible; code released. Prompt templates fully given (Tables 6-15).

Contribution vs. Claim기여 대 주장

Abstract claims a KB "derived from shared underlying concepts mined across numerous papers" enabling cross-domain idea cross-pollination. Experiments demonstrate co-occurrence-matrix entity retrieval improves idea scores, but the KB coverage is admittedly limited (titles/abstracts only, ~3 entities/paper from an open-domain linker; Limitations). Whether the store genuinely captures "cross-domain concept relationships" vs. surface co-occurrence is not directly validated — a gap to check in deep read.

Implementation Feasibility구현 실현가능성
  • Needs: Semantic Scholar Academic Graph API; BLINK entity linker (Wu et al. 2020); GPT-4 (or open LLMs — quality drops sharply below GPT-4-class); sparse-matrix storage for $K$.
  • Code: available at https://github.com/JinheonBaek/ResearchAgent
  • Modifications: to serve dependency-extraction/KB goals, replace co-occurrence matrix with a typed relation extractor; extend entity extraction beyond titles/abstracts to full text; add intrinsic KB-quality evaluation.
  • Unknowns: missing link: normalization details of P(e_j|e_i) from K; missing link: how entity canonicalization errors propagate (deferred to deep read).

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = scientific paper corpus (titles+abstracts, Semantic Scholar); outputs = a sparse entity-entity co-occurrence matrix + generated research ideas; conditions = open-domain entity linker (BLINK), no relation typing, entities = atomic knowledge units, corpus restricted to post-May-2023 papers to avoid LLM training-data leakage. Our setting: inputs = scientific papers/documents; outputs = a knowledge base with typed dependencies/relations between extracted entities; conditions = focus on dependency/relation extraction quality and KB structure. Differences:

  • KB is co-occurrence-only, no typed relations/dependencies: missing link — our survey targets dependency extraction; this paper has undirected, untyped entity pairs only.
  • Entity extraction limited to titles+abstracts via off-the-shelf open-domain linker (~3 entities/paper): reference only — a low-recall baseline for paper-to-entity extraction.
  • Co-occurrence as the sole "relationship" signal: reference only — shows a lightweight, training-free KB-construction recipe that scales, usable as a baseline against richer dependency-extraction methods.
  • Downstream use is idea generation, not KB accuracy: missing link — no intrinsic KB-quality evaluation.
