Investigating Active Learning for Concept Prerequisite Learning

Chen Liang et al. · AAAI 2018 (EAAI) · 2018 · 76 citations인용 76회

SP: SP3

P0Track A

Problem
해결하려는 문제

Concept prerequisite learning at scale is blocked by the lack of large-scale prerequisite labels needed to train effective data-driven classifiers. Prior work either used student-assessment data (requires pre-set item↔knowledge-component mappings, not scalable) or single hand-designed metrics (e.g. RefD). No prior work had applied active learning to reduce labeling cost for this task, and no systematic feature set / query-strategy comparison existed.

선행관계 분류기를 부트스트랩할 때 라벨 비용이 크다.

Solution / Contribution
해결책 / 기여

Empirical study. Two components:

  1. Feature set — 32 graph-based + text-based features over Wikipedia link structure and article content for representing a concept pair $(A,B)$.
  2. Active-learning comparison — reformulate four query strategies into a common greedy criterion

$$s = \arg\max_{s\in D_u}\ \min_{y\in\{-1,1\}} f(s;y,D_l)$$ and compare (Random, LC, QBC, QUIRE, Diversity) with Random Forest as the base classifier, measuring AUC vs. number of queried labels.

Finding: Random Forest is the best base classifier; query-by-committee (QBC) consistently gives the largest label-efficiency gain; diversity sampling ≈ random; QUIRE < random.

32개 그래프·텍스트 자질과 query-by-committee active learning을 결합하고 RefD를 최상위 신호로 규명하여 라벨 효율적 부트스트랩과 비대칭 방향 신호를 제공한다.

Key Results
주요 결과
  • Random Forest best base classifier across all 4 domains, beating NB/LR/SVM by F1 +5.6% / +13.7% / +3.1% / +3.8% and AUC +6.3 / +6.1 / +4.7 / +2.1 (Data Mining / Geometry / Physics / Precalculus). RF AUC: 92.2 / 97.8 / 93.9 / 97.5.
  • QBC consistently outperforms all other query strategies on all four domains (AUC learning curves, 300-trial avg).
  • Diversity sampling not significantly different from random; QUIRE performs worse than random (and is far slower due to large matrix inverse/determinant).
  • Feature analysis: graph-based features (PageRank diff, HITS authority, RefD, in/out degree) dominate top-10; symmetric/similarity features (tf-idf, word2vec, PMI, NGD) rank low in the in-domain setting.
  • Datasets: 4 domains, 120–224 concepts, 826–2060 pairs, 292–699 positive prerequisites each.
Concepts Introduced도입된 개념

Concept prerequisite relation (learning)

  • What it computes: input = an ordered concept pair $(A, B)$ → binary classifier → output = whether $A$ is a prerequisite of $B$ (must be learned before $B$). Formally a directed, irreflexive, transitive (strict partial order) relation over a concept set.
  • Why it was created: prior educational systems needed prerequisite structure for curriculum planning / intelligent tutoring but obtained it manually; this frames it as a scalable ML classification task.
  • Concrete example: "posterior probability" and "maximum likelihood" are prerequisites of "Hidden Markov model" (Figure 1).
  • How it is used in this paper: the target task — all features and active-learning strategies are evaluated on predicting this pairwise relation over 4 Wikipedia-derived domains.

Pool-based active learning

  • What it computes: maintains labeled set $D_l$ and unlabeled pool $D_u$; each round a query strategy selects instance $s = \arg\max_{s\in D_u}\min_{y\in\{-1,1\}} f(s;y,D_l)$, its label is queried from an oracle, and moved to $D_l$; classifier retrained.
  • Why it was created: to learn a good classifier with far fewer labels — directly targets the paper's stated obstacle (lack of large-scale prerequisite labels).
  • Concrete example: start with 20 seed labels, query one pair at a time, re-train RF, evaluate AUC on held-out test.
  • How it is used in this paper: the framework under study; four query strategies compared against random.

Query strategies (uncertainty / least-confident, query-by-committee (QBC), QUIRE, diversity sampling)

  • What it computes: four scoring functions $f(s;y,D_l)$ all reformulated into the greedy Eq. (1). Uncertainty = $1-P(y|x_s)$; QBC = committee disagreement count $\sum_k \mathbb{1}[y=g^{(k)}(x_s)]$; QUIRE = max-margin informativeness+representativeness via kernel matrix inverse; diversity = lowest average cosine similarity to labeled set.
  • Why it was created: to compare informativeness- vs. representativeness- vs. diversity-based sampling on this task.
  • Concrete example: QBC uses query-by-bagging with a committee of 3 decision trees.
  • How it is used in this paper: head-to-head comparison; QBC wins.

RefD and the graph/text feature set (32 features)

  • What it computes: 15 graph-based features (in/out degree, common neighbors, # links, link proportion, NGD, PMI, RefD, PageRank diff, HITS hub/authority diff) + 17 text-based features (1st-sentence containment, in-title, title Jaccard, length, mention counts, noun-phrase counts, tf-idf sim, word2vec sim, LDA entropy, LDA cross-entropy) computed from an Oct-2016 Wikipedia dump.
  • Why it was created: a "novel set of features tailored for prerequisite classification" — earlier work relied on single metrics (e.g. RefD alone) or student-assessment data not applicable at scale.
  • Concrete example: RefD($A,B$) (Liang et al. 2015) measures how differently $A$ and $B$'s related concepts link to each other; here it is one of the top-ranked features across all four domains.
  • How it is used in this paper: input representation for concept pairs; feature-importance analysis (mean decrease impurity) shows graph features (PageRank diff, authority, RefD) dominate over text features.

5Cs Analysis5Cs 분석

Category: benchmark / empirical study (feature engineering + method comparison) Context: builds on RefD (Liang et al. 2015), Wiki concept map dataset (Wang et al. 2016), Gordon et al. 2016 (LDA cross-entropy feature); the canonical task-definition reference. Predecessor to the follow-up "Active Learning of Strict Partial Orders" (arXiv:1801.06481), which adds relational reasoning the current paper does not use. Correctness: 5-fold CV for classification; active-learning results averaged over 300 random seeds with 95% CIs reported — statistically careful. Contributions: (1) first application of active learning to concept prerequisite learning; (2) a novel 32-feature set for prerequisite classification with importance analysis; (3) empirical comparison identifying QBC + Random Forest as the effective combination. Clarity: reproducible — features, classifiers (RF 200 trees, LR C=1.0, linear SVM), and AL protocol (20 seeds, 2:1 split, 300 seeds) fully specified.

Contribution vs. Claim기여 대 주장

  • Stated claims: active learning reduces training data required; proposed features are effective; QBC is the best query strategy.
  • Demonstrated: QBC beats random/LC/QUIRE/diversity on AUC across 4 domains (300 trials) — supports the AL-reduces-labels and QBC claims with numbers.
  • Gap: all four domains are Wikipedia-derived and evaluated in-domain; the paper itself notes symmetric features "expect to be more important in a cross-domain setting" — cross-domain generalization is hypothesized, not tested. The strict-partial-order structure is used only for dataset expansion, not exploited in the query strategies (that is the follow-up paper).

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = ordered Wikipedia concept pairs with 32 hand-engineered graph+text features; outputs = binary prerequisite label; conditions = 4 small in-domain educational datasets, RF classifier, pool-based single-instance active learning, oracle = human/expert labels. Our setting (SP3, dependency/prerequisite extraction for KB construction): inputs = concept/entity pairs from a corpus (plausibly text spans or LLM-embedded representations); outputs = directed dependency edges for a knowledge base; conditions = larger/open-domain concept space, likely neural/LLM extraction rather than hand-crafted features. Differences:

  • Feature engineering vs. learned representations: reference only — the 32-feature taxonomy (esp. RefD, PageRank diff, link-proportion) is a reusable signal inventory but predates LLM embeddings.
  • Wikipedia link structure dependency: missing link — our KB target may lack a dense hyperlink graph; graph features may not transfer.
  • Label-scarcity remedy (active learning): resolved: how — QBC-style informativeness sampling is directly reusable to cut annotation cost when bootstrapping a dependency KB.
  • In-domain only vs. open/cross-domain: missing link — cross-domain transfer untested here.
  • Strict-partial-order constraint used only for data expansion: reference only — the transitivity/irreflexivity closure trick is borrowable for consistency in KB construction.
