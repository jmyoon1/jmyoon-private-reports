TaxoEnrich: Self-Supervised Taxonomy Completion via Structure-Semantic Representations

Jiang, Wang, Zhou, Xiong, et al. · WWW 2022 · 2022

arXiv:2202.04887 · PDF · SP: SP7

P0Track A

Problem
해결하려는 문제

Prior taxonomy-augmentation work is limited on two axes: (1) expansion methods (TaxoExpan, STEAM) only attach new concepts as leaves, missing internal insertions; (2) embedding choices are weak — static embeddings (Word2Vec/FastText) can't separate semantically-close neighbors, and the recent LM approach (GenTaxo) pairs good embeddings with a lightweight MLP matcher that has limited representation power. No prior method jointly exploits full vertical (ancestor/descendant) and horizontal (sibling) structure with strong semantics for completion.

leaf 부착만으로는 두 기존 노드 사이 내부 삽입을 다루지 못한다.

Solution / Contribution
해결책 / 기여

TaxoEnrich = 4 components (empirical framework, not a single equation):

  1. Taxonomy-contextualized embedding: Hearst-pattern pseudo sentences (ancestral + descendant) → frozen SciBERT → averaged token embedding $x$.
  2. Sequential feature encoder: LSTM over sampled paths → vertical-aware parent/child embeddings $n^{(p)}=x\oplus\mathrm{LSTM}(\cdot)$, $n^{(c)}=x\oplus\mathrm{LSTM}(\cdot)$.
  3. Query-aware siblings encoder: bilinear query-sibling relatedness → softmax → weighted-sum siblings embedding $n^{(s)}$.
  4. Query-position matching: NTN-based scorers combining fine- and coarse-grained relatedness (extends TMN).

부모와 자식을 함께 찾아 내부 삽입하는 taxonomy completion을 SciBERT pseudo-sentence 임베딩으로 수행하는 TaxoEnrich를 제안한다.

Key Results
주요 결과
  • Scaled MRR (×10) on MAG-PSY: TaxoEnrich 0.531 ± 0.007 vs. best baseline 0.464 and vs. static-embedding baseline 0.032.
  • WordNet-Verb MRR: 0.428 (large margin over baselines ~0.354).
  • Claims SOTA "by a large margin" on all metrics (MR, MRR, Recall@k, Precision@k) across 4 datasets (MAG-CS, MAG-PSY, WordNet-Noun, WordNet-Verb).
  • Also outperforms on taxonomy expansion (special case, leaf-only) vs. TaxoExpan, TMN, GenTaxo.
Concepts Introduced도입된 개념

Taxonomy Completion (vs. Expansion)

  • What it computes: input = existing taxonomy $T^0=(N^0,E^0)$ + set of new concepts $C$ → operation = predict the best $\langle$hypernym, hyponym$\rangle$ pair (candidate position) for each new concept → output = enriched taxonomy $T=(N,E)$ with concepts inserted at internal or leaf positions.
  • Why it was created: taxonomy expansion only attaches new concepts as leaves (finds a parent); completion also finds a child, allowing insertion between existing nodes.
  • Concrete example: query "Integrated Circuit" → parent "Hardware", child "GPU" (inserted mid-hierarchy, not as a leaf).
  • How it is used in this paper: the target task; framework follows the completion formulation of TMN [34].

Taxonomy-Contextualized Embedding (pseudo-sentence)

  • What it computes: input = concept node + its ancestral/descendant paths → build "pseudo sentences" via Hearst patterns (e.g., "Electronic Devices, Smart Phone is a superclass of Disk") → feed to a frozen pretrained LM (SciBERT) → average last hidden states of the concept token → output = fixed embedding $x$.
  • Why it was created: static embeddings (Word2Vec/FastText) can't distinguish semantically-close neighbors; prior LM approaches (GenTaxo) used only limited neighbors and required fine-tuning. This fuses all ancestor/descendant context without fine-tuning the LM.
  • Concrete example: 227,609 pseudo sentences generated for the MAG-CS taxonomy.
  • How it is used in this paper: pre-computed, fixed input to all downstream encoders.

Sequential Feature Encoder (vertical view)

  • What it computes: samples an ancestral/descendant path, runs an LSTM over its pseudo-sentence, concatenates the final hidden state with $x$ → parent embedding $n^{(p)}$ and child embedding $n^{(c)}$.
  • Why it was created: to encode "depth" / vertical structural position of a candidate.
  • Concrete example: $n^{(p)} = x \oplus \mathrm{LSTM}(s; \theta_1)$.
  • How it is used in this paper: produces candidate-position node representations feeding the matcher.

Query-Aware Siblings Encoder (horizontal view)

  • What it computes: given query embedding $q$ and candidate siblings (children of candidate parent), a bilinear matrix scores each sibling's relatedness to the query conditioned on the parent-child pair; softmax-normalize; weighted-aggregate into one siblings embedding $n^{(s)}$.
  • Why it was created: siblings add horizontal context but are noisy and can be numerous (thousands under high-level nodes); needs selective attention rather than including all/random siblings.
  • Concrete example: "Artificial Intelligence" in MAG-CS has thousands of children — cannot include all.
  • How it is used in this paper: augments position representation before matching.

Query-Position Matching Model (NTN scorers)

  • What it computes: multiple Neural Tensor Network scorers compute fine-grained (query→parent/child/siblings) and coarse-grained (query→position) relatedness → matching score.
  • How it is used in this paper: extends TMN [34] matcher with the new position representations.

Category: new method (system/framework for taxonomy completion) Context: builds on TMN [34] (completion formulation + matcher), TaxoExpan [19] (position-enhanced GNN), STEAM [30] (mini-paths), GenTaxo [31] (LM embeddings), ARBORIST [12]. Correctness: reports mean ± std over runs (e.g., MRR 0.531 ± 0.007), four datasets — appears statistically careful. Full rigor . Contributions: (1) taxonomy-contextualized pseudo-sentence embedding usable generally; (2) sequential encoder for vertical structure; (3) query-aware sibling encoder for horizontal structure; (4) SOTA on completion + expansion. Clarity: equations and pseudo-sentence construction described; reproducibility of exact NTN scorer config .

Contribution vs. Claim기여 대 주장

— Abstract claims "best performance among all evaluation metrics ... by a large margin" on 4 datasets and generality of the embedding approach. Whether the sibling encoder and each sub-module contribute independently (vs. embedding alone driving gains) requires reading the ablation (Tables 4–5); early skim suggests ablations exist and support component-wise gains, but the exact attribution needs the experiments section.

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = a curated existing taxonomy (tree/DAG of "is-a" edges) + given set of new concepts already extracted; outputs = predicted $\langle$parent, child$\rangle$ insertion position per concept; conditions = single-domain taxonomy, concepts assumed in-domain, embeddings pre-computed with frozen SciBERT, supervision is self-supervised (positions sampled from existing structure). Our setting (SP7, dependency extraction / KB construction): inputs = extracted concepts/entities + relations from a corpus; need to decide equivalence vs. insertion within an evolving hierarchy — i.e., where a new concept belongs relative to existing ones, or whether it duplicates one. Differences:

  • Assumes new concepts are already extracted (entity extraction out of scope): missing link — our SP must produce $C$ upstream.
  • Assumes concepts are in the same semantic domain as $T^0$: reference only — our KB may be cross-domain.
  • Provides parent+child anchoring (not just leaf attachment): resolved/directly supports the "insertion vs. equivalence" decision SP7 needs.
  • Self-supervised from existing taxonomy structure: resolved/no labeled hypernym pairs required — attractive for bootstrapping.
  • Does not model equivalence/merge explicitly (only insertion): missing link — SP7's "equivalence" decision is adjacent but not the paper's target.
