SemEval-2018 Task 7: Semantic Relation Extraction and Classification in Scientific Papers

Gabor et al. · SemEval 2018 (ACL Workshop) · 2018 · 184 citations인용 184회

PDF · SP: SP2

P0Track A

Problem
해결하려는 문제

Before this task there was no standard benchmark for extracting and classifying domain-specific, contextual semantic relations between technical concepts in scientific text. Existing SemEval relation tasks were either domain-agnostic (2010 Task 8), context-free (2012 Task 2), or offered only coarse taxonomic relations (2017 Task 10) — none captured the fine-grained relations (a method is used for a task, a phenomenon is found in a context, results are compared) that define the information content of a scientific abstract.

과학 논문 내 개념 간 의미 관계를 추출·분류하는 표준 평가가 필요하였다.

Solution / Contribution
해결책 / 기여

Empirical/benchmark contribution. The organizers built a shared task over ACL Anthology abstracts:

  • Data: derived from ACL RD-TEC 2.0 (manual entity annotation → clean subtasks 1.1/2) and ACL-RelAcS (automatic entity annotation → noisy subtask 1.2). Training = 350 annotated abstracts (~100 words each); test = 150 abstracts. Relations annotated only between already-annotated entities, within a single sentence, and only when explicit.
  • Task: classify/extract relations into the six-type typology with directionality.
  • Evaluation: per-class P/R/F1; official ranking by macro-average F1 (subtask 1.1/1.2); subtask 2 evaluated in two stages (extraction F1 ignoring labels, then classification macro-F1 over correctly extracted pairs).
  • Baseline: memory-based k-NN (k=5) over 500-dim positive-random-projection context vectors weighted by PPMI, with majority voting.

6-way 관계(USAGE/RESULT/MODEL-FEATURE/PART-WHOLE/TOPIC/COMPARE)와 3개 하위과제로 구성된 SemEval-2018 Task 7을 정의하며, gold 개체 제거 시 추출+분류가 20.34 macro-F1로 붕괴함을 통해 추출 병목을 정량화한다.

Key Results
주요 결과
  • 32 participants, 158 submissions across subtasks; subtask 1.1 most popular (28 teams), 1.2 (19), subtask 2 (11).
  • Best-ranking system on subtask 1.1 reached macro-F1 0.81 vs human upper bound 0.91.
  • Average system macro-F1: 47.28 (1.1 clean classification), 62.51 (1.2 noisy classification), 30.8 (extraction, subtask 2), 20.34 (extraction+classification, subtask 2) — relation-instance identification is by far the hardest step.
  • Top-5 systems' per-relation F1 on 1.1 ranged 0.85 (easiest) down to 0.59; PART_WHOLE (F1=0.83) and RESULT (0.89) were the hardest relations even for human annotators.
  • Method popularity: CNN (10 systems), LSTM (5), SVM (5); domain-specific embeddings (ACL/arXiv) improved performance. UWNLP — the only system treating complete specialized entities as semantic units — ranked first in relation extraction.
Concepts Introduced도입된 개념

Six-way domain-specific semantic relation typology (USAGE, RESULT, MODEL-FEATURE, PART_WHOLE, TOPIC, COMPARE)

  • What it computes: input = an ordered pair of annotated entities (concepts) co-occurring within one sentence of a scientific abstract → operation = assign one of six mutually exclusive relation labels plus a directionality (ARG1, ARG2) → output = a typed, directed relation instance, e.g. MODEL-FEATURE(H01-1041.10, H01-1041.11).
  • Why it was created: prior shared tasks were not tailored to scientific text — SemEval-2010 Task 8's 9 relations were domain-agnostic; SemEval-2012 Task 2 treated entity pairs as context-free static instances; SemEval-2017 Task 10 offered only 3 coarse taxonomic relations (synonym/hypernym/unrelated). This task provides a finer-grained, contextual, science-specific typology derived from a data-driven corpus study.
  • Concrete example: USAGE links a method to a task (MT system — Japanese); RESULT links a property to an outcome (order — performance); COMPARE links two results (result — standard).
  • How it is used in this paper: it is the label space for all three subtasks; the six coarse types are an automatic collapse of a finer-grained annotation typology used by human annotators.

Three-subtask evaluation design (1.1 clean classification, 1.2 noisy classification, 2 extraction+classification)

  • What it computes: partitions the pipeline into (1.1) classify relations given manually annotated entities and gold relation instances; (1.2) same but on automatically annotated ("noisy") entities; (2) both identify which entity pairs are related and classify them, given only annotated entities (no gold relation instances at test).
  • Why it was created: to isolate and quantify the effect of each pre-processing step (entity-annotation noise; relation-instance identification) on downstream classification accuracy.
  • Concrete example: average system F1 = 47.28 (1.1) vs 62.51 (1.2) vs 30.8 (extraction only in 2) vs 20.34 (extraction+classification in 2), showing relation-instance identification is the hardest step.
  • How it is used in this paper: the official ranking metric is macro-averaged F1 per subtask; the design directly supports the paper's conclusion about which pipeline step dominates error.

5Cs Analysis5Cs 분석

Category: benchmark / shared-task evaluation. Context: builds on ACL RD-TEC 2.0 (QasemiZadeh & Schumann 2016) and ACL-RelAcS (Gábor et al. 2016) corpora; positioned against SemEval-2010 Task 8, 2012 Task 2, 2017 Task 10. In this survey it is the foundational SP2 benchmark for scientific relation extraction, alongside SciERC-style datasets. Correctness: rigorous annotation protocol (3-week annotator training, double annotation). Inter-annotator agreement: overall class-label agreement 90.8%, macro-F1 0.91 against one annotator as gold — the best system (0.81) sits ~10 points below human ceiling, a valid headroom signal. Class distribution is heavily imbalanced and differs between train/test, which the authors flag as the main difficulty driver. Contributions: (1) the six-way domain-specific relation typology; (2) an annotated ACL-abstract dataset (350 train / 150 test); (3) the three-subtask decomposition isolating extraction vs classification error. Clarity: task setup, metrics, and data provenance are reproducible from the description; the full dataset was released post-competition at the organizers' site.

Implementation Feasibility구현 실현가능성
  • Needs: the released dataset (abstracts + entity + relation annotations) from the organizers' site (lipn.univ-paris13.fr/~gabor/semeval2018task7/); standard IE tooling; the six-type label schema.
  • Code: dataset available (post-competition release); baseline is described but no official code repo cited in the abstract/intro .
  • Modifications: for our KB-construction setting we would need a joint entity-extraction front-end (SemEval provides gold/auto entities), full-text (not abstract-only) coverage, and an LLM-prompting classifier mapped onto the six relation types.
  • Unknowns: missing link: exact per-system architectures and whether the six types map cleanly onto prerequisite/dependency relations needed for SP2 — requires reading participant system papers and the methods appendix.
Follow-up Papers관련 후속 논문
  • SciERC / Multi-Task Identification of Entities, Relations, and Coreference (Luan et al., 2018) — the parallel scientific IE benchmark with joint entity+relation extraction on full sentences; complements this task for SP2.
  • Individual SemEval-2018 Task 7 participant systems (e.g. UWNLP, ETH-DS3Lab) — report the concrete CNN/LSTM architectures that set the SOTA F1 referenced here.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = scientific abstracts (~100 words, computational-linguistics domain) with pre-annotated entities; outputs = six-way typed+directed relation labels per entity pair; conditions = supervised training on 350 abstracts, single-sentence explicit relations only, macro-F1 evaluation. Our setting: inputs = full paper body text (not just abstracts) in a general/ML-research domain; outputs = concept dependency signals (uses/extends/requires-style relations) feeding KB construction; conditions = LLM-prompting-based extraction, ideally without task-specific training, entities not pre-given. Differences:

  • Entities pre-annotated here vs must-be-extracted in our pipeline: missing link (SemEval assumes gold or auto entity spans; our setting needs joint entity+relation extraction — subtask 2's low 20.34 F1 quantifies this gap).
  • Abstract-only, single-sentence, explicit relations vs full-text, possibly cross-sentence dependencies: reference only (typology transfers as a label vocabulary but coverage differs).
  • Supervised CNN/LSTM/SVM systems vs LLM prompting: reference only (baseline numbers are a target/ceiling reference, not directly adoptable methods).
  • Domain = computational linguistics vs general ML: resolved: relation typology (USAGE/RESULT/MODEL-FEATURE/PART_WHOLE/TOPIC/COMPARE) is domain-general enough to reuse as a candidate dependency label set.
