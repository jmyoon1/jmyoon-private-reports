S2abEL: A Dataset for Entity Linking from Scientific Tables

Lou et al. · EMNLP 2023 · 2023 · 6 citations인용 6회

arXiv:2305.00366 · PDF · SP: SP4

P0Track A

Problem
해결하려는 문제

Existing entity-linking datasets and methods (a) target free-form text or general-domain web tables, (b) assume a closed-world KB where every mention has a referent, and (c) prior scientific-table work (leaderboard construction, e.g. AxCell/Kardas 2020) only extracts raw strings without canonicalization or linking to an external KB. None handle scientific tables where the KB is highly incomplete (42.8% outKB), mentions are abbreviated/opaque, and disambiguation requires the surrounding paper text — so there is no dataset or method for reuse-vs-novel decisions on scientific table entities.

과학 KB는 매우 불완전하여 mention 중 다수가 KB에 없는 outKB 개체이나 표준 EL은 closed-world를 가정한다.

Solution / Contribution
해결책 / 기여

Empirical/dataset contribution + neural baseline:

  1. S2abEL dataset: hand-labeled ML results tables from PapersWithCode-linked papers — 8,429 cells from 732 tables, with 52,257 cell-type, 9,564 attributed-source, and 8,429 entity-link annotations (incl. explicit outKB labels). Built by bootstrapping SegmentedTables (Kardas 2020) for CTC, a T5-small reference parser + Semantic Scholar lookup for ASM, and a Flask annotation interface with three IAA training rounds (Cohen's Kappa: CTC 100%, ASM 82%, EL 66%).
  2. Neural baseline over sub-tasks: SciBERT with a trainable segment-type embedding + BM25-retrieved context sentences for CTC (cross-entropy, minority oversampling); SciBERT pair encoder for ASM (BCE, non-attributed sources as negatives); Candidate Entity Retrieval combining dense bi-encoder retrieval (DPR-style, triplet loss) with attributed-source retrieval; and Entity Disambiguation with outKB identification.

과학 표에서 outKB 검출(42.8%)을 일급 라벨 과제로 만든 S2abEL 데이터셋을 도입하고 링킹을 4단계로 분해하며 attributed-source matching으로 근접 표면형을 구분한다.

Key Results
주요 결과
  • 42.8% of entity mentions are outKB (3,610 outKB vs 4,819 inKB) — establishes the incomplete-KB regime. [verified from p.3]
  • Baseline "significantly outperforms TURL, a SOTA general-domain table EL method" — exact deltas not on pp.1–5.
  • "Best baselines fall below human performance" — quantified gap .
  • CTC prunes ~74% of cells (numeric result cells) before linking. [verified from p.3]
Concepts Introduced도입된 개념

outKB mention (out-of-knowledge-base mention)

  • What it computes: given a table cell mention + candidate entity set from a target KB → classify whether the mention refers to any existing KB entity or to none (label "outKB") → binary/decision output feeding entity disambiguation.
  • Why it was created: rejects the traditional "closed-world" EL assumption that every mention has a KB entry; scientific KBs are highly incomplete (42.8% of mentions here are outKB), so a system must decide reuse-existing vs. novel.
  • Concrete example: S2abEL contains 3,610 outKB vs. 4,819 inKB mentions; e.g. a table cell naming a brand-new model absent from PapersWithCode → outKB.
  • How it is used in this paper: the Entity Disambiguation (ED) sub-task jointly predicts referent entity OR outKB; distinguishing an unusual surface form of a long-tail inKB entity from a genuine outKB mention is framed as the core difficulty.

Attributed Source Matching (ASM)

  • What it computes: given a table cell + the document's reference section (+ the document itself) → identify which reference paper(s) the authors attribute the concept to → set of source papers per cell (BCE-scored via SciBERT).
  • Why it was created: similar surface forms (e.g. "BlenderBot 1" vs "R2C2 BlenderBot") can only be disambiguated by knowing which cited paper introduced each; disambiguation needs paper-level attribution, not just string matching.
  • Concrete example: "BlenderBot 1" → attributed to Roller et al. 2021; "R2C2 BlenderBot" → Shuster et al. 2022, resolving two near-identical surface forms.
  • How it is used in this paper: a dedicated sub-task and also a candidate-retrieval signal (attributed-source retrieval, ASR) combined with dense retrieval.

Task decomposition: CTC → ASM → CER → ED

  • What it computes: pipeline of four sub-tasks — Cell Type Classification (method/dataset/metric/dataset&metric or non-entity), Attributed Source Matching, Candidate Entity Retrieval, Entity Disambiguation with outKB identification.
  • Why it was created: end-to-end table EL in science is too hard to model monolithically; decomposition lets CTC prune ~74% non-entity cells (numeric results) and lets each stage reduce downstream cost.
  • Concrete example: CTC removes numeric result cells; only positively-typed cells get ASM/EL annotations (52,257 CTC vs 9,564 ASM vs 8,429 EL data points).
  • How it is used in this paper: defines both the dataset annotation layers and the neural baseline's modular architecture.

5Cs Analysis5Cs 분석

Category: benchmark/dataset + accompanying neural baseline Context: builds on SciBERT (Beltagy 2019), DPR (Karpukhin 2020), BM25/BM25F, AxCell/SegmentedTables (Kardas 2020) for bootstrapping, and TURL (Deng 2020) as the general-table EL comparison; PapersWithCode as target KB. Correctness: IAA reported via Cohen's Kappa with post-hoc independent annotation by a PhD author (CTC 100%, ASM 85.5%, EL 60.6%); the modest EL kappa is honestly disclosed as reflecting task difficulty. Contributions: (1) first table-EL dataset in the scientific domain with realistic outKB mentions; (2) modular SciBERT baseline for each sub-task and end-to-end; (3) error analysis motivating future work. Clarity: method described stage-by-stage with losses; reproducibility plausible but exact hyperparameters/tables not on pp.1–5.

Contribution vs. Claim기여 대 주장

— Abstract claims the method "significantly outperforms" a SOTA generic table-EL method and that baselines fall below human performance; confirming the exact margins and whether the outKB-detection claim holds across sub-tasks requires the experiments section (p.6+), not read in skim mode.

Implementation Feasibility구현 실현가능성
  • Needs: SciBERT, DPR bi-encoder, BM25/BM25F retrieval, T5-small for reference parsing, PapersWithCode 2022/07 dump, Semantic Scholar API; moderate GPU for fine-tuning.
  • Code: available — dataset at https://github.com/allenai/S2abEL/ ; code repo under allenai/S2abEL.
  • Modifications: retarget from PwC 4-type ontology to our dependency-KB schema; extend from table cells to free-text dependency mentions; adapt outKB head to reuse-vs-new-node classifier.
  • Unknowns: missing link — exact model result numbers and the outKB-detection performance breakdown (need deep read of p.6+ tables).

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = a scientific paper's ML results table + surrounding document context + PapersWithCode KB (304,611 papers / 6,550 datasets / 1,942 methods); outputs = per-cell entity link to PwC or "outKB"; conditions = human-annotated ML-domain tables only, closed set of 4 entity types (method/dataset/metric/dataset&metric), highly incomplete KB. Our setting: inputs = dependency mentions extracted from scientific papers (methods/datasets/components a work builds on); outputs = link each mention to an existing KB node OR create a new node (reuse-vs-new decision) for dependency-graph / KB construction; conditions = free text + tables, growing KB, cross-domain. Differences:

  • outKB detection ≈ our "new-node vs reuse-node" decision: resolved / directly reusable as the formal framing of the reuse-vs-new decision.
  • Source is tables only, not full-text dependency statements: missing link — our extraction is text-centric; ASM/CTC assume tabular structure.
  • Target KB is PapersWithCode with 4 fixed types: reference only — our KB schema (dependency relations) differs; entity typology must be redefined.
  • Attributed-source matching leverages reference sections for disambiguation: resolved / borrow ASM as a disambiguation signal for citation-grounded dependency linking.
  • ML-domain only, English: reference only — generalization to other domains unverified.
