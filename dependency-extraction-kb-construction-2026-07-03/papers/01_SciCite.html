Structural Scaffolds for Citation Intent Classification in Scientific Publications

Cohan et al. · NAACL 2019 · 2019 · 307 citations인용 307회

arXiv:1904.01608 · PDF · SP: SP1

P0Track A

Problem
해결하려는 문제

Citation-intent classification (labeling why a paper cites another: background, method use, result comparison) is critical for automated analysis of scientific literature, but prior models were feature-based — relying on hand-engineered linguistic patterns / cue phrases and external resources — and ignored the structural signal latent in how scientific papers are organized. Existing datasets were also small and single-domain, limiting robustness.

인용의 의도(배경/방법 사용/결과 비교)를 분류하는 것은 과학 문헌의 자동 분석에 핵심이나, 기존 모델은 수작업 언어 자질에 의존하며 논문의 구조 신호를 활용하지 못하였다.

Solution / Contribution
해결책 / 기여

Neural multitask "scaffold" model.

  • Encode citation context: x_i = [GloVe; ELMo] → single-layer BiLSTM → dot-product attention → context vector z.
  • Main head predicts citation intent; two auxiliary scaffold heads predict (1) citation worthiness and (2) section title.
  • Joint loss: $L = \sum_{(x,y)\in D_1} L_1(x,y) + \sum_{i=2}^{n} \lambda_i \sum_{(x,y)\in D_i} L_i(x,y)$, with $n=3$ and $\lambda_i$ tuned on validation (grid 0.0–0.3). Scaffold labels are auto-harvested from paper structure, so no extra manual annotation.

공유 BiLSTM 인코더 위에 인용 위치(섹션 제목)와 인용 필요성을 예측하는 auxiliary scaffold 과제를 결합한 다중과제 신경망을 제안하며, ACL-ARC macro-F1 67.9와 대규모 다중 도메인 SciCite 데이터셋을 확립한다.

Key Results
주요 결과
  • ACL-ARC: macro F1 67.9 (best model: BiLSTM-Attn w/ ELMo + both scaffolds) vs. Jurgens et al. 2018 prior SOTA 54.6 (+13.3 absolute).
  • Scaffold ablation on ACL-ARC: BiLSTM-Attn baseline 51.8 → +section-title 56.9 → +citation-worthiness 56.3 → +both 63.1; scaffolds add +13.6 on top of the ELMo baseline.
  • SciCite: macro F1 84.0 (best model) vs. best baseline BiLSTM-Attn+ELMo 82.6 (+1.4; smaller gain because dataset is ~5x larger).
  • Results obtained without hand-curated features or external linguistic resources.
Concepts Introduced도입된 개념

Structural scaffold (auxiliary task)

  • What it computes: input = citation-context token sequence encoded by a shared BiLSTM+attention into vector z; operation = each scaffold is an auxiliary classification head (MLP + softmax) trained jointly with the main head; output = auxiliary predictions (section title / citation-worthiness) whose gradients shape the shared lower layers.
  • Why it was created: prior feature-based citation-intent models relied on hand-engineered linguistic/cue-phrase features and ignored the structural signal of scientific papers; scaffolds inject that structure without manual annotation.
  • Concrete example: predicting the section title ("introduction", "method", "experiments"...) in which a citation occurs — a method citation is more likely to appear in the methods section.
  • How it is used in this paper: two scaffolds (citation worthiness + section title) are added as auxiliary heads; scaffold training data is auto-generated (~47K–91K instances) and improves the main citation-intent classifier.

Citation worthiness (scaffold task)

  • What it computes: input = a sentence; operation = binary classifier over the shared encoder; output = whether the sentence needs a citation.
  • Why it was created: citation-bearing language is distinctive; modeling it improves the encoder's representation of citation contexts. Labels occur naturally (sentences with vs. without citation markers), so no manual annotation is needed.
  • Concrete example: "[12]" or "Lee et al (2010)" markers used to auto-label positive sentences; markers stripped so the task is not trivial.
  • How it is used in this paper: one of the two scaffolds; alone it lifts ACL-ARC macro F1 from 51.8 → 56.3.

SciCite dataset

  • What it computes: a citation-intent benchmark with a simplified 3-way taxonomy — Background (0.58), Method (0.29), Result comparison (0.13).
  • Why it was created: existing datasets (ACL-ARC) are small (1,941 instances) and single-domain (computational linguistics); SciCite is >5x larger and spans CS + Medicine.
  • Concrete example: 11,020 citation instances from 1,941 papers.
  • How it is used in this paper: the second evaluation benchmark and a lasting contribution — a primary evaluation standard for the task.

Contribution vs. Claim기여 대 주장

— abstract/intro claim new SOTA and a broadly useful multi-domain dataset. Ablation numbers on p.5-6 support the scaffold contribution on both datasets; whether cross-domain generality claims hold per-category needs the full experiments/analysis section.

Implementation Feasibility구현 실현가능성
  • Needs: GloVe (6B) + ELMo vectors, AllenNLP, single-layer BiLSTM (hidden 50/dir), MLP scaffold heads; modest compute. Auto-harvested scaffold data (section titles + citation markers from a large corpus, e.g. Semantic Scholar).
  • Code: available at https://github.com/allenai/scicite (code + SciCite data).
  • Modifications: for KB construction, would need to (a) retain/extend Uses/Extends-style labels, (b) add entity/artifact linking to resolve the cited target, (c) emit relation triples rather than flat labels.
  • Unknowns: missing link — cross-domain robustness beyond CS/Medicine; how scaffold auto-labels transfer to other domains.
Follow-up Papers관련 후속 논문
  • SciBERT (Beltagy et al., 2019) [04] — pretrained scientific LM that supersedes GloVe+ELMo encoders and reports on SciCite.
  • Jurgens et al. 2018 [03] — source of ACL-ARC benchmark and the 6-way frame taxonomy (Uses/Extends) more directly tied to dependency lineage.
  • Teufel et al. 2006 [02] — foundational citation-function annotation scheme underlying the task vocabulary.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = citation context text spans from citing papers; outputs = 1-of-3 (SciCite) or 1-of-6 (ACL-ARC) citation-intent labels; conditions = supervised classification, GloVe+ELMo embeddings, multitask with auto-harvested scaffold data, domains CS/Medicine/CL. Our setting: dependency extraction for knowledge-base construction — extracting typed dependency relations between scientific artifacts (e.g. which paper uses/extends which method/dataset) to populate a KB. Differences:

  • Task granularity: intent of a citation vs. typed dependency relation between artifacts — reference only (intent labels Method/Uses/Extends are a proxy dependency signal, not the full relation extraction).
  • Taxonomy: SciCite's 3-way (Background/Method/Result) collapses derivation-lineage frames (Uses/Extends kept separate in ACL-ARC, which is more directly dependency-relevant) — missing link: SciCite alone under-specifies dependency edges.
  • Output structure: flat classification vs. graph/KB triples — missing link: no entity linking or relation-target resolution here.
  • Signal source: citation context sentence — resolved: reusable as the primary input signal for dependency detection.
