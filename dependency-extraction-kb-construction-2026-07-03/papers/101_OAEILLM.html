OAEI-LLM: A Benchmark Dataset for Understanding Large Language Model Hallucinations in Ontology Matching

Wang, Qiang, Taylor, et al. · EACL 2026 (Main) · 2024

arXiv:2409.14038 · PDF · SP: SP7

P0Track B

Problem
해결하려는 문제

LLM-based ontology matchers frequently miss true mappings and invent false ones (hallucinations), yet the standard OAEI datasets only measure aggregate precision/recall/F1 and give no visibility into which type of error occurred. There is also no dataset providing baseline references to fine-tune LLMs against these hallucinations.

집계 P/R/F1은 명백한 오류와 그럴듯하나 다른 매핑을 구분하지 못한다.

Solution / Contribution
해결책 / 기여

Empirical/resource contribution. Construct OAEI-LLM by: (1) running an LLM-based matcher on OAEI source/target ontologies to produce $A_{llm}$; (2) comparing against the human OAEI reference $R_{oaei}$; (3) classifying every discrepancy via an LLM-based evaluator into the missing/incorrect taxonomy (with sub-types false/disputed/align-up/align-down); (4) recording results in an extended EDOAL schema with a per-LLM <hallucination> tag. Non-hallucination condition: a mapping $(e_1,e_2)\in R_{oaei}$ is not a hallucination iff $\exists (e_1',e_2')\in A_{llm}$ with $e_1=e_1'$ and $e_2=e_2'$. Mappings constrained to one-to-one equivalence only.

missing/incorrect(false/disputed/align-up/align-down) 환각 오류 taxonomy와 OAEI-LLM 벤치마크를 제시하며, disputed mapping을 신규 노드 생성 신호로 제공한다.

Key Results
주요 결과
  • No empirical numbers reported — this is a dataset/position paper describing methodology and schema only.
  • Concrete artifact: extended EDOAL schema example showing claude-3-sonnet (missing) and llama-3-8b (incorrect/disputed) on the Chairman/Chair mapping.
  • Planned variants: OAEI-LLM-T (TBox), OAEI-LLM-A (ABox / KG-like instance matching), OAEI-LLM-I (interactive).
Concepts Introduced도입된 개념

OAEI-LLM benchmark schema (extended EDOAL/SSSOM)

  • What it computes: input = source ontology $O_s$, target ontology $O_t$, OAEI human reference alignment $R_{oaei}$, and an LLM-generated alignment $A_{llm}$ → operation = a matching-assessment step compares $R_{oaei}$ against $A_{llm}$, labels each mapping with a hallucination category, and records it in an extended <Cell> tag → output = a per-mapping, per-LLM record of hallucination type stored in a new <hallucination> sub-tag.
  • Why it was created: traditional OAEI datasets only provide matched entities and, when run through an LLM matcher, only yield precision/recall/F1 — they cannot say what kind of error the LLM made or provide fine-tuning targets for hallucination mitigation.
  • Concrete example: for the valid match cmt#Chairman=conference#Chair, claude-3-sonnet records <category>missing</category><source>llm</source> (it dropped the true mapping); llama-3-8b records <category>incorrect</category> mapping instead to cmt#ConferenceChair with <type>disputed mapping</type>.
  • How it is used in this paper: it is the core deliverable — the dataset format and the error taxonomy that populates it.

LLM hallucination error taxonomy for OM

  • What it computes: given a reference pair $(e_1,e_2)\in R_{oaei}$ and LLM pairs $(e_1',e_2')\in A_{llm}$, it assigns one of: Missing from LLM (reference pair has no LLM counterpart), Missing from OAEI (LLM pair has no reference counterpart), Incorrect (one entity matches, the other does not). Incorrect is further split into False-mapping (mapped to an irrelevant entity), Disputed-mapping (mapped to a relevant entity that disagrees with OAEI — OAEI may be wrong), Align-up (mapped to intended superclass), Align-down (mapped to intended subclass).
  • Why it was created: to distinguish outright errors from cases where the LLM found a plausible-but-different mapping — directly relevant to deciding whether to accept an equivalence or create a new node.
  • Concrete example: conference#Chair → cmt#ConferenceChair is a "disputed mapping" (relevant but off-reference), not a false mapping.
  • How it is used in this paper: defines the labels the matching-assessment (an LLM-based evaluator) applies to build the benchmark.

5Cs Analysis5Cs 분석

Category: benchmark / dataset (position + resource paper, 5 pages) Context: builds directly on OAEI campaigns (conference track, EDOAL alignment format, SSSOM) and on LLM-OM systems Agent-OM, LLMs4OM, OLaLa, plus the hallucination survey "Siren's Song in the AI Ocean". Correctness: taxonomy is well-defined formally, but the "disputed/false/align-up/align-down" labels are assigned by a single unspecified LLM evaluator — authors admit "we cannot recommend a particular LLM" and that assessment "may vary depending on the LLM used." No quantitative results, no inter-rater agreement, no error bars. Contributions: (1) an error taxonomy separating missing vs. incorrect vs. disputed LLM mappings; (2) an extended EDOAL/SSSOM schema recording per-LLM hallucination provenance; (3) two use cases — richer benchmarking and fine-tuning data generation. Clarity: schema and taxonomy are reproducible; the matching-assessment (evaluator LLM, prompts) is under-specified, so exact dataset labels are not reproducible.

Contribution vs. Claim기여 대 주장
  • Stated claims: the dataset lets one "quantify LLM errors by counting each type," supports fine-tuning to mitigate hallucinations, and could enhance an OM leaderboard.
  • Demonstrated results: only a schema design and a single worked example are shown; no counts, no fine-tuning experiment, no leaderboard.
  • Gap: the quantification and fine-tuning utility are asserted but not empirically validated in this paper (no dataset statistics, no downstream results). The taxonomy's reliability depends on an unvalidated LLM-based evaluator.
Implementation Feasibility구현 실현가능성
  • Needs: OAEI conference-track ontologies, an LLM-based matcher (e.g., Agent-OM), an LLM-based evaluator for assessment, EDOAL/SSSOM tooling. Modest compute.
  • Code: no direct code/dataset link in the paper body; OAEI data is publicly available under the OAEI use policy. Dataset itself described as forthcoming with three variants.
  • Modifications: for SP7 we would reuse only the error taxonomy (especially the disputed/align-up/align-down distinction) as a confidence-signal schema; the OM-specific pipeline is not needed.
  • Unknowns: missing link — which LLM evaluator, exact prompts, and any reliability/agreement statistics are unspecified; no acceptance-threshold logic provided.
Follow-up Papers관련 후속 논문
  • Agent-OM: Leveraging LLM Agents for Ontology Matching (Qiang, Wang & Taylor, 2023) — the matcher used to generate LLM alignments; source of the "green" text in the schema example.
  • LLMs4OM: Matching Ontologies with Large Language Models (Giglou et al., 2024) — comparable LLM-OM system whose hallucinations this benchmark would characterize.
  • Siren's Song in the AI Ocean: A Survey on Hallucination in LLMs (Zhang et al., 2023) — foundational hallucination taxonomy the paper cites for its error-type framing.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = pairs of small conference-domain OWL ontologies + human reference alignment; outputs = per-mapping hallucination labels; conditions = one-to-one equivalence mappings only, TBox/schema level, LLM-based matcher + LLM-based evaluator. Our setting (SP7, dependency extraction / KB construction): inputs = extracted entities/relations that must be merged into a knowledge base; outputs = decisions on whether a candidate is equivalent to an existing node (accept mapping) or a new node (create); conditions = need confidence/acceptance criteria to avoid spurious equivalence merges. Differences:

  • Task framing (ontology matching vs. KB node merge/creation): reference only — the accept-vs-create-new-node decision is analogous to accept-equivalence-vs-flag-disputed.
  • The "disputed-mapping" and "align-up/align-down" categories: resolved/directly transferable — they give a vocabulary for "relevant but not equivalent," which is exactly the signal for creating a new node rather than merging. This is the key borrowable idea for SP7's confidence criteria.
  • Equivalence-only, one-to-one constraint vs. our need for subsumption and one-to-many: missing link — the paper explicitly defers subsumption/complex matching to future work; our KB setting needs those.
  • No confidence threshold mechanism given (measure=1.0 hard-coded in example): missing link — paper records hallucination post-hoc, does not propose an acceptance threshold.
