TaxoExpan: Self-supervised Taxonomy Expansion with Position-Enhanced Graph Neural Network

Shen, Shen, Xiong, Wang, Wang, Han · WWW 2020 · 2020

arXiv:2001.09522 · PDF · SP: SP7

P0Track A

Problem
해결하려는 문제

Existing automatic taxonomy methods construct the entire hierarchy from scratch; adding emerging concepts requires re-running the whole pipeline, which (1) discards the expert-curated top-level design, (2) may produce a new taxonomy inconsistent with the deployed one, and (3) cannot exploit signal from the existing taxonomy. Prior expansion-specific methods depend on scarce manually-labeled insertion positions or on WordNet-specific synset structure, limiting model power and generality.

개념 하나를 추가하려 전체 taxonomy를 재구성하면 전문가 큐레이션 상위 구조가 파괴된다.

Solution / Contribution
해결책 / 기여

Self-supervised framework (TaxoExpan). Reduce taxonomy expansion to $|C|$ independent single-parent search problems: $a_i^* = \arg\max_{a_i \in N^0} \log P(n_i \mid a_i, \theta)$. Auto-generate ⟨query, anchor⟩ training pairs from the existing DAG (each node's true parent = positive; sampled non-parents/non-descendants = negatives). A matching model scores query–anchor compatibility, where the anchor is encoded by a position-enhanced GNN over its ego network (siblings + grandparents of the query) and trained with a noise-robust InfoNCE objective that groups same-query pairs and selects the positive.

기존 DAG의 부모-자식 edge에서 자기지도한 position-enhanced GNN으로 새 개념을 leaf로 배치하는 TaxoExpan을 제안하여 라벨 없는 증분 확장을 가능케 한다.

Key Results
주요 결과
  • On MAG-CS, TaxoExpan achieves the best MRR 0.6929 (±0.003) vs. second-best ~0.6653 across compared methods.
  • All four self-supervised methods outperform rule-based baselines (Closest-Parent, Closest-Neighbor); TaxoExpan beats the second-best "by a large margin."
  • Claims state-of-the-art on the WordNet (SemEval-2016 Task 14) taxonomy expansion challenge.
  • Evaluated on three large-scale datasets (MAG-Full ≈660K concepts / 700K relations, MAG-CS, WordNet).
Concepts Introduced도입된 개념

Taxonomy expansion task (vs. taxonomy construction)

  • What it computes: input = existing taxonomy $T^0=(N^0,E^0)$ (a DAG) + a set of new concepts $C$ → operation = find, for each new concept, a single anchor (parent) node in $N^0$ → output = expanded taxonomy $T=(N^0\cup C,\ E^0\cup R)$ where existing relations $E^0$ are left unchanged.
  • Why it was created: prior taxonomy-construction methods build the whole hierarchy from scratch, so adding new concepts forces a full rebuild — destroying the expert-designed top level and destabilizing downstream apps.
  • Concrete example: given a computer-science field-of-study taxonomy, place new concept "UDA" under "Semi-supervised Learning" and "TPU" under "Integrated Circuit" without touching existing edges.
  • How it is used in this paper: the entire problem framing; reduced to $|C|$ independent single-parent search problems $a_i^*=\arg\max_{a_i\in N^0}\log P(n_i|a_i,\theta)$.

Self-supervision from existing taxonomy

  • What it computes: input = existing taxonomy → operation = treat each node as a query and one of its true parents as a positive anchor, sample non-parent/non-descendant nodes as negative anchors → output = labeled ⟨query, anchor⟩ pairs with no human annotation.
  • Why it was created: prior expansion methods need manually labeled ⟨concept, true-position⟩ data, which is small and limits model power.
  • Concrete example: ⟨"GPU", "Integrated Circuit"⟩ = positive; ⟨"GPU", "Label Propagation"⟩ = negative.
  • How it is used in this paper: generates all training data for the matching model.

Position-enhanced GNN (ego-network encoder)

  • What it computes: input = anchor's ego network (parent = grandparents + siblings of query) → operation = GCN/GAT propagation + position embeddings injected to distinguish a node's relative role (grandparent vs. sibling) + graph readout → output = anchor representation vector.
  • Why it was created: representing an anchor by its own embedding alone loses surrounding structure; plain GNNs cannot tell apart nodes at different relative positions to the query.
  • Concrete example: query "high dependency unit" looks dissimilar to anchor "hospital room" by name, but its sibling children "intensive care unit"/"low dependency unit" signal a correct placement.
  • How it is used in this paper: core anchor encoder; 2-layer position-enhanced GAT (50-dim position embeddings).

Noise-robust InfoNCE training objective

  • What it computes: input = all ⟨query, anchor⟩ pairs sharing one query grouped into a single instance → operation = InfoNCE-style objective that selects the positive anchor among the group's negatives (rather than independent per-pair binary classification) → output = matching model robust to label noise in self-supervision data.
  • Why it was created: self-supervision labels are noisy (a sampled "negative" may actually be a valid position); per-pair binary loss is sensitive to that noise.

Category: new method (self-supervised framework + GNN technique) Context: builds on GNNs (GCN [Kipf], GAT [Velickovic]), InfoNCE/contrastive learning [van den Oord], and prior taxonomy-expansion work (SemEval-2016 Task 14 [Jurgens & Pilehvar], hyperbolic-embedding attachment [Aly et al.], feature-ranking [Vedula et al.]). Related in this survey to SP7 new-node placement vs. existing-node reuse. Correctness: reports mean ± std over runs (e.g., MRR 0.6929 ±0.003); three real-world datasets. Full validity of assumptions to be checked in deep read. Contributions: (1) self-supervised expansion framework needing no labeled data; (2) position-enhanced GNN injecting hierarchical relative-position info; (3) noise-robust InfoNCE training objective. Clarity: architecture and hyperparameters (2-layer GAT, 250-dim word2vec embeddings, 50-dim position embeddings, Adam lr 0.001) are given; full reproducibility to confirm in deep read.

Contribution vs. Claim기여 대 주장

— abstract/intro claim effectiveness + efficiency across three domains and SOTA on WordNet; the experiments section (pages 6+) must be read fully to confirm whether the position-embedding and noise-robust-loss ablations independently justify each claimed contribution, and whether "generality" holds beyond the two MAG variants + WordNet.

Implementation Feasibility구현 실현가능성
  • Needs: an existing taxonomy/DAG, concept embeddings (word2vec 250-dim used), PyTorch + DGL for GNN, Adam optimizer; moderate compute.
  • Code: — authors are UIUC/MSR (Jiaming Shen, Jiawei Han); public implementation expected (mnqu/TaxoExpan on GitHub — verify in deep read).
  • Modifications: add an explicit new-vs-existing reuse decision layer; support multi-parent / batch insertion; swap word2vec for contextual LM embeddings for our concepts.
  • Unknowns: missing link — exact InfoNCE grouping/negative-sampling details and WordNet SOTA number not yet read (pages 6+).
Follow-up Papers관련 후속 논문
  • STEAM / TMN (Taxonomy expansion via matching, KDD 2021) — extends the query-anchor matching paradigm with triplet matching.
  • HEF, TaxoEnrich (WWW 2022) — later taxonomy-enrichment methods that build directly on TaxoExpan's self-supervision framing.
  • Musubu / GenTaxo — generation-based node insertion, contrasts with TaxoExpan's discriminative matching.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = existing curated taxonomy (DAG) + set of new concepts with initial 250-dim word2vec embeddings from surface names; outputs = single anchor parent per new concept; conditions = existing edges frozen, one concept inserted at a time (inter-concept interactions ignored), leaf-masking evaluation. Our setting: inserting a newly extracted concept node into an existing dependency/knowledge-base hierarchy — deciding new-node placement vs. reuse of an existing node (SP7 / O2). Differences:

  • New-node placement mechanism: resolved / TaxoExpan directly models "find best parent for a new concept," matching O2's placement half.
  • New-node vs. existing-node reuse decision: — TaxoExpan always inserts (assumes the concept is genuinely new); it does not natively decide "this concept already exists, reuse it." Likely missing link for the reuse branch.
  • Edges frozen / no modification of existing relations: reference only — our setting may need to revise existing dependencies.
  • Single-parent, one-at-a-time insertion: — ignores interactions among simultaneously added concepts; may be a missing link for batch dependency extraction.
  • Input = pre-trained concept embeddings from surface names: resolved / standard, replaceable with LM embeddings.
