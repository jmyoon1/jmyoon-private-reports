Enhancing Long-Term Welfare in Recommender Systems: An Information Revelation Approach

Zhao et al. · arXiv · 2025

arXiv:2510.10511 · PDF · SP: SP9

P4Track B

한 줄 요약

정보 공개(information revelation)로 장기 후생을 높이는 접근(LoRe, 2025). 휴리스틱 exposure 조향에 대한 비판을 담고 있으며, LLM agent는 행동 시뮬레이터로만 사용한다.

Problem해결하려는 문제

Existing approaches to steering creator behavior in recommender systems rely on heuristic re-ranking that adjusts item exposure (e.g., boosting exposure of small creators to retain them). When embedded in the recommendation pipeline, this exposure lever conflicts with the short-term objective of recommendation accuracy — the same ranking slots must serve both accuracy and steering — degrading performance and yielding suboptimal long-term welfare. A second limitation: reward-based creator-influencing methods (adjusting reward weights, promising clicks) directly alter creators' realized rewards, harming them and introducing unfairness.

Solution / Contribution해결책 / 기여

LoRe steers creators through a channel that is independent of the ranking algorithm: information revelation. [SKIM - verify method details]

  1. Map the cheap-talk model onto RS: platform = sender, creators = receivers, state = item-genre distribution (unknown to individual creators), messages = genre suggestions.
  2. Because creators are boundedly rational (their response to messages cannot be computed analytically), formulate the platform-creator interaction as an MDP: state = per-creator genre-creation vector of the previous round, action = per-creator suggestion vector, reward = total user clicks per round.
  3. Optimize $J(\pi_\theta) = \mathbb{E}_{\tau \sim \pi_\theta}[\sum_t \gamma^t \mathcal{R}(\boldsymbol{s}_t, \boldsymbol{a}_t, \boldsymbol{s}_{t+1})]$ with an actor-critic algorithm (clipped surrogate loss + TD critic loss + generalized advantage estimation), alternating inference (data collection in the LLM-agent simulator) and training phases.

LoRe is a standalone module — it makes no assumption about the creator behavior model and can be composed with any ranking/re-ranking algorithm.

Concepts Introduced도입된 개념

Cheap talk (applied to RS)

  • What it computes: sender's information-revelation strategy $\pi: \mathcal{W} \times M \mapsto [0,1]$ — given true state $w$ (known only to sender), outputs a probability of sending each message $m$; receiver updates beliefs on the message and picks an action.
  • Why it was created: classical economics mechanism for steering a less-informed party's behavior via information asymmetry, without transferring payments or altering the receiver's rewards.
  • Concrete example: platform sends creator $c_i$ the message "I recommend that you create an item with genre $g_j$"; the suggestion set is $\mathcal{A}_p = \{a_i\}_{i=0}^{|\mathcal{G}|}$ with $a_0$ = no suggestion.
  • How it is used in this paper: the mapping is platform=sender, creators=receivers, state=distribution of item genres, messages=genre suggestions (Table 1 of the paper); the platform learns the suggestion strategy instead of computing it analytically.

Suggestion-strategy MDP

  • What it computes: an MDP $(\mathcal{S}, \mathcal{A}, \mathcal{T}, \mathcal{R}, \gamma)$ where state $\boldsymbol{s}_t$ = vector of genres created by each creator in round $t-1$, action $\boldsymbol{a}_t$ = per-creator suggestion vector, reward $\mathcal{R}$ = total user clicks in the round; the policy $\pi_\theta(\boldsymbol{a}_t|\boldsymbol{s}_t)$ is trained with an actor-critic RL algorithm (clipped policy loss + GAE, i.e. PPO-style).
  • Why it was created: classical cheap-talk theory assumes fully rational receivers whose responses can be computed in closed form; real creators are boundedly rational, so the optimal revelation strategy must be learned from interaction instead.
  • Concrete example: $\boldsymbol{s}_t \in \mathbb{R}^{|\mathcal{C}| \times (|\mathcal{G}|+1)}$, $\boldsymbol{a} \in \mathbb{R}^{|\mathcal{C}|}$; training loop alternates $N$ inference rounds (fill replay buffer with $(\boldsymbol{s}_t, \boldsymbol{a}_t, \boldsymbol{s}_{t+1}, \mathcal{R})$) and $M$ training epochs.
  • How it is used in this paper: this IS the method — LoRe = the MDP formulation + the RL optimizer for the platform's suggestion strategy.
