Construction of the Literature Graph in Semantic Scholar

Ammar et al. · NAACL 2018 (Industry) · 2018 · 440 citations인용 440회

arXiv:1805.02262 · PDF · SP: SP2

P0Track C

Problem
해결하려는 문제

Scientific search engines store papers as opaque documents, so researchers cannot answer structured questions ("percentage of female subjects in depression trials", "which co-authors worked on coreference"). Extracting structured data from scientific PDFs is hard because (a) papers often arrive with missing/no metadata, and (b) standard NLP formulations (NER on CoNLL/ACE) assume large in-domain labeled data and fixed entity-type sets that match at train/test — unrealistic across many scientific domains and at production scale.

대규모 과학 문헌 그래프를 실서비스 규모로 구축하는 방법이 필요하였다.

Solution / Contribution
해결책 / 기여

Empirical/system contribution: reduce literature-graph construction to a pipeline of familiar NLP tasks, each addressing an unrealistic standard-benchmark assumption, deployed at Semantic Scholar scale.

  1. Structure: directed property graph with 4 node types (papers, authors, entities, entity mentions) and 5 edge types (citations, authorship, entity-linking, mention–mention relations, entity–entity relations). Property-graph (not RDF) chosen for richer internal node/edge structure.
  2. Metadata (§3): ScienceParse BiLSTM+CRF over PDF-layout token features to recover title/authors/references from raw PDFs; trained on 1.4M PubMed Central PDFs with heuristic distant labeling (76% of docs successfully labeled).
  3. Entity extraction + linking (§4): BILOU BiLSTM with fixed LM embeddings for mention spans (no type prediction), then a neural pair-scoring linker grounding mentions in UMLS (biomedical) and a CS-filtered DBpedia. Three approaches (statistical, hybrid rule-based, off-the-shelf TagMe/MetaMap Lite) are pooled in production because each catches entities the others miss.
  4. Open research problems (§5): author disambiguation (supervised pairwise merge → incremental clustering), ontology matching, limited-KB-coverage via distant supervision for definitions/relations, figure/table extraction, citation understanding/prediction.

Semantic Scholar의 문헌 그래프를 개체 추출·링킹·관계 구성으로 대규모 구축한 배포 파이프라인을 기술하여 과학 KB 구성·유지의 참조 아키텍처를 제공한다.

Key Results
주요 결과
  • ScienceParse metadata (54K PubMed Central held-out, exact match): title F1 85.5, authors F1 92.1, bibliography titles 89.3, authors 97.0, venues 90.7, years 98.0 (Table 1).
  • Entity extraction (SemEval-2017 Task 10 dev): 49.9 F1 without LM → 54.1 with LM (+4.2) → 55.2 with 15-model ensemble (+1.1) (Table 3).
  • Entity linking (Bag-of-Concepts F1): CS 84.2 baseline → 84.6 neural; Bio 54.2 baseline → 85.8 neural (+31.6) (Table 4).
  • Extract/link approach tradeoff (500 papers, Table 2): statistical highest precision/lowest yield (CS 98.4 prec / 712 yield); hybrid highest yield lowest precision (CS 91.5 / 1990); pooling all approaches gives higher yield than any single one while keeping precision reasonable.
  • Deployed graph scale: >280M nodes (37M papers, 12M authors, 0.4M entities, 237M mentions).
Concepts Introduced도입된 개념

Literature graph (heterogeneous property graph)

  • What it computes: input = corpus of scientific papers (PDFs + metadata) → operation = extract nodes (papers, authors, entities, entity mentions) and directed edges (citations, authorship, entity-linking, mention–mention relations, entity–entity relations) → output = a directed property graph (>280M nodes; 37M papers, 12M authors, 0.4M entities, 237M mentions).
  • Why it was created: existing scientific search engines cannot answer structured queries (e.g., "which co-authors published on coreference resolution?"); a symbolic graph enables algorithmic discovery over the literature.
  • Concrete example: computing an author's Erdős number = shortest undirected path over edges labeled "authored" between that author and Paul Erdős.
  • How it is used in this paper: the organizing framework; the whole paper reduces its construction to NLP subtasks (metadata extraction, entity extraction, entity linking, relation extraction).

ScienceParse metadata extraction (BiLSTM+CRF over PDF tokens)

  • What it computes: input = PDF pages tokenized via PDFBox, each token carrying visual/positional features (font size, space width normalized -0.5..0.5, XY position, case features, page-number bucket) + GloVe word embedding → operation = 2-layer BiLSTM with label-bigram (CRF-style) layer, Viterbi decoding → output = paper title, authors, and structured references (title/authors/venue/year).
  • Why it was created: many papers arrive with incomplete or no metadata (web-crawled); metadata must be recovered from raw PDF layout.
  • Concrete example: title F1 = 85.5, authors F1 = 92.1, bibliography years F1 = 98.0 (Table 1). Failure mode: "Clinical review: Foo Bar" → extracts only "Foo Bar".
  • How it is used in this paper: §3, populates paper nodes, author nodes, authorship + citation edges.

Entity extraction + linking pipeline (statistical / hybrid / off-the-shelf pooled)

  • What it computes: input = paper title+abstract tokens → operation = BILOU sequence labeling (CNN char embeddings + GloVe + fixed bidirectional RNN-LM embeddings, 2-layer BiLSTM) to find mention spans (no entity typing), then a neural linker scores mention–candidate pairs against a curated KB → output = entity mentions grounded to KB entity IDs.
  • Why it was created: standard NER assumes fixed entity types and matching train/test type sets — unrealistic across scientific domains; grounding in a curated KB avoids concept-typing and boosts user trust.
  • Concrete example: entity extraction dev F1 rises 49.9 (no LM) → 54.1 (with LM) → 55.2 (ensemble of 15) on SemEval-2017 Task 10. Linking "ILP" → "Inductive_Logic_Programming" over distractors.
  • How it is used in this paper: §4, populates the 237M mention nodes and entity-linking edges.

5Cs Analysis5Cs 분석

Category: system / engineering (deployed production pipeline with empirical evaluations per component) Context: builds on neural sequence tagging (Lample 2016; Collobert 2011), semi-supervised LM embeddings (Peters 2017; Ammar 2017 — precursor to ELMo), distant supervision (Mintz 2009), entity linking (Ling 2015; Bhagavatula 2015). Directly relevant SP2 neighbors: relation-extraction and KB-construction papers in this survey. Correctness: per-task evaluations with precision/recall/F1 and human CrowdFlower annotation (3+ annotator agreement). Reports "yield" instead of recall — honestly flags that true recall is unmeasurable at scale. No CIs/error bars. Relation-extraction models omitted "due to space constraints", so that edge type is unvalidated here. Contributions: (1) a reference architecture for scientific KB construction as pooled NLP subtasks; (2) ScienceParse layout-aware metadata extraction; (3) KB-grounded entity extraction/linking that drops the fixed-entity-type assumption. Clarity: architecture and features described concretely (reproducible for metadata + entity models); linking model equations given. But relation extraction and exact pooling/merge thresholds are underspecified.

Contribution vs. Claim기여 대 주장

  • Stated claims: a scalable deployed system organizing 280M+ nodes; reduces graph construction to familiar NLP tasks while fixing their impractical benchmark assumptions.
  • Demonstrated: strong component-level metrics for metadata extraction and entity extraction/linking on held-out academic benchmarks; honest per-approach precision/yield tradeoffs.
  • Gap: relation extraction models (mention–mention, entity–entity edges) are explicitly not described or evaluated here, yet those edges are part of the graph structure — the "relation construction" implied by the abstract is unvalidated in this paper. Author disambiguation, ontology matching, and KB-coverage models are described as active work, not evaluated. "Yield" substitutes for recall, so end-to-end coverage of the graph is not quantified.
Implementation Feasibility구현 실현가능성
  • Needs: PDF token features (Apache PDFBox), GloVe embeddings, BiLSTM+CRF sequence taggers, RNN-LM embeddings, curated KBs (UMLS license, DBpedia), CrowdFlower-style annotation for eval, large training corpora (1.4M PubMed Central PDFs).
  • Code: available — ScienceParse (github.com/allenai/science-parse), deepfigures-open, citeomatic; corpus metadata at labs.semanticscholar.org/corpus.
  • Modifications: for SP2 dependency extraction we would need the (absent) relation-extraction component; extend from title+abstract to full text; adapt KB grounding to the target domain's ontology.
  • Unknowns: missing link: relation-extraction model architecture and its evaluation; missing link: exact pooling and author-merge thresholds.
Follow-up Papers관련 후속 논문
  • Peters et al. 2017, "Semi-supervised sequence tagging with bidirectional LMs" (ACL) — the LM-embedding technique that gives +4.2 F1 here; ELMo precursor.
  • Bhagavatula et al. 2018, "Content-based citation recommendation" (NAACL) — cite-prediction over the graph's citation edges.
  • Siegel et al. 2018, "Extracting scientific figures with distantly supervised neural networks" (JCDL) — figure/table extraction subtask.
  • Valenzuela et al. 2015, "Identifying meaningful citations" (AAAI Workshop) — important-vs-incidental citation classification, enriching citation edges.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = scientific-paper PDFs + partial publisher metadata; outputs = a heterogeneous property graph; conditions = production scale (280M nodes), CS + biomedical domains, curated KBs (UMLS, DBpedia) as grounding targets, English scientific text, title+abstract only for entities. Our setting: dependency extraction and KB construction (SP2) — extracting structured dependencies/relations among scientific concepts and building/maintaining a KB from literature. Differences:

  • Relation/dependency extraction: missing link — the paper omits its relation-extraction models entirely, which is exactly SP2's core interest; only node/citation backbone is validated.
  • Full-text vs. title+abstract: reference only — paper restricts entity extraction to title+abstract for salience; dependency extraction may need full text.
  • KB grounding to fixed curated ontologies (UMLS/DBpedia): resolved/reference — shows a working pattern for grounding, but flags KB coverage gap for novel scientific concepts (distant-supervision remedy proposed, not evaluated).
  • Deployed scale + pooling of heterogeneous extractors: resolved/pattern reusable — pooling multiple extractors to raise yield is a directly transferable design decision.
  • Domain scope (CS/Bio only, no physics/affiliations): reference only — coverage limits acknowledged.
