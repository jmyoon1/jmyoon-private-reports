Evaluating Position Bias in Large Language Model Recommendations

Ethan Bito et al. · arXiv · 2025

arXiv:2508.02020 · PDF · SP: SP2

P4Track B

한 줄 요약

LLM recommender가 prompt 내 후보 위치에 민감하게 반응하는 내부 position bias를 측정한다. 표시 position bias 위에 측정되지 않은 두 번째 bias 곡선이 쌓임을 보인다.

Problem해결하려는 문제

LLM-based recommenders built on list-wise ranking prompts ("rank the following items: item1, item2, ...") had no systematic study of whether the output ranking depends on the order in which candidates appear in the prompt. The paper shows this dependence is strong: the same candidate set in a different order yields substantially different rankings, meaning the LLM relies on candidate position rather than relevance — a direct exposure-distortion mechanism in any pipeline that uses an LLM as re-ranker.

Solution / Contribution해결책 / 기여

Two parts:

  1. Measurement: prompt the LLM $T$ times with randomly shuffled candidate lists (and original-vs-reversed pairs), compute pairwise Kendall's tau between output rankings. Metrics: Positional Consistency (PC, original vs. reversed), Output Similarity (Sim, across shuffles), Input Sensitivity (Sens, tau between input order and output order — lower is better, since a high value means the output copies the input order).
  2. Mitigation (RISE): replace one big list-wise ranking call with $K$ iterative single-item selection calls (select → append to ranking → remove from candidates → repeat). No fine-tuning, no post-processing.
Key Results주요 결과
  • Standard prompting degrades with candidate-list size: MovieLens PC 0.67 ($K{=}10$) → 0.47 ($K{=}30$); RISE@1 stays 0.75 → 0.69. Amazon Books: standard 0.55 → 0.47; RISE@1 0.65 → 0.65.
  • RISE reduces position bias up to 25% vs. baselines while keeping Recall@5/NDCG@5 competitive (MovieLens $K{=}10$: RISE NDCG@5 0.71 vs. standard 0.66; Amazon Books shows a small accuracy cost: 0.68 vs. 0.75).
  • Bootstrapping (aggregate rankings over shuffled prompts, $T{=}9$, Borda count) improves output similarity but not positional consistency (e.g. MovieLens $K{=}30$: PC 0.48 vs. standard 0.47).
  • Popularity-bias probe (most SP2-relevant): sampling candidates from Top/Middle/Bottom popularity strata changes PC/Sim only modestly (PC 0.63–0.67 standard, 0.74–0.76 RISE) — the authors conclude popularity bias is not the primary driver of the LLM's ordering instability; position in the prompt is. Middle/Bottom strata actually get slightly higher accuracy.
  • RISE@N degrades as $N$ grows: PC 0.72 ($N{=}1$) → 0.61 ($N{=}5$) at $K{=}20$ — smaller per-call decisions mean less order sensitivity.

Concepts Introduced도입된 개념

Position bias (LLM-recommendation variant)

  • What it computes: given the same candidate set $C$ presented in different orders inside the prompt, the divergence between the output rankings $R$ the LLM produces — quantified as average pairwise Kendall's tau across $T$ shuffled-prompt runs (low tau = high bias).
  • Why it was created: prior LLM-recommender work formulated recommendation as list-wise ranking prompts without checking whether the output depends on input item order rather than item relevance.
  • Concrete example: reversing a 5-movie candidate list changes which movie the LLM ranks first (Fig. 1); standard prompting on MovieLens drops from positional consistency 0.67 at $K{=}10$ to 0.47 at $K{=}30$.
  • How it is used in this paper: the central measured phenomenon; defined formally (Def. 3.1) and quantified via Positional Consistency (Kendall's tau between rankings from an original vs. reversed list) and Output Similarity (avg pairwise tau across shuffles).

RISE (Ranking via Iterative SElection)

  • What it computes: input = user history $I$ + candidate list $C$; operation = prompt the LLM to select exactly one item ($r_k \sim P_\theta(r \mid \text{INST}, I, C)$), append it to the ranked list, remove it from $C$, repeat until $C$ is empty; output = full ranking built one item per LLM call.
  • Why it was created: single-pass list-wise ranking prompts exhibit high position bias; shrinking the per-call decision to "pick one item" reduces order sensitivity without fine-tuning or post-processing.
  • Concrete example: on MovieLens $K{=}30$, RISE@1 holds positional consistency at 0.69 ± 0.11 vs. 0.47 ± 0.17 for standard prompting.
  • How it is used in this paper: the proposed mitigation, compared against standard prompting and bootstrapping; RISE@N variant (select $N \in \{1,3,5\}$ per call) shows $N{=}1$ works best.

Follow-up Papers관련 후속 논문
  • Hou et al. 2024, "Large Language Models are Zero-Shot Rankers for Recommender Systems" — source of the bootstrapping baseline; the predecessor mitigation this paper improves on.
  • Ma et al. 2023, "Large Language Models are Not Stable Recommender Systems" — two-stage Bayesian probe-and-calibrate alternative mitigation for the same instability.
  • Shi et al. 2025, "Judging the Judges: A Systematic Study of Position Bias in LLM-as-a-Judge" — the same bias in the evaluation (judge) role; relevant if SP2 covers LLM-judged ranking quality.
  • Liu et al. 2023, "Lost in the Middle" — the context-position mechanism most likely underlying the observed bias (the root cause RISE routes around).
Condition Delta — transfer to our setting조건 차이 — 우리 설정으로의 전이

Paper's setting: inputs = user interaction history + candidate list of $K \in \{10,20,30\}$ item titles in a text prompt; outputs = full ranking of the candidates; conditions = zero-shot LLaMA 3.3 70B, MovieLens-1M and Amazon Books, ground truth = user's own top-3 rated items among candidates. Our setting: exposure frequency used as a proxy for ranking quality/relevance — we care about mechanisms by which an item's exposure is distorted independently of its relevance. Differences:

  • Bias axis: paper measures prompt-position bias (where the item sits in the input), our SP2 concern is exposure-frequency distortion. The paper's Sec. 4.4 explicitly tests the popularity→position interaction and finds it weak — this is evidence that in LLM re-rankers, prompt position is an exposure-distortion channel separate from item popularity: resolved: Sec. 4.4 directly addresses the link.
  • Scale/model: single 70B open model, two benchmark datasets; production re-rankers use other models and larger candidate pools: reference only.
  • Feedback loop: paper is one-shot evaluation; exposure distortion in our setting compounds over repeated ranking rounds (position bias → exposure → training data). The paper does not model the loop: missing link.
  • Metric: Kendall's-tau stability, not item-level exposure counts; converting PC/Sim into per-item exposure-probability skew would be needed for our use: missing link.
Root-cause vs. Symptom근본 원인 대 증상

Symptom — RISE reduces the observable order sensitivity by shrinking each LLM call to a one-item decision, a prompting workaround; it neither explains nor removes the underlying cause of positional attention (e.g., serial-position effects in the context window), which the paper leaves untouched. The measurement half is honest diagnosis, but the mitigation patches the effect.
