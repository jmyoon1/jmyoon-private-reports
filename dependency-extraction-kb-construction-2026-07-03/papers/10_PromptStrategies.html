Prompting Strategies for Citation Classification

Kunnath et al. · CIKM 2023 · 2023 · 22 citations인용 22회

SP: SP1

P0Track A

Problem
해결하려는 문제

Prior citation classification relies on supervised models trained on citation-context datasets annotated for intent/function. Such labelled datasets are small and scarce, capping supervised performance. It was unclear whether prompt-based learning and LLMs (which excel in zero/few-shot regimes) can beat supervised baselines for this specific low-resource task, and which prompting regime works best.

LLM 시대에 인용 분류를 위한 다양한 prompting 전략의 상대적 효과가 정리되지 않았다.

Solution / Contribution
해결책 / 기여

Empirical head-to-head comparison of five strategies on citation classification: (1) Promptless fine-tuning (baseline), (2) Fixed-prompt LM tuning, (3) Dynamic Context-prompt LM tuning (proposed), (4) Prompt + LM fine-tuning (proposed), (5) Tuning-free zero-shot prompting with GPT-3.5. Empirical finding: strategies that update LM parameters significantly outperform the promptless baseline and the tuning-free LLM; the proposed Dynamic Context-prompt method is best overall.

고정 프롬프트/prompt tuning/instruction 기반 등을 지도 baseline과 정면 비교하여, tuning-free zero-shot GPT-3.5가 세분화 ACL-ARC에서 취약하고 LM 파라미터를 갱신하는 전략만이 우세함을 보인다.

주요 결과
  • Dynamic Context-prompt method achieves top scores on ACL-ARC and ACT2,

surpassing the highest-performing system in the 3C shared task benchmark. (Exact F1 numbers .)

Citation classification (citation function / intent)

  • What it computes: input = a citing sentence / citation context (the text

around a [CITATION] marker) → operation = classify the author's purpose for citing → output = a discrete label (e.g., background, method-use, comparison, motivation, future work).

  • Why it was created: reference lists alone do not say why a paper is cited;

intent labels convert a flat citation edge into a typed relation.

  • Concrete example: "We build on the transformer of [CITATION]" → method-use.
  • How it is used in this paper: the target task; evaluated on ACL-ARC and ACT2

datasets and the 3C shared task benchmark. Directly relevant to SP1 (citation intent/function classification for derivation-lineage dependency extraction).

Fixed-prompt LM tuning

  • What it computes: input = citing sentence wrapped in a fixed natural-language

prompt template with a [MASK]/verbalizer slot → operation = fine-tune LM parameters while the prompt template stays fixed → output = predicted label token.

  • Why it was created: adapts prompt-based learning to low-data citation

classification where full supervised datasets are scarce.

  • Concrete example: template like "The citation is used for [MASK]."
  • How it is used in this paper: strategy (2), one of four LM-updating strategies.

Dynamic Context-prompt LM tuning (proposed)

  • What it computes: input = citing sentence + a dynamically constructed

context prompt (context varies per instance rather than a single fixed template) → operation = LM parameter update during training → output = citation label. Exact prompt-construction rule .

  • Why it was created: fixed prompts underuse instance-specific context; dynamic

context aims to inject more relevant signal per example.

  • How it is used in this paper: the headline contribution — reported to achieve

Prompt + LM fine-tuning (proposed)

  • What it computes: jointly tunes both the (soft) prompt and LM parameters.
  • Why it was created: strategy (4); tests whether tuning prompt and LM together

Tuning-free prompting (zero-shot GPT-3.5)

  • What it computes: input = prompt-wrapped citing sentence → operation = LLM

inference with no parameter update → output = label.

  • Why it was created: baseline for whether large LLMs can classify citation

intent without any training data.

  • Concrete example: prompt GPT-3.5 to name the citation purpose zero-shot.

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = single citing sentence / citation context; outputs = citation intent/function label (ACL-ARC ~6-class, ACT2 scheme); conditions = low-resource, LM fine-tuning on small annotated datasets, English scientific text. Our setting: extract derivation-lineage dependencies (which prior work a paper builds on / uses / extends) to construct a KB; input = reference list + citation contexts across a paper; output = typed dependency edges between papers. Differences:

  • Granularity: paper classifies individual citation sentences; we need

paper-to-paper dependency edges aggregated across all mentions of a reference — missing link (aggregation from sentence-level to edge-level not addressed here).

  • Label scheme: ACL-ARC/ACT2 intents ≈ background/method-use/comparison map partly

onto derivation-lineage; "method-use"/"extension" are the edges we care about — reference only (label-to-dependency mapping is ours to define).

  • Method: prompt-tuning small LMs vs. our likely LLM extraction pipeline — resolved

in principle: their finding "LM-updating prompts > zero-shot LLM" is a directly usable adoption signal for SP1 (prefer prompt/LM-tuning over pure zero-shot).

  • Multi-hop lineage / KB construction beyond single-citation labeling — reference only.
