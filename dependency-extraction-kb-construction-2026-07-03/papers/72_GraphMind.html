GraphMind: Interactive Novelty Assessment System for Accelerating Scientific Discovery

Italo Luis da Silva et al. · EMNLP 2025 (Demo) · 2025 · 2 citations인용 2회

arXiv:2510.15706 · PDF · SP: SP5

P0Track B

Problem
해결하려는 문제

Existing scientific-review/recommendation tools fall into two disjoint camps and neither grounds a novelty judgement traceably. Macro-level tools (Connected Papers, Inciteful, Litmaps, ResearchRabbit; SPECTER embeddings) build citation/keyword graphs but miss deeper semantic relationships, are opaque, and never extract a paper's internal content. Micro-level tools (PeerRead, SciND, SchNovel, NovaScore) judge from content alone — SchNovel even proxies novelty by publication date. No tool integrates the interaction between a paper's internal claim→method→experiment structure and the surrounding literature, which is exactly what a defensible novelty assessment requires, nor does it offer verifiable, retrieval-grounded traceability.

단발 LLM 신규성 라벨은 불안정하며 내용만으로는 신뢰도가 낮다.

Solution / Contribution
해결책 / 기여

System (not a new objective). Pipeline in three stages:

  1. Graph extraction: arXiv LaTeX → Pandoc → Markdown; LLM (GPT-4o / GPT-4o-mini / Gemini 2.0 Flash) extracts claims/methods/experiments + interconnections + supporting excerpts → hierarchical Paper Structured Graph.
  2. Related-papers retrieval: (a) bibliography citations, filtered by semantic similarity, polarity-classified into supporting/contrasting; (b) semantic neighbours from Semantic Scholar recommendations, matched via background/target decomposition and MiniLM cosine similarity, labeled background/target.
  3. Novelty assessment: topologically sort the graph into a node-paragraph chain (micro) + evidence list from related papers (macro) → evaluation LLM emits a novelty label + structured rationale (result summary, supporting evidence, contradictory evidence); score = average over repeated promptings.

Web tool: vanilla-TypeScript multi-page frontend + Python/FastAPI backend (REST + Server-Side Events), three input modes — pre-computed Library (ICLR 2022–25, NeurIPS 2022–24), live arXiv search, and title+abstract draft evaluation (macro-only, no graph).

claim→method→experiment 내부 그래프와 stance-typed 계보를 결합해 반복 샘플링으로 신규성 점수를 산출하는 GraphMind로 F1 0.75를 달성하여 내용·인용 단독 baseline을 능가한다.

Key Results
주요 결과
  • GraphMind-Gemini best overall: F1 0.7500, Accuracy 0.7400 vs. Basic-Gemini (F1 0.6619, Acc 0.5300) and Search-Gemini (F1 0.6923, Acc 0.6800).
  • GraphMind-GPT-4o: F1 0.7033, Acc 0.7300 vs. Basic-GPT-4o F1 0.6931 / Acc 0.6900; the curated retrieval beats the LLM's own web search (Search-GPT-4o F1 0.6531).
  • Ablation: removing the structured graph ("No graph") drops F1 to 0.6942 / Acc 0.6900; "No related" Acc 0.6900 — both graph and related-paper components contribute.
  • Bradley-Terry rationale tournament (GPT-4o judge): GraphMind exceeds human reviews on Faithfulness (1552 vs 1476), Factuality (1609 vs 1470), Specificity (1657 vs 1443); roughly ties on Clarity, below humans on Contributions (1540 vs 1584).
  • Cost/time: Gemini 2.0 Flash is cheapest ($0.023) and fastest (62s) and best-performing.
Concepts Introduced도입된 개념

Paper Structured Graph (micro-level idea graph)

  • What it computes: full LaTeX paper content (via arXiv API → Pandoc → Markdown) → LLM extraction of key components (claims, methods, experiments) plus their interconnections and supporting excerpts → a hierarchical graph where claims link to the methods that validate them and methods link to the experiments that evidence them.
  • Why it was created: prior review/recommendation tools operate only at the macro (citation/semantic) level or only on isolated content, and cannot show how a paper's internal argument (claim→method→experiment) maps onto the external literature.
  • Concrete example: for a "Reward Learning From Preference With Ties" paper, nodes are Title → Claim ("Introduction of BTT for preference modeling") → Method ("Bradley-Terry model with ties", "Bias-correction method") → Experiment ("Simulation experiment with BTT", "Evaluation on synthetic preference dataset").
  • How it is used in this paper: rendered as an interactive node graph on the Detail page; also serialized (topological sort → linear node chain → paragraph per node) as micro-level input to the novelty-assessment LLM.

Background/Target abstract decomposition

  • What it computes: an abstract → LLM splits it into a background component (works that inform/motivate) and a target component (the paper's own goal) → each is embedded (all-MiniLM-L6-v2) and matched by cosine similarity against candidate papers' backgrounds/targets.
  • Why it was created: retrieving related work on whole-abstract similarity conflates "same problem" with "same background", producing noisy neighbours; decomposing improves semantic retrieval precision.
  • Concrete example: a related paper is tagged background if its content matches the main paper's background, or target if it addresses a similar goal; each shown with a percentage similarity score.
  • How it is used in this paper: drives the semantic branch of the Related Papers retrieval and the background/target labels in the report.

Citation-context polarity classification

  • What it computes: for each bibliography citation, all in-text sentences where it appears → LLM classifies the aggregate context as positive/negative → citation labeled supporting (mostly positive) or contrasting (mostly negative).
  • Why it was created: co-citation/bibliographic-coupling graphs (Connected Papers, Inciteful) treat all citations as undifferentiated edges, losing the stance that matters for novelty judgement.
  • Concrete example: citations filtered by semantic similarity first, then polarity-labeled; feeds Supporting vs. Contradictory Evidence in the report.
  • How it is used in this paper: macro-level input; supporting/contrasting citations become evidence items linked to the novelty rationale.

Novelty score (repeated-sampling percentage)

  • What it computes: paper graph (as text) + related-paper evidence list → evaluation LLM prompted multiple times for a novelty label → predictions averaged into a percentage score.
  • Why it was created: single-shot LLM novelty labels are unstable; averaging reduces variance.
  • Concrete example: score shown as e.g. "40%" on the Metadata section; binary ground truth = median peer-review originality (1–3 "not novel", 4–5 "novel").
  • How it is used in this paper: the headline output, evaluated against OpenReview originality scores.

5Cs Analysis5Cs 분석

Category: system / engineering (EMNLP 2025 demo paper). Context: builds on SPECTER (citation embeddings), Scholar Inbox, PeerRead (novelty definition + dataset), SciND (KG novelty triplets), SchNovel, NovaScore (ACU novelty); integrates arXiv + Semantic Scholar APIs, SentenceTransformers (all-MiniLM-L6-v2). Directly relevant to SP5 (agent's new-node decision O2): a concrete design for grounding a novelty judgement in retrieved lineage. Correctness: evaluated on 3,063 papers with peer-review median originality as ground truth; reports precision/recall/F1/accuracy + ablations + a Bradley-Terry rationale tournament. No confidence intervals; binary thresholding of a 5-point rubric is coarse; LLM-as-judge (GPT-4o) may favor GraphMind's own style. Contributions: (1) micro+macro integrated idea graph for novelty; (2) background/target decomposition for semantic retrieval; (3) working interactive tool with open source + live demo. Clarity: architecture and pipeline (Appendix C) are reproducible at a high level; exact prompts not given.

Contribution vs. Claim기여 대 주장

Stated claims: verifiable, traceable novelty assessment integrating macro+micro levels, outperforming direct prompting and matching/exceeding human reviews. Demonstrated: F1/accuracy gains over Basic/Search baselines on ICLR/NeurIPS + the BT rationale tournament support these. Potential gap: "verifiable/traceable" is a UI affordance not directly measured; ground truth is binarized peer-review originality (a noisy proxy); LLM-as-judge favoring the system's own verbose rationales is a plausible confound for the "exceeds human" claim. Evaluation is ML-venue-only (authors note extending beyond ML as future work).

Implementation Feasibility구현 실현가능성
  • Needs: arXiv API, Semantic Scholar API, Pandoc, SentenceTransformers (all-MiniLM-L6-v2), an LLM (GPT-4o / GPT-4o-mini / Gemini 2.0 Flash), FastAPI backend + TypeScript frontend. ~62s and $0.023 per paper with Gemini Flash.
  • Code: available at https://github.com/oyarsa/graphmind; live demo https://oyarsa.github.io/graphmind; video https://youtu.be/wKbjQpSvwJg.
  • Modifications: for our KB agent, keep the graph-extraction + background/target decomposition + citation-polarity modules; replace the Semantic-Scholar-recommendation retrieval with retrieval over our own KB; replace the human-facing novelty report with a machine-consumable new-node/link decision and persistent graph insertion.
  • Unknowns: missing link — exact extraction prompts and the graph schema (node/edge types) are not fully specified in the paper; SciNova benchmark composition only partially described.
Follow-up Papers관련 후속 논문
  • NovaScore (Ai et al., 2024) — ACU-based document novelty; the retrieval-by-salience baseline GraphMind argues against.
  • SciND (Gupta et al., 2024) — triplet knowledge-graph for novelty detection; closest KG-based comparator, relevant to our dependency-KB framing.
  • SchNovel (Lin et al., 2024) — LLM novelty assessment over 150k arXiv papers; the date-proxy baseline GraphMind critiques.
  • PeerRead (Kang et al., 2018) — source of the novelty definition and an evaluation aspect set.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = a full arXiv paper (LaTeX) or title+abstract draft; outputs = novelty label/score + structured rationale + interactive idea graph + supporting/contrasting related papers; conditions = ML papers (ICLR/NeurIPS), human-in-the-loop web tool, external arXiv + Semantic Scholar APIs, commercial/open LLMs. Our setting: an autonomous agent constructing a dependency KB, needing to decide whether an incoming paper/idea is a genuinely new node vs. a duplicate/child of existing lineage (SP5, objective O2). Differences:

  • Interactive human-in-the-loop tool vs. autonomous agent decision: reference only — the novelty pipeline (graph + retrieved lineage → judgement) is the borrowable core, the web UI is not.
  • Output is a per-paper novelty percentage vs. our needed new-node/link decision over a KB: missing link — no mechanism to insert the assessed paper as a node with typed edges into a persistent graph.
  • Retrieval scoped to arXiv + Semantic Scholar recommendations vs. our own KB corpus: resolved conceptually / background-target decomposition + citation-polarity are corpus-agnostic and reusable against our own index.
  • Novelty defined via PeerRead originality rubric (peer-review proxy) vs. structural novelty in a dependency graph: missing link — different notion of "new".
