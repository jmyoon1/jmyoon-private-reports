Enquire One's Parent and Child Before Decision: Fully Exploit Hierarchical Structure for Self-Supervised Taxonomy Expansion

Wang et al. · WWW 2021 · 2021

arXiv:2101.11268 · PDF · SP: SP9

P0Track A

Problem
해결하려는 문제

Prior taxonomy-expansion methods (TaxoExpan ego-nets, STEAM mini-paths, Arborist, ETF) neglect the hierarchical structure: they model only a local subgraph and evaluate the correctness of a single added edge (anchor→query), downgrading the task to node-pair scoring or mini-path classification. They (i) do not model a structure identical to the taxonomy, (ii) ignore the coherence of the whole expanded taxonomy, and (iii) treat the anchor as an isolated node without hierarchical context (path/level correctness).

기존 확장법은 단일 anchor→query edge만 평가하여 모든 조상·형제와 동시에 비교하지 못한다.

Solution / Contribution
해결책 / 기여

The Hierarchy Expansion Framework (HEF) maximizes coherence of the expanded taxonomy rather than fitness of one edge, via three designs:

  1. Ego-tree self-supervision — subtree (all ancestors + sampled children) as the training/inference unit, auto-generated from the seed taxonomy.
  2. Coherence modeling — an ego-tree Transformer encoder combining DistilBERT node-pair hypernymy detection with absolute/relative level embeddings.
  3. Fitting Score — Pathfinder (path selection) + Stopper (level selection) with parent/child information exchange for disambiguation and level self-correction, trained self-supervised multi-task.

조상+형제로 구성된 ego-tree 전체의 coherence(Pathfinder: 경로, Stopper: 레벨)를 채점하는 HEF를 제안하여 계층 링크 예측의 평가 단위를 edge에서 트리 이웃으로 재정의한다.

Key Results
주요 결과
  • HEF surpasses prior SOTA on three SemEval-2016 Task 13 benchmark datasets by an average +46.7% accuracy and +32.3% mean reciprocal rank (MRR).
  • Ablation studies reportedly confirm effectiveness of each component.
Concepts Introduced도입된 개념

Ego-tree

  • What it computes: node → subtree extraction → structure containing all ancestors (root-path) plus a sample of children of a candidate anchor node, used as the self-supervision unit instead of ego-nets or mini-paths.
  • Why it was created: prior methods (TaxoExpan ego-nets [34], STEAM mini-paths [47]) model only a fragment of the hierarchy and evaluate a single edge, so they cannot compare a query against all ancestors (to pick the right hypernym level) and siblings (to check similarity) simultaneously.
  • Concrete example: for query "black tea", the ego-tree of anchor "tea" holds ancestors {beverage, food} and children {iced tea, oolong} so the model can distinguish "tea" from "beverage"/"food" and confirm sibling similarity.
  • How it is used: the input structure fed to the ego-tree Transformer encoder for coherence modeling.

Coherence Modeling Module

  • What it computes: query + anchor's ego-tree → hypernymy detection (per node-pair) + hierarchy-exclusive features (absolute-level and relative-level embeddings) → representation of whether the ego-tree stays coherent after inserting the query.
  • Why it was created: prior work scores only single-edge correctness, ignoring expert-curated level conventions (e.g., "add one adjective = exactly one level deeper").

  • How it is used: middle stage of HEF, producing the ego-tree representation used by the Fitting Score.

Fitting Score (Pathfinder + Stopper)

  • What it computes: expanded ego-tree → Pathfinder (path-selection correctness) + Stopper (level-selection correctness) → scalar score; uses the anchor's parent and one child to disambiguate and self-correct level ("go up / go down").
  • Why it was created: to evaluate path vs. level correctness separately and enable self-correction, which single-edge scoring cannot.
  • Concrete example:
  • How it is used: final position-selection criterion, trained by a self-supervised multi-task paradigm with auto-generated data from the seed taxonomy.

Description generation algorithm descr(·)

  • What it computes: term surface name → dynamic-programming assembly of tokens into explainable noun phrases → WordNet-based description; picks the description most relevant to the taxonomy root for domain relevance.
  • Why it was created: surface names alone are insufficient for hypernymy detection and WordNet has low coverage / cannot explain complex phrases (e.g., "bacon lettuce tomato sandwich").
  • How it is used: feeds the DistilBERT hypernymy detection module input [<CLS> descr(a) <SEP> descr(b) <SEP>].

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = seed taxonomy (arborescence) + query terms + WordNet definitions; outputs = best-suited parent (single-parent attachment, no restructuring of seed); conditions = SemEval-2016 Task 13 taxonomies, self-supervised training data auto-generated from the seed tree. Our setting: dependency-extraction / KB construction (SP9) — hierarchy-structured link prediction where new items are placed relative to existing structured knowledge; likely larger/noisier corpus-derived graphs, possibly multi-parent (DAG) rather than strict tree. Differences:

  • Strict single-parent arborescence vs. our possible DAG/multi-parent dependencies: missing link — HEF assumes an arborescence.
  • WordNet-definition availability: reference only — our domain may lack curated definitions; descr(·) would need a domain description source.
  • "Coherence" objective (parent+sibling comparison) as an evaluation standard for hierarchy link prediction: resolved: / directly transferable framing — SP9 cites this as a foundational evaluation standard for hierarchy-structured link prediction.
