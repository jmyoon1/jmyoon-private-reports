Group Fairness Meets the Black Box: Enabling Fair Algorithms on Closed LLMs via Post-Processing

Ruicheng Xian et al. · arXiv · 2025

arXiv:2508.11258 · PDF · SP: SP2

P4Track A

한 줄 요약

닫힌 LLM 위의 분류 공정성에서 引出된 출력 확률이 최적 공정 post-processor의 충분통계량임을 증명한다. ranking/exposure에 대한 대응 정리는 부재하며, 이것이 본 조사의 open gap 5의 근거이다.

Problem해결하려는 문제

Existing approaches to enforcing group fairness on LLM-based classifiers (fine-tuning, prompt-tuning, head-tuning on last-layer embeddings) require access to model weights or internal representations, which closed-weight commercial LLMs do not provide — the API exposes only output tokens and at best top-k token log-probabilities. The alternative, instructing the model in the prompt to "assign labels equally across groups," is sensitive to prompt phrasing and gives no principled way to iterate when the fairness target is missed. Prior post-hoc work that operates only on predictions (Baldini et al., 2022) is limited to the attribute-aware setting where group membership is observed at test time.

Solution / Contribution해결책 / 기여

A three-step post-processing framework treating the closed LLM as a frozen feature extractor:

  1. Design prompts to elicit predictions of the joint (A, Y) — either a single MCQA prompt with G×K choices, or a factorization P(A,Y|X) = P(Y|X)·P(A|Y,X) using (1+K) prompts (e.g., "Given the income is >$50k, what is the race?").
  2. Query the LLM per example, extracting logits of the MCQA option tokens (missing top-k logits filled with −50 for GPT-4o).
  3. Featurize and apply a fair algorithm: reshape logits into a G×K vector encoding log P(A,Y|X) (exact log-sum-exp composition rules in Appendix C), calibrate with logistic regression against ground-truth (A,Y) labels, then train a lightweight fair classifier (Reductions, MinDiff, or LinearPost) on the calibrated simplex features.

Justification: recent post-processing theory (Chen et al. 2024; Xian & Zhao 2024) shows the Bayes-optimal fair classifier is a simple (e.g., linear) function of P(Y|X) and P(A,B|X) — so accurate LLM estimates of these quantities are sufficient statistics for optimal fair classification. Supports the attribute-blind setting (A unobserved at test time), generalizing Baldini et al.

Key Results주요 결과
  • Across 5 datasets × 4 fairness criteria × 4 LLMs × 3 fair algorithms, preds (proposed) achieves consistently strong AUTC; on Adult with fewer than ~1000 training examples, preds outperforms both embeds (LLM embeddings) and tabular features, attributed to sample complexity: 4-dim features vs. 4096-dim embeddings (Fig. 4).
  • Three regimes emerge as training size grows: preds wins in very-low-data (<1000), tabular in low-to-mid, embeddings in high-data; preds plateaus due to its information bottleneck (Fig. 4).
  • Ablation: dropping the group-attribute prediction (Y-only features) degrades AUTC in nearly all cases (e.g., Adult SP top score drops 0.305 → 0.268), except where the LLM predicts A poorly (COMPAS: balanced accuracy 0.656) or Y already encodes A (BiasBios: A predictable from Y-logits at 0.877 balanced accuracy) (Fig. 5, Sec. 6.3).
  • Works with a genuinely closed model: GPT-4o with only top-20 logits, generally the strongest model per-configuration.
5Cs Analysis5Cs 분석

Category: new method (system/framework combining existing fair algorithms with prompt-based feature elicitation) Context: builds on Bayes-optimal fair post-processing theory (Xian & Zhao 2024 LinearPost; Chen et al. 2024; Zeng et al. 2024), TabLLM few-shot tabular classification (Hegselmann et al. 2023), and generalizes Baldini et al. 2022's attribute-aware prediction-level mitigation. Within this survey it is the immutable-scorer downstream-intervention exemplar for SP2. Correctness: results averaged over 5 seeds with uncertainty bars; Pareto-filtering on validation, scoring on test — reasonable protocol. Assumption that elicited predictions are sufficient statistics holds only if the LLM is Bayes-optimal, which the authors themselves flag as unrealistic (Sec. 7). AUTC cutoff/penalty choices are somewhat ad hoc. [SKIM — statistical rigor of the weighted-FPR variant for overlapping groups not verified in detail] Contributions: (1) a prompts-to-sufficient-statistics pipeline making any standard fair algorithm applicable to closed-weight LLMs in the attribute-blind setting; (2) evidence of data efficiency — the low-dimensional (KG) features beat 4096-dim embeddings and raw tabular features below ~1000 training examples; (3) the AUTC metric plus an ablation showing group-attribute predictions are usually necessary. Clarity: reproducible — prompt templates (Listings 1–5), feature formulas (App. C), hyperparameters (App. D), and code are all provided.

Contribution vs Claim기여 대 주장
  • Stated claims: framework enables any fair algorithm on closed LLMs; data-efficient; supports attribute-blind setting and broad fairness criteria.
  • Demonstrated: classification only, zero-shot prompting only, 5 datasets, one closed-weight model (GPT-4o). Data-efficiency regime analysis only on Adult.
  • Gap: the "sufficient statistics" optimality argument requires Bayes-optimal LLM predictions — acknowledged as unlikely in practice (Sec. 7); the observed performance plateau vs. continued embedding gains is direct evidence of this gap. Generality across prompting styles (few-shot, CoT) is claimed but not evaluated.
Concepts Introduced도입된 개념

LLM-as-sufficient-statistic feature extraction for fair classification

  • What it computes: input example x → MCQA prompts eliciting token log-probabilities for the task label Y and the sensitive attribute A (jointly or factored as P(Y|X)·P(A|Y,X)) → a G×K logit vector semantically encoding log P(A,Y|X), calibrated via logistic regression into probabilities on the simplex Δ^{G×K}, used as the feature vector for a downstream fair algorithm.
  • Why it was created: fine-tuning, prompt-tuning, and head-tuning on last-layer embeddings all require weight or representation access, which closed-weight LLMs (GPT-4, Gemini, Claude) do not expose; prompt-based fairness instructions are inconsistent and offer no principled iteration path.
  • Concrete example: on Adult (K=2, G=2 for sex), the extracted feature is only 4-dimensional vs. 4096-dim Llama 3.1 8B embeddings; on ACSIncome it is 10-dimensional (2 income classes × 5 race groups).
  • How it is used in this paper: the calibrated P(A,Y|X) estimates are fed to three fair algorithms (Reductions, MinDiff, LinearPost) to train a lightweight fair classifier; the LLM stays frozen throughout.

Area Under the Tradeoff Curve (AUTC)

  • What it computes: a set of (accuracy, fairness-violation) points from a tolerance sweep → Pareto-filter, interpolate, integrate accuracy above the majority-class base rate within a violation cutoff v with penalty weight (v−u)^γ → a single score in [0,1] summarizing the accuracy-fairness tradeoff.
  • Why it was created: comparing 24 configurations per dataset via full tradeoff curves is unwieldy; naive area metrics reward high-accuracy/high-violation classifiers, so the γ-penalty re-weights the low-violation regime.
  • Concrete example: on Adult SP, AUTC scores range roughly 0.024–0.305 across configurations (Fig. 3), with γ=1 and per-dataset cutoffs (e.g., v=0.177 for Adult SP).
  • How it is used in this paper: the primary evaluation metric across all five datasets and all configurations (Figs. 3–5).

Fairness criteria (statistical parity, equalized odds, TPR/FPR parity) and the fair algorithms (Reductions, MinDiff, LinearPost) are reused from prior work, defined on pp. 4, 22–24.

Implementation Feasibility구현 가능성
  • Needs: LLM API with token log-probabilities (top-k suffices), labeled examples with group annotations for calibration + fair-algorithm training, AIF360 (Reductions), Gurobi (LinearPost LP), small MLP training (MinDiff). Compute is light — the heavy cost is per-example LLM queries ((1+K) prompts per example under decomposition).
  • Code: available at https://github.com/uiuctml/fair-classification
  • Modifications: for the survey's ranking setting, the post-processor's output space must change from label decisions to score adjustments or re-ranking; the fairness constraint must change from output-distribution parity to exposure parity — substantial, both the elicitation targets and the fair algorithm would need ranking analogues.
  • Unknowns: missing link: whether a LinearPost-style Bayes-optimal linear post-processing result exists for exposure-based ranking fairness; missing link: sensitivity of the pipeline when only sampled outputs (no logits) are available (the paper defers to Monte-Carlo temperature sampling / verbal elicitation in a footnote, untested).

  • Ţifrea et al., "FRAPPÉ: A Group Fairness Framework for Post-Processing Everything" (ICML 2024) — the general-purpose post-processing baseline; compared conceptually via Atwood et al.
  • Atwood et al., "Inducing Group Fairness in Prompt-Based Language Model Decisions" (ICLR 2025 workshop) — head-tuning post-processing on LLM decisions; closest concurrent work.

Condition Delta — transfer to our setting조건 차이 — 우리 설정으로의 전이

Paper's setting: inputs = text/serialized-tabular examples; outputs = a fair classifier (K-way label decisions); conditions = group fairness (SP/TPR/FPR/EO) on classification, LLM logits accessible (at least top-k), labeled (x, a, y) training data available (100–20000 examples), attribute-blind at test time. Our setting (SP2 — exposure/frequency as a ranking proxy): inputs = items/documents scored by an immutable (often closed) scorer; outputs = a ranking whose exposure distribution across groups is the fairness object; conditions = fairness is about position-based exposure, not label parity. Differences:

  • Classification vs. ranking: the paper's fairness criteria equalize label distributions; exposure fairness equalizes position-weighted attention — missing link (the paper itself does not extend to ranking; the abstract's note that "the formulation transfers to ranking" is the searcher's inference, not the paper's claim).
  • Immutable scorer: matches — the LLM is fully frozen and only its output scores are consumed; resolved: this is exactly the downstream-only intervention pattern SP2 needs, at the classification level.
  • Sufficient-statistic recipe: eliciting P(A,Y|X) as the post-processor's input could translate to eliciting P(A, relevance|item) for exposure post-processing — reference only (would need the ranking analogue of the Bayes-optimal fair post-processing results).
  • Requires labeled (x, a, y) calibration data — missing link if group labels are unavailable in the ranking corpus.
Root-cause vs. Symptom근본 원인 대 증상

Mixed — the framework directly targets the root cause of the access problem (closed weights) by proving-out that output probabilities alone carry the sufficient statistics for fair post-processing, grounded in Bayes-optimality theory; but the optimality guarantee is conditional on Bayes-optimal LLM predictions, which fails in practice (the plateau in Fig. 4), so residual unfairness/accuracy loss from imperfect elicitation is left unresolved rather than characterized.
