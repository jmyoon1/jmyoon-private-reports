Neural Entity Linking: A Survey of Models Based on Deep Learning

Sevgili et al. · Semantic Web 2022 · 2020

arXiv:2006.00575 · PDF · SP: SP4

P1Track A

Problem
해결하려는 문제

Prior EL surveys (Shen 2015, Ling 2015, Martínez-Rodríguez 2020, Al-Moslmi 2020, Oliveira 2020, Möller 2021) either predate the neural wave, cover too many adjacent tasks broadly, or focus on a single method type / KG. None gives a detailed, dedicated systematization of post-2015 deep-learning EL models, and none covers entity/context-mention encoding, zero-shot/domain-independent EL, cross-lingual EL, or the application of EL to pretrained LMs.

신경망 개체 링킹의 다양한 설계가 통합적으로 정리되지 않았다.

Solution / Contribution
해결책 / 기여

A survey that (1) distills one generic neural EL architecture — candidate generation, mention-context encoding, entity ranking — plus NIL prediction; (2) groups architecture variants by theme (joint MD+ED end-to-end, global linking, zero-shot/distant-supervision domain-independent, cross-lingual); (3) overviews entity embedding techniques; (4) compiles model features and benchmark results into tables/taxonomies; (5) surveys EL applications, notably entity-aware pretraining of Transformer LMs (ELMo/BERT modifications).

candidate generation→encode→rank→NIL의 재사용 가능한 4-요소 아키텍처로 신경 EL 분야를 정리하며 NIL 예측을 일급 구성요소로 유형화한다.

Key Results
주요 결과
  • Survey scope: neural EL models published from 2015 onward; comprehensive feature table (Table 2) and benchmark performance summary (Section 4). Specific comparative numbers require deep read.
  • Compiles collection of public official EL implementations (Table 7, Appendix A).
Concepts Introduced도입된 개념

Entity Linking (EL)

  • What it computes: raw text → detect entity mention span + disambiguate → link each mention to an entry (URI) in a Knowledge Graph. Decomposed into Mention Detection (MD: C → M^n) and Entity Disambiguation (ED: (M,C)^n → E^n).
  • Why it was created: resolves lexical ambiguity of entity mentions (e.g., "Michael Jordan" basketball vs. mathematician) so downstream IE/NLU pipelines can exploit structured KG knowledge.
  • Concrete example: mention "Scott Young" in a soccer context → link to Scott_Young_(Welsh_footballer) rather than Scott_Young_(writer).
  • How it is used in this paper: the organizing task the entire survey taxonomizes.

Generic neural EL architecture

  • What it computes: three-stage pipeline — (1) Candidate Generation (select possible KG entities for a mention), (2) Mention-Context Encoding (vectorize mention + surrounding context), (3) Entity Ranking (score each candidate via vector comparison, pick highest).
  • Why it was created: distills a single reusable framework covering the majority of post-2015 neural EL models to structure a sprawling literature.
  • Concrete example: for "Scott Young" → candidates {writer, American football, Welsh footballer} → ranking model scores → Welsh footballer.
  • How it is used in this paper: backbone taxonomy (Figs. 2/5) under which all surveyed models are grouped.

NIL / unlinkable-mention prediction

  • What it computes: extends ED to (M,C)^n → (E ∪ NIL)^n — decides when a mention has no correct KG entry.
  • Why it was created: KGs are incomplete, so some mentions cannot be mapped to any entry.
  • Concrete example: a newly coined entity absent from Wikidata → predicted NIL.
  • How it is used in this paper: one of the four core components taxonomized (per SP4 framing).

Entity embeddings / entity encoding

  • What it computes: encode KG structure (relations), entity definitions, or word/entity co-occurrence into low-dimensional vectors.
  • Why it was created: distributed entity representations improve generalization of EL models on unbalanced training data.
  • Concrete example: entity vector for Wales_national_football_team learned from KG neighborhood + text co-occurrence.
  • How it is used in this paper: dedicated overview section; feeds the ranking stage.

Correctness: — authors note they did not run software or compare compute/runtime; comparison is on reported benchmark numbers only. Contributions: (1) generic neural EL architecture + feature systematization; (2) survey of entity/context-mention embedding techniques; (3) coverage of domain-independent (zero-shot), cross-lingual EL, and EL-for-LM applications. Clarity: clear task formalization (MD/ED/NIL functions given); reproducibility N/A for a survey.

Contribution vs. Claim기여 대 주장

— distinguishing stated claims from demonstrated coverage requires reading Sections 3–5. Abstract claims comprehensiveness for general-purpose EL on well-established benchmarks; explicitly excludes fine-grained entity typing and domain-specific (biomedical, social-media) EL, so "comprehensive" is scoped to general-domain models.

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = raw general-domain text (news, Wikipedia-linked corpora like AIDA); outputs = mentions linked to general KG entries (Wikipedia/DBpedia/Wikidata URIs); conditions = post-2015 neural models, general-purpose EL, evaluated on standard EL/ED benchmarks and an entity-relatedness dataset. Our setting: dependency extraction & KB construction (SP4) — likely scientific/technical text, linking extracted entities to a domain KB, possibly with emerging/NIL-heavy entities and few labeled examples. Differences:

  • Domain: general-domain Wikipedia/Wikidata EL vs. our specialized/scientific KB — reference only (survey excludes biomedical/domain-specific EL).
  • KG target: mature large public KG vs. an under-construction domain KB — missing link: high NIL rate and cold-start entities more central for us.
  • Task role: EL as standalone task vs. EL as a component of a dependency-extraction/KB-construction pipeline — resolved: survey's generic architecture (cand-gen → encode → rank → NIL) is directly reusable as a design template.
