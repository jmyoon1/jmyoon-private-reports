Enhancing Large Language Model for Knowledge Graph Completion via Structure-Aware Alignment-Tuning

Liu et al. · EMNLP 2025 · 2025

arXiv:2509.01166 · PDF · SP: SP9

P0Track B

Problem
해결하려는 문제

Embedding-based LLM-enhanced KGC methods (e.g., KoPA, GraphGPT) inject graph embeddings into LLMs but (1) ignore the representational gap between graph-structure space and natural-language space, limiting the LLM's ability to reason with structural knowledge, and (2) design separate task-specific instructions for each KGC task, causing redundancy and inefficiency.

임베딩 기반 LLM-KGC는 그래프 공간과 언어 공간의 표상 격차를 조정하지 못한다.

Solution / Contribution
해결책 / 기여

Two-stage framework SAT. Stage 1 — Hierarchical Knowledge Alignment: multi-task contrastive learning aligns graph embeddings to the NL space at node level (entity ↔ Wikipedia description) and subgraph level (GPT-4-extracted subgraph ↔ document), joint loss $L_{HKA}=L_{local}+L_{global}$. Stage 2 — Structural Instruction Tuning: a unified graph instruction $X=[I;Q;\text{EMB}(S_Q)]$ with k-hop query subgraph embeddings; freeze LLM + graph encoder, tune only a lightweight knowledge adapter with objective $L_{SIT}=\mathbb{E}[-\log P_M(Y|X)]$.

contrastive learning으로 그래프 임베딩과 텍스트 설명을 정렬한 뒤 경량 adapter만 튜닝하는 SAT를 제안하여 FB15k-237N에서 Hit@1 상대 +29.1%를 달성한다.

Key Results
주요 결과
  • Link prediction Hit@1: +29.1% (FB15k-237N) and +29.8% (CoDeX-S) relative over second-best; +13.1% (FB15k-237) and +8.7% (YAGO3-10) relative vs RED-GNN.
  • Triple classification F1: +4.5% (FB15k-237N) and +5.1% (CoDeX-S) relative vs PKGC; +2.8% / +2.9% vs KoPA.
  • Abstract headline: link-prediction improvements range 8.7%–29.8%. — full tables in pages 5+.
Concepts Introduced도입된 개념

SAT (Structure-aware Alignment-Tuning)

  • What it computes: input = incomplete KG + query (triple or (h,r,?)) → two-stage pipeline (hierarchical knowledge alignment then structural instruction tuning) → output = triple correctness (True/False) or ranked tail entities.
  • Why it was created: existing embedding-based LLM-KGC methods ignore the representational gap between graph structure space and natural-language space, and craft separate instructions per KGC task (redundant/inefficient).
  • Concrete example: on FB15k-237N link prediction, improves Hit@1 by 29.1% relative over the second-best model.
  • How it is used in this paper: the full proposed framework; backbone Llama2-Chat-7B with frozen LLM + frozen graph encoder, only a knowledge adapter tuned.

Hierarchical Knowledge Alignment (HKA)

  • What it computes: node-level: align each entity's graph embedding $H$ (graph transformer) with its Wikipedia-derived description embedding $D$ (vanilla transformer) via bidirectional contrastive/CE loss over similarity matrix $\alpha$ with identity labels; subgraph-level (global): align GPT-4-extracted subgraph embeddings (mean-pooled) with document embeddings. Joint loss $L_{HKA}=L_{local}+L_{global}$.
  • Why it was created: to close the inconsistent representation spaces between graph structure and natural language so the LLM can comprehend structural knowledge.
  • Concrete example: $\alpha = (H D^\top)\cdot\exp(\tau)$, label $Y=I_N$; local loss $L_{local}=\frac12(CE(\alpha,Y)+CE(\alpha^\top,Y))$.
  • How it is used in this paper: pre-trains the graph encoder whose embeddings feed the instruction-tuning stage.

Structural Instruction Tuning (SIT) + Knowledge Adapter

  • What it computes: unified graph instruction template (human question + graph information + model response); prompt $X=[I;Q;\text{EMB}(S_Q)]$ where $S_Q$ is the k-hop query subgraph; freezes LLM and graph encoder, tunes only a lightweight adapter (two-layer FFN / projection) to maximize $-\log P_M(Y|X)$.
  • Why it was created: to unify diverse KGC tasks (triple classification + link prediction) under one instruction paradigm, avoiding per-task instruction crafting.
  • Concrete example: LP template — "Given a question <question>... Graph Information: Given a sequence of graph embeddings <graph>..."; top-3 answers via beam search.
  • How it is used in this paper: second stage; the tuned adapter is the only trainable component.

5Cs Analysis5Cs 분석

Category: new method (LLM-enhanced KGC framework) Context: builds on embedding-based LLM-KGC — KoPA (prefix adapter), GraphGPT (graph instruction tuning), MKGL, CSProm-KG; contrasts with text-based KG-LLaMA/MPIKGC/GS-KGC. Directly relevant to SP9 (LLM/lightweight-fine-tuning link prediction for structured KBs). Correctness: reports Acc/Precision/Recall/F1 (triple classification) and Hit@1/MRR (link prediction) on 4 datasets; ablations claimed in appendix. Statistical rigor (variance/CIs) . Contributions: (1) framework integrating graph structure into LLMs via structure-aware alignment-tuning; (2) hierarchical knowledge alignment for space consistency + structural instruction tuning to unify KGC tasks; (3) SOTA on 2 tasks × 4 datasets. Clarity: method equations given in main text; reproducibility good with released code, but subgraph construction / adapter details .

Contribution vs. Claim기여 대 주장

— abstract/intro claim generality across "two KGC tasks across four benchmark datasets" and superiority especially on link prediction; experiments section (pages 5+) needed to confirm whether gains hold uniformly and whether ablations isolate HKA vs SIT contributions. Distinguishing claimed unification benefit from demonstrated per-task numbers requires the full results/ablation tables.

Implementation Feasibility구현 실현가능성
  • Needs: Llama2-Chat-7B, graph transformer + vanilla transformer encoders, Wikipedia entity descriptions, GPT-4 access for subgraph-document pair generation, ~2× A800 GPUs, PyTorch.
  • Code: available at https://github.com/liuyudiy/SAT
  • Modifications: replace Wikipedia descriptions with domain-specific entity text; substitute/skip GPT-4 global-alignment step if descriptions scarce; adapt query-subgraph k-hop construction to dependency-graph topology.
  • Unknowns: missing link: exact adapter architecture size, k-hop value, negative-sampling / evaluation protocol details (in appendices, pages 5+ — ).
Follow-up Papers관련 후속 논문
  • KoPA (Zhang et al., 2024b) — prefix-adapter baseline SAT builds on and compares against; key prior embedding-based LLM-KGC method.
  • GraphGPT (Tang et al., 2024) — graph instruction tuning predecessor for citation networks; SIT generalizes its idea to KGC.
  • MKGL (Guo et al., 2024a) — KG-language token augmentation; alternative structure-into-LLM approach worth contrasting.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = incomplete KG (E, R, T) with entity textual descriptions from Wikipedia; outputs = triple classification (True/False) and link prediction (ranked tail entities); conditions = 4 curated benchmarks (FB15k-237N, CoDeX-S, FB15k-237, YAGO3-10), Llama2-Chat-7B backbone, 2× A800 GPUs, GPT-4 used offline for subgraph-document pair construction. Our setting: dependency extraction / KB construction — inferring missing dependency links in a structured KB, likely a domain-specific graph without clean Wikipedia descriptions, aiming at lightweight-fine-tuning-based link prediction. Differences:

  • Benchmark KGs vs domain dependency KB: missing link — our entities may lack Wikipedia-linked descriptions that HKA relies on (falls back to entity names).
  • GPT-4 dependency for subgraph-document pairs: reference only — extra API cost/pipeline for global alignment.
  • Task set (triple classification + LP) vs our target (dependency link prediction): resolved / LP formulation directly transferable.
  • Compute (2× A800, 7B LLM + graph transformer pretraining): reference only — heavier than a pure lightweight-adapter baseline.
