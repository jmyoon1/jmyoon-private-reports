SP6: LLM-Agent and RAG Pipelines for KG Construction

16 papers · targets O1, O2, O3

SP6 is the cross-cutting SP (O1/O2/O3): does the LLM-agent/RAG-for-KG-construction field give a working end-to-end blueprint — extraction, link-vs-spawn, incremental maintenance — and where does it stop short of a consistent dependency DAG?

SP6: 지식그래프 구성을 위한 LLM 에이전트·RAG 파이프라인

16편 논문 · O1, O2, O3 목표 대상

SP6는 교차 SP(O1/O2/O3)이다. LLM 에이전트/RAG 기반 KG 구성 분야가 추출·링크-대-생성·증분 유지의 작동하는 종단간 청사진을 제공하는지, 그리고 어디서 일관된 의존 DAG에 못 미치는지를 묻는다.

Entry Assumption vs. What Was Found

Expected (revised after SP2, SP3, SP5): This field directly demonstrates end-to-end LLM-agent-driven KG/KB construction pipelines combining extraction, linking, and graph updates, matching the target architecture across O1–O3. (Revised after SP2/SP3/SP5: probe whether prompting matches supervised prerequisite SOTA, whether pipelines OUTPUT typed edges, and note the toolchain provides zero citation-lineage extraction.)

Found: A single dominant spine recurs across every system: (1) prompt an instruction-tuned LLM for open-schema extraction, (2) decide link-vs-spawn by embedding-similarity threshold (optionally LLM-verified), (3) merge incrementally. It is operationalized by a mature toolchain (GraphRAG, LightRAG, Graphiti, KARMA, EDC, iText2KG) and the link-or-spawn threshold converges independently (KARMA ρ, iText2KG θ_E=0.8, Neo4j 0.97+edit-5, EDC verify).

Assessment: Surprised — 'graph updates' universally means incremental append/merge; every tool lacks DAG structural invariants (no cycle avoidance, no transitive-redundant pruning, no stale-link detection), because these are property graphs built for retrieval where cycles are harmless. And extraction is uniformly generic body-text entity/relation — none extracts citation-lineage. O3 is only half-answered here.

사전 가정과 실제 발견

기대 (SP2, SP3, SP5 이후 수정됨): 이 분야가 추출·링킹·그래프 갱신을 결합한 종단간 LLM 에이전트 KG/KB 구성 파이프라인을 직접 시연하여 O1–O3에 걸친 목표 아키텍처와 부합한다고 기대하였다. (SP2/SP3/SP5 이후 수정: prompting이 지도 선행 SOTA에 필적하는지, 파이프라인이 유형 edge를 출력하는지 탐색하고, 툴체인이 인용-계보 추출을 전혀 제공하지 않음을 명시.)

발견: 단일 지배적 척추가 모든 시스템에서 반복된다: (1) instruction-tuned LLM으로 open-schema 추출, (2) 임베딩 유사도 임계값으로 링크-대-생성 결정(선택적 LLM 검증), (3) 증분 병합. 성숙한 툴체인(GraphRAG, LightRAG, Graphiti, KARMA, EDC, iText2KG)이 이를 구현하며 링크-대-생성 임계값이 독립적으로 수렴한다(KARMA ρ, iText2KG θ_E=0.8, Neo4j 0.97+edit-5, EDC 검증).

평가: 예상과 다름 — '그래프 갱신'은 보편적으로 증분 추가/병합을 뜻하며, 모든 도구가 DAG 구조 불변식을 결여한다(cycle 회피 없음, 이행-중복 pruning 없음, stale-link 검출 없음). 이는 cycle이 무해한 검색용 property graph이기 때문이다. 또한 추출은 일률적으로 일반 본문 개체/관계이며 인용-계보를 추출하지 않는다. O3는 여기서 절반만 답해진다.

What SP6 Establishes

The dominant family is open-schema LLM extraction plus post-hoc canonicalization (EDC), which broke the constraint that valid triplets require the schema in-prompt. The load-bearing move is the LLM verifier that lets similarity search reject over-merges. KARMA extends this to a verify-by-decomposition agent making link-or-spawn a first-class step; iText2KG/Graphiti close the loop on incremental append. But every tool ships a property-graph builder, not a DAG builder — the O3 consistency layer must be sourced from SP8/SP9.

SP6이 확립한 것

지배적 계열은 open-schema LLM 추출과 사후 정규화(EDC)이며, 유효 삼중항이 스키마를 프롬프트에 요구한다는 제약을 깼다. 핵심은 유사도 검색이 과병합을 기각하게 하는 LLM 검증기이다. KARMA는 이를 분해-검증 에이전트로 확장해 링크-대-생성을 일급 단계로 만들고, iText2KG/Graphiti가 증분 추가의 고리를 닫는다. 그러나 모든 도구는 DAG 빌더가 아닌 property-graph 빌더를 배포하며, O3 일관성 층은 SP8/SP9에서 와야 한다.

Key Concepts

핵심 개념

Open-schema (schema-free) LLM extraction
What it is: 정의: An instruction-tuned LLM emits free-form (entity, type, description) / (src, dst, relation, description) records with no schema in the prompt; conformance is deferred to canonicalization.instruction-tuned LLM이 프롬프트에 스키마 없이 자유형 (개체, 유형, 설명) / (출발, 도착, 관계, 설명) 레코드를 방출하고, 준수는 정규화로 미룬다.
Why it matters here: 이 조사에서의 역할: The O1 mechanism shared by ≥7 systems; the 'adoptable via prompting, no training' path.7개 이상 시스템이 공유하는 O1 메커니즘이며 'prompting으로 채택 가능, 학습 불필요' 경로이다.
Link-or-spawn by embedding threshold
What it is: 정의: Embed a new mention, find the nearest KB node; link if within threshold, spawn if best distance > ρ; optionally an LLM confirms borderline matches.새 mention을 임베딩해 가장 가까운 KB 노드를 찾고, 임계값 내면 링크, 최선 거리가 ρ 초과면 생성하며, 선택적으로 LLM이 경계 사례를 확인한다.
Why it matters here: 이 조사에서의 역할: The O2 mechanism converged on independently by four systems.네 시스템이 독립적으로 수렴한 O2 메커니즘이다.
Bi-temporal validity window
What it is: 정의: Record event time and ingestion time; a superseded fact is invalidated, not deleted, so the graph answers point-in-time queries.사건 시각과 수집 시각을 기록하고, 대체된 사실은 삭제가 아닌 무효화하여 그래프가 시점 질의에 답하게 한다.
Why it matters here: 이 조사에서의 역할: The closest thing here to stale-link handling (O3) — but at fact-freshness level, not the DAG-invariant level.여기서 stale-link 처리(O3)에 가장 가까우나, DAG 불변식 수준이 아닌 사실-신선도 수준이다.

Research Questions Extracted

추출된 연구 질문

RQ6.1 — What is the standard prompting-adoptable mechanism for deciding link-to-existing-node vs. spawn-new-node? — 기존-노드 링크 대 신규-노드 생성을 결정하는 표준 prompting 채택 메커니즘은 무엇인가?
RQ6.2 — How can an LLM pipeline extract typed entities/relations without an in-prompt schema, and recover conformance afterward? — LLM 파이프라인이 프롬프트 스키마 없이 유형 개체/관계를 추출하고 이후 준수를 회복하는 방법은?
RQ6.3 — What mechanisms let a KB absorb a new document incrementally without full rebuild, and how are superseded facts handled? — KB가 전면 재구축 없이 새 문서를 증분 흡수하는 메커니즘과 대체 사실 처리는?
RQ6.4 — How do agent pipelines mitigate extraction hallucination and schema drift? — 에이전트 파이프라인은 추출 환각과 스키마 표류를 어떻게 완화하는가?
RQ6.5 — What agent-architecture decomposition supports end-to-end construction, and which stages are ablation-justified? — 어떤 에이전트 아키텍처 분해가 종단간 구성을 지원하며 어느 단계가 절제로 정당화되는가?

Open Questions After SP6

  • DAG structural consistency is unaddressed by every system — cycle avoidance, transitive-edge pruning, stale-link detection are all flagged missing (→ SP8/SP9).
  • Citation-lineage / derivation extraction is out of scope for the whole toolchain — extraction is uniformly generic body-text entity/relation (→ SP1/SP5).
  • No gold-standard evaluation of KB-construction accuracy for dependency graphs — correctness is LLM-self-judged or retrieval-proxied.

SP6 이후 미해결 질문

  • DAG 구조 일관성이 모든 시스템에서 다루어지지 않는다 — cycle 회피, 이행-edge pruning, stale-link 검출 모두 미해결로 표시된다(→ SP8/SP9).
  • 인용-계보/파생 추출이 전체 툴체인의 범위 밖이다 — 추출은 일률적으로 일반 본문 개체/관계이다(→ SP1/SP5).
  • 의존 그래프에 대한 KB 구성 정확도의 gold 표준 평가가 없다 — 정확도는 LLM 자체 판정 또는 검색 대리로 측정된다.
