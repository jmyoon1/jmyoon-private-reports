CitationSum: Citation-aware Graph Contrastive Learning for Scientific Paper Summarization

Zheheng Luo et al. · WWW 2023 · 2023 · 19 citations인용 19회

arXiv:2301.11223 · PDF · SP: SP5

P0Track A

Problem
해결하려는 문제

Citation graphs can supply background/method/finding context useful for summarizing a scientific paper, but existing PLM-based summarizers either ignore references entirely or (CGSum) incorporate only reference abstracts indiscriminately. Reference abstracts are shown to be uninformative — low ROUGE overlap with the gold summary — while the truly relevant signal lives in a few key sentences scattered across the reference's full text, at varying correlation strengths. The open problem: how to locate salient reference content and weight each reference by its actual relevance to the source.

참조 초록은 정보량이 낮고 모든 인용 edge가 동등하지 않아 요약에 잡음을 준다.

Solution / Contribution
해결책 / 기여

Self-supervised citation-aware summarization (CitationSum): (1) greedy ROUGE-based content selection extracts salient sentences from each reference's full text; (2) build a weighted citation graph (edge weight = ROUGE similarity, prune below $\rho$) plus a bipartite document–token graph; (3) two graph-contrastive losses — Document Representation Alignment (DRA, Eq. 3) and Token Representation Alignment (TRA, Eq. 4) — align source and reference representations at document and token levels via a normalized graph Laplacian, fusing salient reference info into source representations proportionally to citation correlation; (4) feed citation-aware token representations to a transformer decoder. Total loss $\mathcal{L}=\text{NLL}+\alpha\mathcal{L}_{DRA}+\beta\mathcal{L}_{TRA}$.

ROUGE 상관으로 인용 edge를 가중하고 임계값 ρ=0.7 이하를 pruning하며 graph-contrastive salient-content fusion을 적용하여 구체적 edge-강도 가중과 pruning 규율을 제공한다.

Key Results
주요 결과
  • CitationSum+PubMedBERT beats all baselines on both datasets; PubMedCite inductive R-1 41.62 vs CGSum 40.52 (+1.10), transductive R-1 41.76 vs 39.91 (+1.85); significant at $p<0.05$.
  • On SSN inductive, CitationSum+BERT R-1 44.72 vs CGSum 44.28; +PubMedBERT 45.01.
  • Ablation: removing contrastive learning (W/O Contra) causes the largest drop (PubMedCite inductive R-1 41.62→40.16), confirming the graph contrastive component, not just reference concatenation, drives the gain.
  • $\rho=0.7$ optimal; lower admits noisy references, higher filters useful ones.
Concepts Introduced도입된 개념

Citation-aware graph contrastive learning (CitationSum framework)

  • What it computes: source paper + selected salient sentences of its cited references (nodes) → build a two-level hierarchical heterogeneous graph (weighted citation graph over documents + bipartite document–token graph) → contrastive alignment (DRA loss on document nodes, TRA loss on token nodes) that pulls citation-linked representations together and pushes non-linked ones apart → citation-aware token representations fed to a transformer decoder → abstractive summary.
  • Why it was created: prior citation-graph summarizer CGSum uses only reference abstracts, which the paper shows have low ROUGE overlap with the source (abstracts R-1=0.24 vs full-content R-1=0.67 against gold), so shallow use of references adds noise rather than signal.
  • Concrete example: edge weight between source and reference is the mean of ROUGE-1/ROUGE-2 between source abstract and the reference's selected salient content; edges below threshold $\rho=0.7$ are pruned to avoid noise.
  • How it is used in this paper: it is the core method; the DRA/TRA contrastive losses are added to the standard seq2seq NLL loss as $\mathcal{L} = \text{NLL} + \alpha\mathcal{L}_{DRA} + \beta\mathcal{L}_{TRA}$ ($\alpha=\beta=1$).

Weighted citation graph with salient-content selection

  • What it computes: for each source doc, a greedy oracle ROUGE-maximizing selection extracts top sentences from each reference's full text (using the source introduction at test time, since gold abstract is unseen); edges are weighted by ROUGE similarity between selected contents.
  • Why it was created: to identify which parts of a long reference are actually relevant, since references contain redundant/irrelevant content and different sections (intro/method/exp) have varying similarity to the source.
  • Concrete example: top-7 sentences per reference selected via greedy search; max 16 neighbour references, 100 tokens each.
  • How it is used in this paper: produces the node contents and edge weights the graph contrastive learning operates on.

Graph contrastive learning as implicit matrix factorization

  • What it computes: derivation showing the TRA/DRA losses' optimum reconstructs shifted-log token appearance frequency (IDF-regularized, akin to topic modeling) and the shifted-log weighted citation matrix (akin to a graph auto-encoder).
  • Why it was created: to give a theoretical justification of what the contrastive objective captures.
  • Concrete example: optimum $h_d \cdot h_j = \log n_{d,j} + \log(N_d/n_j) - \log k$.
  • How it is used in this paper: Section 4.3 theory section explaining why the method improves source representations.

5Cs Analysis5Cs 분석

Category: new method + new benchmark dataset (PubMedCite) Context: builds on CGSum (An et al. 2021, the prior citation-graph summarizer using the SSN dataset), BERTSUMABS/EXT (Liu & Lapata 2019), BART, and graph contrastive/topic-modeling lines (GRETEL, VGAE). Directly positions against CGSum. Correctness : significance testing reported ($p<0.05$, † markers), results averaged over 5 runs, ablations and $\rho$ sweep included — appears rigorous from the tables read. Contributions: (1) CitationSum self-supervised citation-aware graph contrastive framework; (2) PubMedCite dataset — 192K biomedical papers, 917K citation edges, first biomedical citation-summarization dataset, keeping reference full contents + weighted correlations; (3) empirical SOTA over CGSum and PLM baselines. Clarity : equations and hyperparameters given; code released, so likely reproducible.

Contribution vs. Claim기여 대 주장

— abstract/intro claim the method "accurately locates and incorporates salient contents from references" and "captures varying relevance." The experiments section (read) supports gains over CGSum and an ablation isolating the contrastive component. Fully confirming the "accurately locates salient content" claim vs. the greedy-ROUGE-oracle mechanism (which uses the source introduction as proxy at test time) warrants a deep pass; the selection is heuristic ROUGE-based, not learned, which is a potential claim/mechanism gap.

Implementation Feasibility구현 실현가능성
  • Needs: PubMedBERT/BERT/BART checkpoints (HuggingFace), PyTorch, multi-GPU 32G memory; PubMed Central Open Access Subset + pubmed_parser to rebuild PubMedCite (contents not directly redistributable due to license; citation graph + build scripts released).
  • Code: available at https://github.com/zhehengluoK/CitationSum
  • Modifications: to reuse for dependency extraction / KB construction, drop the decoder and repurpose the weighted-citation-graph + salient-content-selection components as an edge-weighting and content-linking mechanism; the ROUGE-weighted edges are directly usable as dependency-strength signals.
  • Unknowns : missing link: how well the greedy-ROUGE salient selection transfers when the objective is dependency extraction rather than ROUGE-target summarization.

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = source paper full text + full-text-derived salient sentences of cited references + citation adjacency; outputs = abstractive summary (the paper's abstract as gold target); conditions = biomedical (PubMedCite 192K/917K) and mixed-domain (SSN) scientific corpora, supervised generation with self-supervised graph contrastive auxiliary losses, ROUGE-weighted citation edges.

Our setting (SP5, dependency-extraction / KB-construction from citation graphs): inputs = papers + citation relationships; outputs = extracted inter-paper dependencies / lineage structure for a knowledge base (O1/O2 — modelling a paper's relationship to its cited lineage); conditions = structure extraction, not summary generation.

Differences:

  • Task target: summary generation vs. dependency/relation extraction — reference only (CitationSum is a downstream consumer of citation structure, not a dependency extractor).
  • Edge semantics: ROUGE content-similarity weights on citation edges — resolved: gives a concrete, computable notion of "citation correlation strength" our KB could reuse to weight dependency edges.
  • Salient-content localization: greedy ROUGE selection of the relevant span of a cited paper — resolved: directly relevant as a way to pinpoint what content of a cited work a citing paper actually depends on.
  • Domain: biomedical/scientific — resolved: matches survey's scientific-paper scope.
  • Supervision: needs gold summaries (abstracts) as targets — missing link: our dependency-extraction setting may lack an analogous supervision signal.
