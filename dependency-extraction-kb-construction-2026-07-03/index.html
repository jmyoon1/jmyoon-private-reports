What the literature says

O1 — extraction is machinery, not the edge. Two complementary signal sources exist and are benchmarked: citation-intent classification separates USES/EXTENDS lineage on the ACL-ARC 6-way schema (~68–71 macro-F1, with a scientific-pretrained encoder the single largest lever, +12 F1), and span-based body-text extraction reaches ~67 entity / ~48 relation F1 on SciERC. For an agent, the no-training route is open-schema LLM extraction (prompt for schema-free records, canonicalize afterward). But the dependency edge the KB needs sits one methodized step beyond every benchmark: citation-intent labels are sentence-level and no schema carries a requires type.

O2 — matching is solved; the decision threshold is imported, not calibrated. Description/name-based dense matching (bi-encoder retrieve-then-rerank, or metric-learning cosine) links a mention to a KB node zero-shot, so a newly-added node is linkable without retraining. The reuse-vs-spawn decision, deferred by the academic entity-linking literature, is supplied operationally by the KG-construction toolchain, which converges independently on an embedding-similarity threshold (KARMA ρ, iText2KG θ_E=0.8, Neo4j 0.97). Crucially, the prerequisite direction must be asymmetric — symmetric similarity gives precision 0.98 / recall 0.16.

O3 — append and prune are solved, cycle handling is not. Incremental set-merge against the existing graph and bi-temporal invalidate-not-delete handle append; fully-dynamic transitive reduction prunes redundant edges exactly at O(m) per update. But cycle detection and resolution on edge insertion is answered by no surveyed paper — even gold LectureBank contains 12 cyclic pairs, so the problem is real, and the fix must come from classical graph-algorithm literature outside the surveyed scope.

문헌이 말하는 것

O1 — 추출은 기계일 뿐, edge가 아니다. 상보적 신호 원천 둘이 벤치마크되어 존재한다. 인용 의도 분류는 ACL-ARC 6-way 스키마에서 USES/EXTENDS 계보를 분리하며(~68–71 macro-F1, 과학 사전학습 인코더가 최대 레버로 +12 F1), span 기반 본문 추출은 SciERC에서 ~67 개체 / ~48 관계 F1을 달성한다. 에이전트에게 학습-free 경로는 open-schema LLM 추출이다(스키마 없는 레코드를 프롬프트하고 이후 정규화). 그러나 KB가 필요로 하는 의존 edge는 모든 벤치마크에서 한 단계 너머에 있다: 인용 의도 라벨은 문장 수준이고 어떤 스키마도 requires 유형을 담지 않는다.

O2 — 매칭은 해결됨; 결정 임계값은 보정이 아닌 수입. 설명/이름 기반 밀집 매칭(bi-encoder retrieve-then-rerank 또는 metric-learning cosine)이 mention을 KB 노드에 zero-shot 링킹하므로 신규 노드가 재학습 없이 링킹된다. 학술 개체 링킹 문헌이 미룬 재사용-대-생성 결정은 KG 구성 툴체인이 운용적으로 공급하며, 임베딩 유사도 임계값에 독립 수렴한다(KARMA ρ, iText2KG θ_E=0.8, Neo4j 0.97). 결정적으로 선행 방향은 비대칭이어야 한다 — 대칭 유사도는 정밀도 0.98 / 재현율 0.16이다.

O3 — 추가와 pruning은 해결, cycle 처리는 미해결. 기존 그래프에 대한 증분 set-merge와 bi-temporal 무효화-비삭제가 추가를 처리하고, 완전 동적 이행 축소가 중복 edge를 갱신당 O(m)으로 정확히 제거한다. 그러나 edge 삽입 시 cycle 검출·해소는 어떤 조사 논문도 답하지 않는다 — gold LectureBank조차 12개 순환 쌍을 포함하므로 문제는 실재하며, 해법은 조사 범위 밖 고전 그래프 알고리즘에서 와야 한다.

What we should do

  1. Stage 1 — Extract, two channels. Run open-schema LLM extraction over the body (SciBERT embeddings for Stage 2) and ACL-ARC citation-intent tagging over each citation context; then add the two steps the literature omits — sentence-intent → paper-edge aggregation and an asymmetric prerequisite-typing judgment. Open-schema extraction is the only no-training route with converging multi-tool support.
  2. Stage 2 — Link-or-spawn by threshold. Embed each mention and retrieve the nearest KB node; link within θ, spawn beyond it, gate the borderline band with an LLM equivalence check and the disputed/outKB signal. Start θ conservatively (θ≈0.8) and bias toward precision, then set edge direction with the Stage-1 asymmetric check.
  3. Stage 3 — Maintain the DAG. Set-merge incrementally, invalidate-not-delete stale links, run fully-dynamic transitive reduction to prune redundant edges — then supply the missing piece yourself: a classical incremental topological-order / cycle check on every insertion (reject or route cycle-closing edges). No surveyed paper provides it, so it is an engineering import.

해야 할 것

  1. 1단계 — 두 채널 추출. 본문에 open-schema LLM 추출(2단계용 SciBERT 임베딩)과 각 인용 문맥에 ACL-ARC 인용 의도 태깅을 수행하고, 문헌이 생략한 두 단계 — 문장 의도 → 논문 edge 집계와 비대칭 선행-유형 판단 — 을 추가한다. open-schema 추출은 다중 도구가 수렴 지지하는 유일한 학습-free 경로이다.
  2. 2단계 — 임계값 기반 링크-대-생성. 각 mention을 임베딩해 가장 가까운 KB 노드를 검색하고, θ 이내면 링크, 초과면 생성하며, 경계 구간을 LLM 동등성 검사와 disputed/outKB 신호로 게이팅한다. θ를 보수적으로(θ≈0.8) 시작해 정밀도로 편향하고, 1단계 비대칭 검사로 edge 방향을 설정한다.
  3. 3단계 — DAG 유지. 증분 set-merge, stale 링크 무효화-비삭제, 완전 동적 이행 축소로 중복 edge를 pruning한 뒤, 누락 부분을 스스로 공급한다: 모든 삽입에 고전 증분 위상정렬/cycle 검사(순환을 닫는 edge는 기각 또는 라우팅). 어떤 조사 논문도 제공하지 않으므로 공학적 수입이다.

What the literature does not answer

  • Gap 1 (P1): Cycle detection and resolution on edge insertion — blocks Step 3. Resolves with: a classical incremental topological-ordering algorithm (Pearce–Kelly) measured on a corpus-derived dependency graph.
  • Gap 2 (P1): Reuse-vs-spawn threshold calibration on a terse, canonical-name-poor scientific dependency KB — blocks trusting Step 2 unsupervised. Resolves with: a θ-abstention calibration study with gold merge/spawn labels.
  • Gap 3 (P1): Sentence-intent → paper-level dependency-edge aggregation — blocks Step 1. Resolves with: validating a pooling rule (any-EXTENDS-wins) against gold paper-level edges.
  • Gap 4 (P2): No task emits TYPED dependency edges (requires/derives-from/extends) with P/R evaluation. Resolves with: a dependency-KB benchmark with gold edges and construction-side P/R/F1.
  • Gap 5 (P2): Transfer of all methods to noisy, definition-free, auto-extracted scientific graphs is uniformly untested.

문헌이 답하지 못하는 것

  • Gap 1 (P1): edge 삽입 시 cycle 검출·해소 — 3단계를 차단한다. 해결: 코퍼스 유래 의존 그래프에서 측정한 고전 증분 위상정렬(Pearce–Kelly).
  • Gap 2 (P1): 간결하고 정본 이름이 빈약한 과학 의존 KB에서의 재사용-대-생성 임계값 보정 — 2단계의 비지도 신뢰를 차단한다.
  • Gap 3 (P1): 문장 의도 → 논문 수준 의존-edge 집계 — 1단계를 차단한다.
  • Gap 4 (P2): 어떤 과제도 P/R 평가와 함께 유형 의존 edge(requires/derives-from/extends)를 방출하지 않는다.
  • Gap 5 (P2): 모든 방법의 잡음·정의-부재 자동 추출 과학 그래프로의 전이가 일률적으로 미검증이다.
