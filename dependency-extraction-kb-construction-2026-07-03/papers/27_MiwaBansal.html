End-to-End Relation Extraction using LSTMs on Sequences and Tree Structures

Miwa & Bansal · ACL 2016 · 2016 · 1263 citations인용 1263회

arXiv:1601.00770 · PDF · SP: SP2

P0Track C

Problem
해결하려는 문제

Before this work, extracting entities and their relations was done either as a two-stage pipeline (NER then relation classification), which propagates entity errors and ignores that relations inform entity types, or as feature-based joint structured-prediction models requiring hand-engineered features. Neural models existed for relation classification but (a) used only one linguistic structure — word sequence OR dependency tree, not both — and (b) no NN model performed end-to-end joint entity+relation extraction at all. LSTM-based relation classifiers also underperformed CNN-based ones.

개체·관계를 종단간으로 공동 추출하는 신경망 패러다임이 정립되지 않았다.

Solution / Contribution
해결책 / 기여

A single end-to-end neural model stacking a bidirectional tree-structured LSTM-RNN (dependency layer) on top of a bidirectional sequential LSTM-RNN (sequence layer), with shared embedding/sequence parameters updated by both entity and relation labels.

  • Sequence layer: standard Bi-LSTM over word+POS embeddings → $s_t=[\overrightarrow{h_t};\overleftarrow{h_t}]$.
  • Entity detection: greedy left-to-right BILOU tagging via a 2-layer NN over $[s_t; v^{(e)}_{t-1}]$ (previous-word label embedding), Eqs. 2–3.
  • Dependency layer: new typed-child bidirectional Tree-LSTM (Eq. 4) over the shortest dependency path between a candidate entity pair; relation-candidate vector $d_p=[\uparrow h_{pA};\downarrow h_{p1};\downarrow h_{p2}]$, optionally augmented with averaged entity hidden states (Pair).
  • Relation classification: 2-layer NN + softmax over $d_p$ (Eqs. 5–6), predicting type+direction; both directions predicted, more-confident positive label kept.
  • Training: BPTT + Adam, gradient clipping, L2, dropout, plus entity pretraining and scheduled sampling.

sequence BiLSTM과 dependency-tree LSTM을 쌓은 최초의 신경 joint 개체·관계 추출 모델을 제시하여 이후 과학 IE의 종단간 공동 추출 패러다임을 확립한다.

Key Results
주요 결과
  • End-to-end RE on ACE05: relation F1 0.556 vs. Li & Ji (2014) feature-based 0.495 (+6.1 abs; 12.1% relative error reduction). Entity F1 0.834 vs 0.808.
  • ACE04: relation F1 0.484 vs 0.453 (5.7% relative error reduction). Entity F1 0.818 vs 0.797.
  • SemEval-2010 Task 8 relation classification: Macro-F1 0.844 (no external resources) vs dos Santos CNN 0.841; 0.855 with WordNet — first LSTM model competitive with CNNs on this task.
  • Ablation (ACE05 dev): removing entity pretraining drops relation F1 0.518→0.492 (p<0.05); removing all enhancements →0.484 (p<0.01).
  • Structure finding: FullTree(-SP) hurts significantly; SPTree ≈ SubTree ≈ FullTree when shortest-path nodes are type-distinguished — "selecting the appropriate tree structure (shortest path) is more important than the choice of LSTM-RNN structure."
Concepts Introduced도입된 개념

Stacked sequence + tree LSTM-RNN architecture (SPTree model)

  • What it computes: input = tokenized sentence + POS tags + dependency parse (Stanford neural parser) → a bidirectional sequential LSTM (sequence layer) produces per-word contextual vectors $s_t=[\overrightarrow{h_t};\overleftarrow{h_t}]$, on top of which a bidirectional tree-structured LSTM (dependency layer) over the dependency subtree produces relation-candidate vectors → output = BILOU entity tags (sequence layer) + relation type+direction labels per detected entity pair (dependency layer).
  • Why it was created: prior RNN relation-classification models used only one linguistic structure (either word sequence OR parse tree), and no NN model did end-to-end joint entity+relation extraction — all joint models were feature-based structured-prediction systems.
  • Concrete example: for "In 1909, Sidney Yates was born in Chicago", the sequence layer tags Sidney Yates as B-PER/L-PER; the dependency layer walks the shortest path Yates → born → in → Chicago to classify the PHYS relation.
  • How it is used in this paper: it is the core model; entity detection and relation classification share the embedding + sequence layers, and parameters are jointly updated by BPTT.

Bidirectional tree-structured LSTM-RNN (new typed-child variant)

  • What it computes: at each tree node with children set $C(t)$, gates aggregate child hidden states via type-specific weight matrices $U_{m(l)}$ (Eq. 4), shared across same-type children, allowing a variable number of typed children; run both bottom-up and top-down → node hidden states.
  • Why it was created: Tai et al. (2015) Child-Sum Tree-LSTM ignores edge types and N-ary Tree-LSTM assumes a fixed number of children — neither fits a typed dependency tree with arbitrary arity; standard bottom-up-only trees fail to send root info to near-leaf argument nodes.
  • Concrete example: relation-candidate vector $d_p=[\uparrow h_{pA};\downarrow h_{p1};\downarrow h_{p2}]$ concatenates the lowest-common-ancestor node (bottom-up) with the two target-word nodes (top-down).
  • How it is used in this paper: builds relation representations along the shortest dependency path (SPTree); ablations compare SubTree/FullTree and Child-Sum variants.

Entity pretraining + scheduled sampling (training enhancements)

  • What it computes: entity pretraining first trains the entity-detection submodel on training data before joint training; scheduled sampling replaces predicted labels with gold labels with probability $\epsilon_i=k/(k+\exp(i/k))$ (inverse-sigmoid decay over epoch $i$).
  • Why it was created: in early joint training, unreliable entity predictions mean few/no correct entity pairs are formed, starving relation classification of positive instances.
  • Concrete example: removing entity pretraining drops relation F1 from 0.518 to 0.492 (p<0.05) on ACE05 dev.
  • How it is used in this paper: applied during joint training; ablated in Table 2.

5Cs Analysis5Cs 분석

Category: new method (neural architecture for joint IE) Context: builds on tree-LSTMs (Tai et al. 2015), shortest-path relation classification (Bunescu & Mooney 2005; Xu et al. 2015a/b), and feature-based joint models (Li & Ji 2014; Miwa & Sasaki 2014). Foundational to SP2 (dependency-based extraction) as the paper that established the neural end-to-end joint-extraction paradigm underlying later scientific IE systems. Correctness: statistical significance via Approximate Randomization test; ablations report significance markers (p<0.05, p<0.01). Results on 3 datasets (ACE05, ACE04, SemEval-2010 Task 8). Solid empirical rigor. Contributions: (1) first NN end-to-end joint entity+relation model; (2) new typed-child bidirectional Tree-LSTM; (3) finding that input tree structure choice (shortest path) matters more than LSTM cell type (sequential vs tree). Clarity: highly reproducible — equations, hyperparameter ranges, data splits, and code library (cnn, clab) all given.

Contribution vs. Claim기여 대 주장
  • Stated claims: a single end-to-end neural model jointly represents entities and relations with shared parameters, improving over SOTA feature-based end-to-end RE and matching SOTA CNN relation classification.
  • Demonstrated results: gains shown on ACE05, ACE04 (end-to-end), and SemEval-2010 Task 8 (relation classification only) with significance tests and extensive ablations.
  • Gap: entity detection is still greedy left-to-right (beam search "did not help" per footnote — not deeply explored); relation candidates enumerate all detected-entity pairs (quadratic), scalability to long documents / many entities not evaluated. Evaluated only on English newswire (ACE) and short single-relation sentences (SemEval) — no cross-domain or scientific-text validation.
Implementation Feasibility구현 실현가능성
  • Needs: dependency parser (Stanford neural parser), POS tagger, word2vec embeddings, the cnn (clab) neural library, ACE05/ACE04 (LDC-licensed) or SemEval-2010 Task 8 data; modest single-GPU/CPU compute (100-dim LSTMs).
  • Code: implementation built on https://github.com/clab/cnn ; parser at Stanford CoreNLP. Original model code widely re-implemented (e.g., in later frameworks).
  • Modifications: replace word2vec sequence encoder with a contextual encoder for modern use; retrain on domain-specific relation schema; extend to cross-sentence candidate generation for KB construction.
  • Unknowns: missing link — behavior on scientific text and on parses with high error rates; scalability of all-pairs relation candidate enumeration on entity-dense documents.
Follow-up Papers관련 후속 논문
  • Zhang et al., "Graph Convolution over Pruned Dependency Trees" (2018) — extends dependency-tree relation extraction with GCNs, directly building on the shortest-path idea.
  • Bekoulis et al., "Joint entity recognition and relation extraction as a multi-head selection problem" (2018) — later neural joint-extraction paradigm this paper seeded.
  • Eberts & Ulges, "SpERT: Span-based Joint Entity and Relation Extraction with Transformers" (2020) — transformer successor to the joint end-to-end framing.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = single English sentences with gold or Stanford-parsed dependency trees + POS; outputs = coarse-grained entities (7 ACE types) and relations (6–7 ACE types / 9 SemEval nominal types); conditions = supervised, sentence-level, newswire/general domain, ~350-doc training sets. Our setting (SP2, dependency-based extraction for scientific KB construction): inputs = scientific text (papers/abstracts), likely long multi-sentence spans; outputs = domain-specific entities and dependency/mechanism relations for KB construction; conditions = specialized vocabulary, possibly few labeled examples, cross-sentence relations. Differences:

  • Domain (newswire → scientific text): missing link — no scientific-domain evaluation; dependency parser quality on scientific text unverified.
  • Sentence-level vs document-level: missing link — model builds candidates within one dependency tree; cross-sentence relations not supported.
  • Relation schema (ACE/SemEval fixed types → scientific mechanism/dependency relations): reference only — architecture is schema-agnostic but needs re-labeled training data.
  • Reliance on shortest dependency path: reference only — a transferable inductive bias for SP2, but depends on parser accuracy in-domain.
  • Pre-transformer embeddings (word2vec): resolved/superseded — the sequence layer can be swapped for contextual encoders in modern re-implementations.
