Fully Dynamic Algorithms for Transitive Reduction

Goranci, Karczmarz, Momeni, Parotsidis · ICALP 2025 · 2025

arXiv:2504.18161 · PDF · SP: SP9

P0Track B

Problem
해결하려는 문제

Prior transitive-reduction maintenance (La Poutré & van Leeuwen, 1987) was only partially dynamic: an incremental algorithm ($O(mn)$ total) or a separate decremental one ($O(m^2)$ general, $O(mn)$ DAG). No structure handled interleaved insertions and deletions, so a growing-and-shrinking DAG forced $O(\min(n^\omega, nm))$ amortized recomputation per edge redundancy check.

기존 transitive reduction 유지는 부분 동적일 뿐이라, 성장·축소하는 DAG는 갱신마다 재계산이 필요하였다.

Solution / Contribution
해결책 / 기여

Two families of fully dynamic data structures:

  1. Combinatorial / amortized (Thm 1.1 DAG, Thm 1.2 general):
  • Maintain per-vertex snapshot $G^u$ and decrementally track Desc$^u$/Anc$^u$ via an extended Italiano [Ita88] structure; on a vertex-centered insertion, rebuild exactly one $D_z$ rooted at $z$; aggregate alternative-path counts globally so redundancy is an $O(1)$ counter check.
  • DAG: $O(m)$ amortized update. General digraph: $O(m+n\log n)$ amortized (extra log for maintaining SCCs + parallel inter-SCC edges via an extended Roditty-Zwick [RZ16] structure; intra-SCC handled by recomputing minimal strongly-connected subgraphs [GKRST91]).
  • Redundancy invariant: $xy\in G^t \iff c(xy)=0 \wedge t(xy)=0$ (Invariant 5.1/5.2).
  1. Algebraic / worst-case (Thm 1.3 DAG, Thm 1.4 general), randomized Monte Carlo:
  • Reduce to dynamic matrix inverse of $I-A(G)$ (Sankowski [San04]); for DAGs, path existence $\iff (I-A(G))^{-1}_{u,v}\neq 0$, computed mod a random prime.
  • Vertex updates: $O(n^2)$ worst-case. Edge updates: $O(n^{1.528}+m)$ (DAG) / $O(n^{1.585}+m)$ (general).
  • General-graph parallel inter-SCC redundancy expressed as a polynomial identity on $2k+1$ inverse entries (Thm 6.5), verified via Schwartz-Zippel with a heavy/light-SCC hitting-set argument [UY91].

삽입·삭제를 교차 처리하는 최초의 fully-dynamic transitive reduction을 제시하여 DAG에서 갱신당 O(m) amortized로 정확한 도달가능성을 보존하며, a→b→c가 있으면 a→c를 제거하는 중복 edge pruning에 정확히 대응한다.

Key Results
주요 결과

  • DAG, combinatorial: $O(m)$ amortized update time (Thm 1.1) — vs. naive $O(\min(n^\omega,nm))$.
  • General digraph, combinatorial: $O(m+n\log n)$ amortized (Thm 1.2); near-optimal for sparse graphs up to a $\log n$ factor.
  • DAG, algebraic: $O(n^2)$ worst-case per vertex update, $O(n^{1.528}+m)$ per edge update (Thm 1.3).
  • General, algebraic: $O(n^2)$ vertex / $O(n^{1.585}+m)$ edge worst-case (Thm 1.4).
  • All structures use $O(n^2)$ space; initialize in $O(n^\omega)$.
Concepts Introduced도입된 개념

Transitive reduction $G^t$

  • What it computes: input = directed graph $G=(V,E)$ → operation = remove all redundant edges (edge $xy$ is redundant if a directed path $x\to y$ still exists in $G\setminus xy$) → output = minimal subgraph $G^t$ preserving the exact reachability relation between every vertex pair.
  • Why it was created: a "sparsifier for transitive closure" (Aho-Garey-Ullman 1972) — store/serve reachability with fewest edges. In some domains (biological network reconstruction, code race recording) removing redundant edges is the goal itself, not size reduction.
  • Concrete example: DAG with edges $a\to b$, $b\to c$, $a\to c$. The edge $a\to c$ is redundant (reachable via $a\to b\to c$), so $G^t = \{a\to b, b\to c\}$.
  • How it is used in this paper: the object being maintained under edge insertions/deletions. For DAGs $G^t$ is unique; for general digraphs it is defined as an inclusion-wise minimal subgraph (minimum-size version is NP-hard because a Hamiltonian cycle iff $G^t$ is a single cycle).

Fully dynamic setting

  • What it computes: maintain $G^t$ under a sequence of updates (both edge insertions AND deletions), reporting the reduction after each update without full recomputation.
  • Why: naive recomputation costs $O(\min(n^\omega, nm))$ amortized per update — prohibitive on a growing graph.
  • Concrete example: extended ops supported — insert a set of edges all incident to one vertex ("centered" insertion), or delete an arbitrary edge set, in one update.
  • How it is used: the paper delivers the first fully dynamic data structures (prior work PL87 was only partially dynamic — incremental OR decremental).

Indirect-paths / snapshot data structure ($G^u$, Desc$^u$, Anc$^u$)

  • What it computes: for each vertex $u$, keeps a decremental snapshot $G^u$ (graph as of the last insertion centered at $u$) plus Desc$^u$ (vertices reachable from $u$) and Anc$^u$ (vertices reaching $u$); a counter $c(xy)$ and flag $t(xy)$ per edge decide redundancy in $O(1)$.
  • Why it was created: to reach $O(m)$ amortized time — naive per-vertex reachability maintenance is $O(mn)$.
  • Concrete example: Invariant — edge $xy\in G^t$ iff $c(xy)=0$ and $t(xy)=0$.
  • How it is used: core of the combinatorial algorithm (Thm 1.1/1.2); built on Italiano's [Ita88] decremental single-source reachability.
5Cs Analysis5Cs 분석

Category: theoretical / new method (dynamic graph algorithms, data structures). Context: builds on transitive-reduction (AGU72), dynamic transitive closure (Sankowski San04, Roditty-Zwick RZ08/RZ16, King-Sagert KS02), decremental reachability (Italiano Ita88, Bernstein-Gutenberg-Wulff-Nilsen BGW19), dynamic matrix inverse (BNS19). Continues the 30-year-old PL87 line. Correctness: proofs sketched in overview; combinatorial results deterministic, algebraic results Monte Carlo (high-probability, correct w.p. $1-1/\text{poly}(n)$). Near-optimality argued via conditional lower bounds (a poly-faster sparse algorithm would beat the best static $O(n^2)$). No experiments — pure theory. Contributions: (1) first fully dynamic transitive-reduction structures for DAGs and general digraphs; (2) near-linear $O(m+n\log n)$ amortized bound, near-optimal for sparse graphs; (3) sub-quadratic worst-case bounds via fast matrix multiplication. Clarity: technical overview + full pseudocode (Algorithm 1) for the decremental DAG core; reproducible in principle for a theory audience, though the general-graph algebraic part is intricate.

Contribution vs. Claim기여 대 주장

This is a theory paper: "results" are proved complexity bounds, not measured numbers. Stated claim = first fully dynamic transitive reduction with near-linear amortized / sub-quadratic worst-case update time. No empirical validation or implementation is presented (no runtime experiments, no benchmark graphs). Gap: practical constant factors, the $O(n^2)$ space cost, and real-world performance on incrementally-built KGs are unaddressed — the utility for our use case rests on the asymptotics alone.

Implementation Feasibility구현 실현가능성
  • Needs: implement the decremental single-source reachability of Italiano [Ita88] (Algorithm 1 fully specified in Appendix A) + the snapshot/counter bookkeeping for the DAG case ($O(m)$ amortized). General-graph case additionally needs the extended Roditty-Zwick [RZ16] SCC-maintenance structure (Appendix B) — substantially more work. Algebraic variants need a dynamic matrix inverse library and finite-field arithmetic — likely overkill for our setting.
  • Code: [not available] — no public repository referenced; only pseudocode for the DAG decremental core.
  • Modifications: for an insertion-dominated KB, the DAG combinatorial structure (Thm 1.1) is the right target; could potentially simplify since deletions are rare. Need an adjacency-list + timestamp scheme (§5.1.3) to hold space at $O(n^2)$.
  • Unknowns: missing link: whether our extracted dependency graph is guaranteed acyclic (determines DAG vs. general algorithm); missing link: constant factors and $O(n^2)$ space at target KB scale; missing link: no empirical baseline to compare against.
Follow-up Papers관련 후속 논문
  • Roditty & Zwick, "A Fully Dynamic Reachability Algorithm ... Almost Linear Update Time" (SICOMP 2016) — the SCC-maintenance backbone reused for the general-graph result; relevant if we need dynamic reachability beyond DAGs.
  • Italiano, "Finding paths and deleting edges in directed acyclic graphs" (IPL 1988) — the decremental DAG reachability primitive to implement first.
  • La Poutré & van Leeuwen, "Maintenance of Transitive Closures and Transitive Reductions of Graphs" (WG 1987) — the partially-dynamic predecessor; closest match to an incremental-only KB workload.
  • Aho, Garey & Ullman, "The Transitive Reduction of a Directed Graph" (SICOMP 1972) — foundational definition; the static algorithm to benchmark against.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = a general directed graph (or DAG) undergoing single-edge, vertex-centered, or arbitrary-edge-set insertions/deletions; output = the maintained transitive reduction $G^t$ (explicit edge set); conditions = adversarial update sequences, $O(n^2)$ space, exact reachability preservation, unweighted loop-less graphs.

Our setting (SP9, dependency-extraction KB construction): inputs = a growing dependency graph incrementally assembled from extracted entity/concept dependencies, expected to be (mostly) acyclic; output = pruned KB with transitively-redundant dependency edges removed; conditions = predominantly incremental (insertion-heavy) growth, moderate scale, correctness of pruning matters more than worst-case latency.

Differences:

  • Update mix (insert+delete adversarial vs. mostly incremental growth): reference only — our workload is closer to the incremental PL87 regime; the fully dynamic machinery is more general than we strictly need but subsumes it.
  • Graph class (general digraph with SCCs vs. our likely-DAG KB): resolved: use the DAG variant (Thm 1.1, $O(m)$) — if our dependency graph is acyclic the simpler, faster, deterministic DAG data structure applies directly.
  • Cycles in extracted dependencies (if any): missing link — need to decide whether to contract SCCs (paper's minimal-subgraph definition) or treat cycles as errors; the paper's general algorithm handles this but adds the $n\log n$ term.
  • Exact vs. approximate reachability: resolved: exact — paper preserves reachability exactly, matching KB-pruning correctness needs.
  • Space $O(n^2)$: missing link — may be a concern for large KBs; paper argues quadratic space is acceptable when redundant-edge elimination is the goal, but our scale must be checked.
  • No implementation available: missing link — we would need to implement from the paper (Algorithm 1 given for the DAG core only).
