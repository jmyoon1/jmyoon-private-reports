Multi-Task Identification of Entities, Relations, and Coreference for Scientific Knowledge Graph Construction

Luan et al. · EMNLP 2018 · 2018 · 785 citations인용 785회

arXiv:1808.09602 · PDF · SP: SP2

P0Track A

Problem
해결하려는 문제

Scientific information extraction had been done task-by-task (entity/keyphrase extraction, relation extraction, coreference each separately), producing cascading errors and ignoring cross-sentence links, and there was no single dataset annotating all three tasks on the same scientific text to enable joint modeling.

과학 지식그래프를 구성하려면 개체·관계·상호참조를 함께 추출해야 하나 표준 벤치마크가 부재하였다.

Solution / Contribution
해결책 / 기여

Empirical/system contribution: (1) release SciERC, 500 abstracts annotated with entities, relations, and coreference; (2) propose SciIE, a unified span-based multi-task model with shared span representations (biLSTM + ELMo) and three output heads that jointly predicts all three, reducing cascading errors and using coreference to surface cross-sentence relations.

500개 초록에 개체/관계/coref를 주석한 SciERC 데이터셋과 span 기반 다중과제 모델(SciIE)을 도입하여 과학 개념·관계 추출의 정본 벤치마크와 관계 스키마를 확립한다.

Key Results
주요 결과
  • Multi-task SciIE outperforms single-task baselines on entity, relation, and coreference without domain-specific features.
  • Entity recognition F1 ≈ 64.2, relation extraction F1 ≈ 39.3, coreference avg F1 ≈ 48.
  • Multi-task setup reduces cascading errors and leverages coreference for cross-sentence relations.
Concepts Introduced도입된 개념

SciERC dataset

  • What it computes: not a computation — a labeled corpus. Input = 500 scientific abstracts (from 12 AI conference/workshop proceedings in the Semantic Scholar corpus); output = span-level annotations for scientific entities (6 types), directed relations (7 types), and coreference clusters.
  • Why it was created: prior scientific-IE corpora (e.g., SemEval keyphrase tasks) annotated entities/keyphrases but lacked a unified corpus covering entities, relations, AND coreference on the same text, blocking joint multi-task training.
  • Concrete example: entity types include Task, Method, Metric, Material, OtherScientificTerm, Generic; relation types include Used-for, Feature-of, Hyponym-of, Part-of, Compare, Conjunction, Evaluate-for.
  • How it is used in this paper: the benchmark on which the proposed SciIE model is trained and evaluated for all three tasks.

SciIE (Scientific Information Extractor) — shared span representations

  • What it computes: input = tokens of an abstract → enumerate candidate spans → encode each span with a bidirectional LSTM + ELMo contextual embeddings into a shared span representation → three task-specific output heads (entity classification, relation classification over span pairs, coreference antecedent scoring) → output = entities + relations + coref clusters.
  • Why it was created: pipelined IE propagates cascading errors between entity, relation, and coref stages; a single shared span encoder lets the three tasks regularize each other and exploit cross-sentence signal via coreference.
  • Concrete example: a span "convolutional neural network" gets one representation reused by the entity head (label = Method) and the relation head (Used-for → "image classification").
  • How it is used in this paper: the core model; multi-task variant is compared against single-task ablations and used to build a scientific knowledge graph.

5Cs Analysis5Cs 분석

Category: benchmark + new method (system) Context: builds on span-based coreference (Lee et al. 2017) and ELMo (Peters et al. 2018); foundational for SP2 (scientific concept and relation extraction) and the reference benchmark cited by later scientific-IE / LLM-IE papers. Correctness: Contributions: SciERC dataset; SciIE multi-task span model; demonstration of scientific KG construction from extracted triples. Clarity: — code/data publicly released (http://nlp.cs.washington.edu/sciIE/), which aids reproduction.

Contribution vs. Claim기여 대 주장

— abstract claims the multi-task setup "reduces cascading errors" and "leverages cross-sentence relations through coreference"; whether the ablations isolate each of these effects (vs. a single aggregate F1 gain) requires the experiments section.

Implementation Feasibility구현 실현가능성
  • Needs: SciERC data (released), ELMo embeddings, biLSTM training compute (modest, ~1 GPU).
  • Code: available at http://nlp.cs.washington.edu/sciIE/
  • Modifications: to serve our KB/dependency task, the relation label set must be remapped to prerequisite/dependency relations, and the encoder swapped for an LLM-prompting or fine-tuning approach; SciERC most useful as an evaluation benchmark rather than as the model.
  • Unknowns: missing link — exact F1 numbers, ablation design, and annotation agreement pending deep read.
Follow-up Papers관련 후속 논문
  • DyGIE / DyGIE++ (Wadden et al., 2019) — extends SciIE's span-graph approach with span-graph propagation; standard SOTA follow-up on SciERC.
  • PL-Marker / packed levitated markers (Ye et al., 2022) — later SciERC entity/relation SOTA relevant to SP2 method comparison.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = single scientific abstracts (short, AI-domain); outputs = typed entity spans + 7 directed relation types + coreference clusters; conditions = supervised training on 500 hand-annotated abstracts, biLSTM+ELMo (pre-transformer era), no LLM. Our setting: inputs = full paper body text for dependency/prerequisite-concept extraction toward KB construction; outputs = concepts and their (prerequisite/uses/extends) relations; conditions = prompting/retrieval or lightweight fine-tuning of LLMs, not from-scratch training. Differences:

  • Relation taxonomy: SciERC's 7 relations (Used-for, Part-of, etc.) overlap but do not equal prerequisite/dependency relations — reference only (label schema is a candidate starting taxonomy).
  • Text scope: abstracts only vs. full body text — missing link (SciIE tuned for short spans within an abstract; scaling to body text and cross-section coref untested here).
  • Method era: biLSTM+ELMo supervised model vs. LLM-prompting adaptation — reference only (benchmark still used to evaluate modern LLM-IE, but the model itself is not the adaptable component).
  • Domain: AI-conference abstracts — resolved: matches our scientific-literature target domain.
