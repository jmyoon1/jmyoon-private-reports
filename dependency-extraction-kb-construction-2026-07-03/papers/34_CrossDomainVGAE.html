Unsupervised Cross-Domain Prerequisite Chain Learning using Variational Graph Autoencoders

Irene Li et al. · ACL 2021 · 2021 · 21 citations인용 21회

arXiv:2105.03505 · PDF · SP: SP3

P0Track A

Problem
해결하려는 문제

Existing prerequisite-relation methods work within a single domain and are supervised or semi-supervised, requiring annotated prerequisite pairs in the domain of interest. When entering an unfamiliar ("information-poor") domain, no target-domain labels exist, so classifier- and single-graph-based approaches cannot infer the study order of new concepts. Pairwise classifiers trained on the source domain transfer poorly because they treat concept pairs independently and ignore graph structure.

라벨이 없는 새 도메인에서 선행관계를 예측하기 어렵다.

Solution / Contribution
해결책 / 기여

Cross-domain variational graph autoencoder (CD-VGAE): build one heterogeneous concept-resource graph spanning source+target domains, encode with a GCN augmented by a domain-neighbor aggregation term (top-10% cosine-similar source→target concept matches, separate weight W_D), and decode target prerequisite links with a DistMult bilinear scorer Â = X̂ R X̂ᵀ. Source concept relations A_{c,s} are observed; the model predicts unobserved target relations A_{c,t} in a fully unsupervised (no target labels) way.

domain-neighbor 인코더와 DistMult decoder를 결합한 cross-domain VGAE와 LectureBankCD 벤치마크를 도입하여 target 라벨 없이 NLP→CV 전이 F1 0.675를 달성한다.

Key Results
주요 결과
  • NLP→CV: CD-VGAE+P2V F1 0.6754 (best baseline VGAE+P2V 0.6202, +5.5 F1); CD-VGAE+BERT F1 0.6391 (vs VGAE+BERT 0.5885).
  • NLP→BIO: CD-VGAE+P2V F1 0.6512 (best baseline VGAE+P2V 0.6177).
  • Much higher recall: NLP→CV recall 0.8837 (VGAE+P2V 0.7349); NLP→BIO recall 0.6667 (VGAE 0.6091) — model predicts more positive edges (CV: predicts 1,209 vs 871 ground truth; BIO: 303 vs 234), favoring recall for study-path completeness.
  • Supervised in-domain upper bound (CLS+Node2vec) NLP-CV F1 0.8172 — a substantial gap remains vs cross-domain.
  • CLS pairwise classifiers give below-random F1 (0.42/0.39), confirming graph structure matters.
Concepts Introduced도입된 개념

Cross-Domain Concept-Resource Graph G = (X, A)

  • What it computes: input = concept nodes + resource (lecture slide) nodes from a source (NLP) and a target (CV/BIO) domain → build one heterogeneous graph with node feature matrix X (BERT or Phrase2Vec embeddings) and adjacency A composed of four edge types (A_{c,s} source-concept edges, A_{rc} resource–concept edges, A_r resource–resource edges, A_{c,t} target-concept edges) → single graph on which link prediction runs. A_{rc} and A_r are set by cosine similarity of node embeddings.
  • Why it was created: prior prerequisite-relation work operated within a single domain; there was no structure that lets known source-domain relations inform an unlabeled target domain in one shared graph.
  • Concrete example: NLP (source, 322 concepts, 1,551 positive relations) + CV (target, 201 concepts) modeled together; A_{c,s} is known, A_{c,t} is the prediction target.
  • How it is used in this paper: the substrate for the whole method — the encoder runs message passing over this graph and the decoder recovers A_{c,t}.

Cross-Domain Graph Encoder (domain neighbors)

  • What it computes: input = node hidden states → standard GCN aggregation over direct neighbors N_i PLUS an extra aggregation over a set of "domain neighbours" N_i^D with a separate weight matrix W_D: h_i^{(l+1)} = σ( Σ_{j∈N_i} W h_j + W h_i + Σ_{k∈N_i^D} W_D h_k ) → cross-domain-aware node embedding.
  • Why it was created: a vanilla GCN only sees within-graph neighbors, so source-domain structure cannot flow to target concepts that have no observed edges.
  • Concrete example: domain neighbors are found by cosine similarity between source and target concept embeddings, normalized to [0,1], keeping the top 10% (e.g., NLP "Text Classification" ↔ CV "Image Classification").
  • How it is used in this paper: the core architectural novelty enabling unsupervised transfer; claimed first to do domain transfer within a single graph.

DistMult Decoder (replacing VGAE inner product)

  • What it computes: input = last-layer node features X̂ → bilinear score Â = X̂ R X̂ᵀ with trainable diagonal relation matrix R, then Sigmoid → positive/negative prerequisite link prediction.
  • Why it was created: the original VGAE inner-product decoder is symmetric and less expressive for directed prerequisite relations.
  • Concrete example: predicts Convolutional Neural Network → Image Classification.
  • How it is used in this paper: the decoder half of the CD-VGAE, reconstructing the target adjacency.

LectureBankCD (benchmark)

  • What it computes: extends LectureBank2.0 (NLP) by adding two annotated target domains — CV (201 concepts, 871 positive relations, 1,041 slide files) and BIO (100 concepts, 234 positive relations, 148 files) — each with pairwise prerequisite annotations by two graduate annotators (Cohen's κ = 0.6396 CV, 0.8038 BIO).
  • Why it was created: no cross-domain prerequisite benchmark existed for transfer evaluation.
  • Concrete example: NLP→CV and NLP→BIO transfer tasks.
  • How it is used in this paper: the evaluation corpus; a key transfer-evaluation reference for SP3.

5Cs Analysis5Cs 분석

Category: new method + benchmark (CD-VGAE model and LectureBankCD dataset). Context: builds directly on VGAE (Kipf & Welling 2016), GCN (Kipf & Welling 2017), Li et al. 2019 (VGAE for prerequisites) and Li et al. 2020 (RelationalVGAE, unsupervised); DistMult (Yang et al. 2015). Central SP3 reference for cross-domain / transfer. Correctness: reports averages over 5 seeded splits; annotation agreement reported (substantial κ). no confidence intervals visible in the 5-page read; imbalance handled by negative sampling to 1:1. Contributions: (1) first single-graph cross-domain prerequisite transfer via domain-neighbor encoder; (2) LectureBankCD benchmark with CV and BIO domains. Clarity: encoder equation, edge types, and decoder are given explicitly; largely reproducible.

Contribution vs. Claim기여 대 주장

Abstract claims "substantially surpassing other baseline models" and first-of-kind single-graph domain transfer. The 5-page read confirms F1/accuracy/recall gains on both target domains and case studies, but the margin over VGAE+P2V is modest on F1 (~+3–5 points) and driven largely by recall (more positive predictions); precision is comparable or lower. Whether "substantial" holds beyond recall needs the full experiments/precision table. Generality claimed via two target domains (CV, BIO) only.

Implementation Feasibility구현 실현가능성
  • Needs: LectureBankCD data (public), BERT + Phrase2Vec embeddings, a VGAE/GCN implementation with a custom domain-neighbor aggregation and DistMult decoder, modest compute (small graphs).
  • Code: available at https://github.com/Yale-LILY/LectureBank/tree/master/LectureBankCD
  • Modifications: for general KB construction, replace curated concept lists with an entity/concept extraction front-end; define resource nodes or drop the resource layer; validate domain-neighbor top-10% heuristic on larger/sparser graphs.
  • Unknowns: missing link: full training objective/KL weighting and hyperparameters not captured in the 5-page skim; scalability and behavior when domains share few concepts.
Follow-up Papers관련 후속 논문

  • Later prerequisite / concept-graph transfer work citing LectureBankCD — relevant to SP3 cross-domain evaluation lineage.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = concept nodes + lecture-slide resource nodes with BERT/P2V embeddings, source-domain prerequisite edges known; outputs = predicted target-domain concept prerequisite links; conditions = education domain, ~100–322 concepts per domain, small graphs, unsupervised on target. Our setting (SP3, dependency extraction for KB construction): inputs = concepts/entities from a corpus, possibly cross-domain; outputs = dependency/prerequisite edges for a knowledge base; conditions = potentially larger, noisier, less curated concept sets, transfer across heterogeneous domains. Differences:

  • Curated, expert-annotated concept lists vs open KB entity extraction: missing link (our concepts may need extraction, not given).
  • Small dense per-domain graphs vs large sparse KB: reference only (scalability of domain-neighbor cosine matching unverified at scale).
  • Requires shared/overlapping concepts across domains for domain neighbors: resolved-conditionally — works when source/target share concepts; missing link if domains are disjoint.
  • Resource (slide) nodes provide A_{rc}/A_r structure: missing link (our setting may lack a comparable resource layer).
