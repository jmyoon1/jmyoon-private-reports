Entity Set Search of Scientific Literature: An Unsupervised Ranking Approach

Shen et al. · SIGIR 2018 · 2018

arXiv:1804.10877 · PDF · SP: SP4

P0Track A

Problem
해결하려는 문제

Scientific-literature search is dominated by entity-set queries (multiple entities of different types) that require modeling inter-entity relations, yet (a) existing rankers score each entity independently and thus rank single-entity-repetition documents too high, and (b) these queries are sparse with little click history, so supervised learning-to-rank models fail. On Semantic Scholar's production system, entity-set queries score ~0.36 NDCG@5 vs. ~0.63 for non-entity-set queries.

라벨 없이 과학 문헌에서 개체 집합을 검색·순위화할 필요가 있다.

Solution / Contribution
해결책 / 기여

SetRank: an unsupervised, entity-aware ranking framework.

  • Document side: each field represented as bag-of-words + bag-of-entities, each fitted with a smoothed (Dirichlet prior) language model; field-weighted token generation $p(t\mid d_i)=\sum_{j=1}^{k} p(t\mid d_{i,j})\,p(d_{i,j}\mid d_i)$.
  • Query side: a heterogeneous graph with word and entity nodes; entity-entity edges weighted by type distance (LCA in a type hierarchy).
  • Matching: query-document relevance defined as a graph covering process — a document is scored by how much of the query graph's information need it covers.
  • Plus an unsupervised model selection algorithm (weighted rank aggregation) that picks parameter settings without any labeled validation data.

비지도 개체 인식 순위화와 라벨 없는 모델 선택을 제안하여 라벨-free 관련도 신호와 rank-aggregation 파라미터 선택을 제공한다.

주요 결과

Concepts Introduced도입된 개념

Entity-set query

  • What it computes: a query consisting of multiple entities of possibly different types (e.g., "GABP TERT CD11b cancer" = 3 genes + 1 disease) → the search must return documents relevant to the whole set and reflecting inter-entity relations, not each entity independently.
  • Why it was created: existing rankers model each entity separately, so they over-rank documents where a single entity repeats and under-rank documents covering many distinct entities.
  • Concrete example: on Semantic Scholar's 100-query benchmark, 40 are entity-set queries; the production system scores NDCG@5 = 0.3622 on them vs. 0.6291 on non-ESQs.
  • How it is used in this paper: the target query class SetRank is designed to improve.

SetRank (unsupervised ranking framework)

  • What it computes: document (bag-of-words + bag-of-entities language models per field) × query (heterogeneous graph of word/entity nodes) → relevance score via a "graph covering" matching process. No labeled training data.
  • Why it was created: entity-set queries are sparse (little click history), so supervised learning-to-rank models cannot be trained effectively; prior unsupervised models ignore inter-entity relations and type.
  • Concrete example: a paper doc field is represented with both a smoothed word LM and a smoothed entity LM (Dirichlet prior, Eq. 2); the query "reinforcement learning Atari" becomes a graph with entity-entity edges weighted by type distance.
  • How it is used in this paper: the core proposed ranker (Section 3).

Bag-of-entities representation + entity language model

  • What it computes: an entity-linking tool maps entity mentions in text to disambiguated KB IDs (Freebase/Wikidata) → a per-field bag-of-entities → a smoothed entity unigram LM alongside the word LM.
  • Why it was created: word-only representations miss polysemy/synonymy resolution (e.g., "USA"/"United States"/"U.S." → one entity; fruit vs. company "Apple" separated).
  • Concrete example: title "Training linear SVMs in linear time" → mention "SVMs" linked to entity "Support Vector Machine" (/m/0hc2f).
  • How it is used in this paper: document-side representation (Section 3.1); reuses bag-of-entities from Xiong et al. [37,38] but combines it with words in an unsupervised way.

Heterogeneous query graph

  • What it computes: query tokens → nodes (word-type and entity-type); edges encode word-word (adjacent, weight 1) and entity-entity relations weighted by type distance via Lowest Common Ancestor (LCA) in a type hierarchy.
  • Why it was created: to explicitly model entity relations and entity type within the set, which flat query representations cannot.
  • Concrete example: entities "reinforcement learning" (education.field_of_study) and "Atari" (computer.game) have LCA = Thing; larger type distance → larger relation weight (Eq. 3).
  • How it is used in this paper: query-side representation (Section 3.2); query-document match = covering this graph.

Unsupervised model selection via weighted rank aggregation

  • What it computes: a set of unlabeled queries + a set of candidate parameter settings → automatically selects the best parameter setting by aggregating the rankings each setting produces and using the aggregate to score settings — no labeled validation set.
  • Why it was created: even "unsupervised" rankers have parameters (field weights, smoothing) usually tuned on a labeled validation set, which is expensive in academic search.
  • Concrete example: candidate settings are ranked against a consensus aggregate ranking derived from weighted rank aggregation.
  • How it is used in this paper: Section 4; a second, independent contribution beyond SetRank.

5Cs Analysis5Cs 분석

Category: new method (unsupervised entity-aware ranking + unsupervised model selection) Context: builds on bag-of-entities / word-entity duet representations (Xiong et al. [37,38]) and entity-aware ranking (ESR [39]); departs by being fully unsupervised and by modeling entity relations + type. Correctness: — abstract claims significant gains; exact CIs/significance tests not yet read. Contributions: (1) formalizes entity-set search of scientific literature; (2) SetRank unsupervised ranker modeling inter-entity relations + type; (3) unsupervised model-selection algorithm via weighted rank aggregation. Clarity: method equations (LM smoothing Eq. 2, type-distance Eq. 3) and graph construction are given; likely reproducible.

Contribution vs. Claim기여 대 주장

— distinguishing stated claims (generalizable unsupervised gains + effective parameter selection) from demonstrated results (two biomedical/CS benchmarks) requires reading Section 5. Note potential scope limit: evaluation is on TREC Genomics (biomedical) + Semantic Scholar; generality across broader domains is asserted but bounded by these two corpora.

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = entity-set queries + a document corpus (scientific papers with title/abstract fields) + an external KB for entity linking; outputs = a ranked document list; conditions = unsupervised (no labeled relevance / no click training), evaluated on TREC Genomics + Semantic Scholar. Our setting (SP4, dependency-extraction / KB construction): inputs = extracted scientific entities to be retrieved/matched against a corpus or KB; outputs = matched/ranked entity-document or entity-KB associations; conditions = unsupervised distributional similarity, no labeled pairs. Differences:

  • Task framing (document retrieval vs. entity-to-KB matching): reference only — SetRank ranks documents for a query set, whereas SP4 may need entity-to-entity/entity-to-record matching; the graph-covering + entity-relatedness modeling transfers as a scoring pattern.
  • Unsupervised inter-entity relatedness via co-occurrence + type distance: resolved / directly reusable as a labeled-data-free relatedness signal for KB construction.
  • Entity linking dependency: missing link — requires an off-the-shelf entity linker + a KB with a type hierarchy (Freebase/Wikidata); availability/quality in SP4's target domain unverified.
  • Model selection without labels: resolved / the weighted rank aggregation trick is domain-agnostic and useful when no validation labels exist.
