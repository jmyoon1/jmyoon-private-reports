Learning Hierarchy-Aware Knowledge Graph Embeddings for Link Prediction

Zhang et al. · AAAI 2020 · 2019

arXiv:1911.09419 · PDF · SP: SP9

P0Track A

Problem
해결하려는 문제

Existing KG embedding models focus on relation connectivity patterns (symmetry/antisymmetry, inversion, composition) but fail to model semantic hierarchies — the fact that entities occupy different levels of a tree-like structure (e.g., hypernymy in WordNet, containment in Freebase). Prior hierarchy-aware work (TKRL, clustering-based methods) requires additional type/hierarchy annotations rather than learning hierarchy automatically from triples.

고전 KGE(TransE, DistMult)는 개체가 트리 어디에 위치하는지 계층 깊이를 표현하지 못한다.

Solution / Contribution
해결책 / 기여

HAKE maps entities into a polar coordinate system, splitting each embedding into a modulus (radial) part and a phase (angular) part:

  • Modulus part: $\mathbf{h}_m \circ \mathbf{r}_m = \mathbf{t}_m$, $\;\mathbf{r}_m \in \mathbb{R}^k_+$, distance $d_{r,m}=\|\mathbf{h}_m\circ\mathbf{r}_m-\mathbf{t}_m\|_2$ (models depth / different levels).
  • Phase part: $(\mathbf{h}_p+\mathbf{r}_p)\bmod 2\pi = \mathbf{t}_p$, distance $d_{r,p}=\|\sin((\mathbf{h}_p+\mathbf{r}_p-\mathbf{t}_p)/2)\|_1$ (distinguishes same-level entities).
  • Combined score: $f_r(h,t) = -\big(\lambda_1 d_{r,m} + \lambda_2 d_{r,p}\big)$, trained with self-adversarial negative-sampling loss (from RotatE).
  • Concentric circles in polar space naturally reflect hierarchy: smaller radius = higher level; different angle = same level, different entity.

개체를 modulus(반경=깊이)와 phase(각도=동일 레벨)로 나누는 극좌표 임베딩 HAKE를 제안하며, modulus 단독 절제(ModE)가 이미 DistMult/ConvE/ComplEx를 능가하여 반경 깊이 신호가 계층을 담음을 보인다.

Key Results
주요 결과
  • WN18RR MRR: HAKE .497 vs RotatE .476 (+.021), H@1 .452 vs .428 (+2.4%), H@10 .582 vs .571.
  • YAGO3-10: HAKE gains +.050 MRR, +6.0% H@1, +4.6% H@3 over RotatE (per intro/analysis text).
  • ModE (modulus-only baseline) already beats DistMult, ConvE, ComplEx on all datasets and beats RotatE on FB15k-237 and YAGO3-10 — shows modulus information alone is powerful.
  • FB15k-237: smaller gains, attributed to many non-hierarchical relation types in that dataset.
Concepts Introduced도입된 개념

HAKE (Hierarchy-Aware Knowledge Graph Embedding)

  • What it computes: input = triple $(h, r, t)$ → maps each entity to a polar-coordinate vector $[e_m; e_p]$ (modulus part $e_m \in \mathbb{R}^k$ + phase part $e_p \in [0,2\pi)^k$); relation applies element-wise modulus scaling $\mathbf{h}_m \circ \mathbf{r}_m = \mathbf{t}_m$ and phase shift $(\mathbf{h}_p + \mathbf{r}_p)\bmod 2\pi = \mathbf{t}_p$ → outputs a score $f_r(h,t) = -d_{r,m}(\mathbf{h}_m,\mathbf{t}_m) - \lambda\, d_{r,p}(\mathbf{h}_p,\mathbf{t}_p)$.
  • Why it was created: prior KGE models (TransE, DistMult, RotatE) model relation patterns (symmetry, inversion, composition) but fail to model semantic hierarchy (that some entities sit at higher/lower tree levels), and prior hierarchy-aware methods (TKRL) require extra type annotations. HAKE learns hierarchy from triples alone.
  • Concrete example: WordNet triple [arbor/cassia/palm, hypernym, tree] — "tree" is higher in the hierarchy. HAKE gives "tree" a smaller radial modulus (closer to root) than "palm"; two same-level entities "rose"/"peony" get similar moduli but different angular phases.
  • How it is used in this paper: it is the proposed model; evaluated on link prediction over WN18RR, FB15k-237, YAGO3-10.

Modulus part (radial coordinate)

  • What it computes: $d_{r,m}(\mathbf{h}_m,\mathbf{t}_m)=\|\mathbf{h}_m\circ\mathbf{r}_m-\mathbf{t}_m\|_2$ with $\mathbf{r}_m\in\mathbb{R}^k_+$ (positive scaling). Radius encodes tree depth; entities near root get smaller modulus.
  • Why it was created: to distinguish entities at different hierarchy levels (category a).
  • Concrete example: "mammal" vs "dog" — different moduli.
  • How it is used: the radial component of HAKE; ablation baseline ModE uses only this part (but allows negative $[r]_i$).

Phase part (angular coordinate)

  • What it computes: $d_{r,p}(\mathbf{h}_p,\mathbf{t}_p)=\|\sin((\mathbf{h}_p+\mathbf{r}_p-\mathbf{t}_p)/2)\|_1$; sine used for periodicity. Same formulation as pRotatE.
  • Why it was created: to distinguish entities at the same hierarchy level (category b), which share moduli and are otherwise indistinguishable by the modulus part.
  • Concrete example: "rose" vs "peony" — same radius, different phase.
  • How it is used: the angular component of HAKE.

5Cs Analysis5Cs 분석

Category: new method (KG embedding for link prediction) Context: builds directly on RotatE (Sun et al. 2019, rotation in complex space) and pRotatE; contrasts with TransE, DistMult, ComplEx, ConvE, TKRL. Belongs to the translational-distance family. Correctness: reports MRR/H@N on three standard benchmarks with a clean ablation (ModE baseline, mixture-bias ablation). No CIs reported (typical for KGE). Full validation of sign/modulus argument deferred to supplementary. Contributions: (1) polar-coordinate KGE that models semantic hierarchy from triples alone; (2) explicit modulus modeling (vs RotatE's incidental moduli); (3) SOTA link-prediction results on WN18RR/FB15k-237/YAGO3-10. Clarity: score function, distances, and loss are given explicitly; code released on GitHub — reproducible.

Contribution vs. Claim기여 대 주장

Abstract claims HAKE "effectively models semantic hierarchies" and "significantly outperforms" SOTA. Headline numbers support the link-prediction claim on 3 datasets. The hierarchy-modeling claim (radius = depth) is argued via relation-modulus histograms (Figure 2) and analysis section — need full read + supplementary to confirm the depth-correlation is quantitatively validated rather than illustrative. Gap to check: whether hierarchy quality is measured directly or only inferred from link-prediction wins.

Implementation Feasibility구현 실현가능성
  • Needs: standard KGE stack (PyTorch), Adam, self-adversarial negative sampling; benchmark triple sets; modest single-GPU compute.
  • Code: available at https://github.com/MIRALabUSTC/KGE-HAKE
  • Modifications: for SP9 would need (a) restricting to prerequisite relation, (b) possibly adding acyclicity regularization on the radial ordering, (c) an evaluation of hierarchy quality, not just link prediction.
  • Unknowns: missing link — how radial modulus ordering behaves on a sparse, automatically-extracted (noisy) prerequisite graph vs curated WordNet/Freebase.
Follow-up Papers관련 후속 논문
  • RotatE (Sun et al., 2019) — direct predecessor; HAKE's phase part reuses pRotatE distance and its self-adversarial loss.
  • TKRL (Xie, Liu & Sun, 2016) — hierarchy-aware baseline requiring type annotations; HAKE's contrast point.
Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = KG triples $(h,r,t)$ from WN18RR/FB15k-237/YAGO3-10; outputs = entity/relation embeddings scoring candidate triples for link prediction; conditions = static, fully-observed triple sets; hierarchy is latent and learned, relations can be arbitrary (not necessarily a DAG). Our setting (SP9, dependency-extraction / KB construction): inputs = extracted concept/skill nodes with prerequisite dependencies; outputs = a prerequisite/dependency structure (ideally a DAG); conditions = we want to model and validate prerequisite hierarchy, possibly acyclicity, over an automatically constructed KB. Differences:

  • HAKE learns hierarchy implicitly (radius ordering) but does not enforce a DAG / acyclicity constraint: missing link — no guarantee the learned hierarchy is acyclic or transitive.
  • Single relation type per prerequisite vs HAKE's multi-relation KG: reference only — HAKE's phase part is for same-level disambiguation, less central if only prerequisite edges matter.
  • Radial-depth idea directly transferable to modeling prerequisite depth: resolved: use modulus/radius as a continuous "prerequisite level" signal.
  • HAKE evaluated on link prediction (completion), not hierarchy extraction from text: missing link — our upstream is extraction; HAKE assumes triples already exist.
