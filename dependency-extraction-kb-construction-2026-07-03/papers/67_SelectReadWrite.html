Select, Read, and Write: A Multi-Agent Framework of Full-Text-based Related Work Generation

Xiaochuan Liu et al. · ACL 2025 Findings · 2025 · 4 citations인용 4회

arXiv:2505.19647 · PDF · SP: SP5

P0Track B

Problem
해결하려는 문제

Existing RWG methods (a) take only limited portions of reference papers as input (abstracts, intro/conclusion, prior RWS, retrieved spans) — causing misinterpretation/hallucination (C1) — and (b) explain each reference in isolation, ineffectively capturing inter-paper relationships (C2). Naively feeding all full texts into a long-context LLM in one pass is not a fix: performance degrades near the context limit.

전문 인 full-text는 LLM 문맥 한계를 초과하며 단일 패스 투입은 성능을 저하시킨다.

Solution / Contribution
해결책 / 기여

A three-agent iterative framework:

  1. Selector picks the next (paper, section) to read from paper abstracts + current memory + history, or emits <End>. Five strategies: Sequential (SR), Random (RR), Vanilla LLM, Graph-Co, Graph-Ci.
  2. Reader digests the selected section and rewrites a size-capped (4096-token) JSON working memory, discarding irrelevant content and tracking per-paper relations.
  3. Writer generates the final RWS from the curated memory, prompted (with an in-context example) to synthesize and group works rather than describe them separately.

The graph-aware selector restricts traversal to a citation or co-occurrence graph, at each step choosing to "stay" on the current paper or "jump" to a graph neighbor. Selector eq: $(R_t, s_t) = \text{Selector}((s_{0,1},\dots,s_{N,1}), M_{t-1}, H_{t-1}, G),\ R_t \in V_{t-1}$.

Selector/Reader/Writer 다중 에이전트와 용량 제한 working memory·그래프 제약 읽기를 결합하여, 섹션의 25.8%만 읽고도 단일 패스 full-context 투입(-9.3%)을 능가한다.

Key Results
주요 결과
  • GPT-4o Graph-Ci reaches Overall LLM-based 3.87 vs best baseline; Graph-Ci improves over retrieval-based (GO) baselines by +4.6% / +2.6% / +3.2% on Llama3-8B / Claude-3-Haiku / GPT-4o.
  • Selector ordering consistent across models: SR < RR < Vanilla < Graph-Co < Graph-Ci.
  • Feeding full text in a single pass hurts: Claude-3-Haiku full-context drops overall LLM score by 9.3% vs abstract-only.
  • Graph-Ci reads only 25.81% of section content (vs 100% SR/RR), i.e. best quality at lowest reading cost; selector mostly reads Related Work (73.5%) and Introduction (61.5%) sections, least Experiments (21%).
  • Human eval win-rate 58.7-61.3% overall vs strongest baseline, largest gain in Logic.
  • Gap to humans remains large: human RWS avg edges 9.48 vs best model 3.24.
Concepts Introduced도입된 개념

Full-text-based Related Work Generation (RWG)

  • What it computes: input = citing paper $C$ (= $R_0$, all sections except the related-work section) + a set of reference papers $R = \{R_1, \dots, R_N\}$ each with its entire section-segmented content $R_i = \{s_{i,1}, \dots, s_{i,L_i}\}$ → operation = iteratively read selected sections and synthesize → output = a related work section (RWS) that cites all of $R$ and stays coherent with $C$.
  • Why it was created: prior RWG feeds only limited portions (abstracts, intro+conclusion, prior RWS, or retrieved spans), causing shallow comprehension and hallucination (C1); it also explains references in isolation, failing to capture inter-paper relationships (C2).
  • Concrete example: OARelatedWork test inputs average ~70k tokens; the framework's best config reads only 25.81% of section content yet writes the RWS.
  • How it is used in this paper: the target task the whole framework solves.

Multi-agent Selector / Reader / Writer framework

  • What it computes: (1) Selector: input = abstracts of all papers $(s_{0,1},\dots,s_{N,1})$ + working memory $M_{t-1}$ + reading history $H_{t-1}$ → outputs a tuple $(R_t, s_t)$ = next paper+section to read, or <End>. (2) Reader: input = selected section content + $M_{t-1}$ → updates a size-capped (e.g. 4096-token) JSON working memory $M_t$, discarding irrelevant info. (3) Writer: input = final memory $M_T$ + history $H_T$ → generates the RWS, prompted to group similar works and avoid isolated descriptions.
  • Why it was created: single-pass long-context feeding degrades near the context limit ("lost in the middle"); delegating reading order and memory update to distinct iterative agents avoids feeding all 70k tokens at once.
  • Concrete example: reading history stored as (paper ID, section) tuples to prevent cyclic re-reading; memory JSON per paper stores have_read, content (issue, relations, ...).
  • How it is used in this paper: the core pipeline; five selector variants plugged into the same reader+writer.

Graph-aware Selector (Graph-Co / Graph-Ci)

  • What it computes: constrains the selector to traverse a paper-level relationship graph $G$. At step $t$ the agent sits at paper $R_{t-1}$, sees its one-hop subgraph, and either "stays" (reads another section of the current paper) or "jumps" to an adjacent paper. Two graph types: Co-occurrence graph $G_{co}$ (edge = two references jointly cited in one sentence of some paper's RWS) and Citation graph $G_{ci}$ (edge = $R_i$ cites $R_j$; $R_0$ cites all references).
  • Why it was created: prior graph-based RWG integrates structure implicitly; explicit graph traversal constraints make inter-paper relationships drive the reading order.
  • Concrete example: Graph-Ci reduces content read to 25.81% (vs 100% for sequential) while achieving the best scores.
  • How it is used in this paper: the paper's headline contribution; Graph-Ci > Graph-Co consistently.
5Cs Analysis5Cs 분석

Category: system / new method (LLM-agent pipeline for RWG). Context: builds on agent-based long-document reading (WebGPT, MemWalker/Chen 2023a, ReAct), memory-compression (Chevalier 2023), and prior RWG (Chen et al. 2021/2022, Li & Ouyang 2024/2025). Dataset from OARelatedWork (Docekal 2024). Graph-based RWG evaluation from Martin-Boyle 2024. Correctness: reports paired t-test significance ($p<0.05$) marked in Table 1; human evaluation (5 grad students, 20 AI papers, pairwise, win-rate >50%); ablations on graph level, prompt robustness, retention/attribution accuracy. Experiments on only 10% of OARelatedWork — sampling caveat. Contributions: (1) multi-agent select/read/write framework for full-text RWG; (2) graph-aware selector with citation + co-occurrence graphs; (3) extensive ablations across 3 base models and input configs. Clarity: method equations + full agent prompts (Figs 6-11) given, so largely reproducible; but graph construction details (co-occurrence extraction, initial node choice) are compressed.

Contribution vs. Claim기여 대 주장

— Stated claim: multi-agent + explicit graph constraints better capture inter-paper relationships for full-text RWG. Demonstrated: consistent but modest gains (2.6-4.6%) over retrieval baselines on 10% of one dataset (OARelatedWork), plus reduced reading cost. Apparent gap to probe in deep read: (a) improvements are small and base-model-dominated (Llama3-8B Graph-Ci still below abstract-only Claude); (b) large residual gap vs human writing (edges 3.24 vs 9.48) acknowledged in Limitations; (c) references assumed given — retrieval of the reference set is out of scope.

Implementation Feasibility구현 실현가능성
  • Needs: OARelatedWork dataset (Docekal 2024, arXiv:2405.01930); an LLM backend (Llama3-8B on-site, or GPT-4o / Claude-3-Haiku APIs); construction of citation + co-occurrence graphs from reference metadata/RWS; prompt templates (provided in appendix Figs 6-11); LLM-judge eval harness (GPT-4o / Claude-3.5-haiku / Gemini-1.5-Pro).
  • Code: available at https://github.com/1190200817/Full_Text_RWG
  • Modifications for our KB/dependency setting: replace the Writer (prose generator) with a structured-relation emitter; repurpose the graph from a reading constraint into an extraction target; adapt co-occurrence edge definition to the entity/dependency domain.
  • Unknowns: missing link: exact co-occurrence graph construction (sentence-level joint-citation parsing) and initial-node selection rule not fully specified in the skimmed sections; missing link: how framework scales beyond the 10% sampled test set.

Condition Delta — transfer to our setting
조건 차이 — 우리 환경으로의 전이

Paper's setting: inputs = a given set of cited full-text papers (section-segmented) + the citing paper; outputs = a natural-language related-work paragraph citing all references; conditions = OARelatedWork open-domain academic papers, ~70k-token inputs, LLM-based + graph-based text-quality metrics, reference set pre-supplied.

Our setting (SP5, dependency-extraction / KB construction): inputs = documents/papers; outputs = structured dependency relations / knowledge-base entries between entities; conditions = extraction of explicit inter-item relationships as a structured graph, agent pipeline target.

Differences:

  • Output form: free-text RWS vs structured dependency/KB triples — missing link (this paper produces prose, not extracted relations).
  • Graph usage: here the citation/co-occurrence graph is an input constraint on reading order, not an output to be constructed — reference only (relevant as a pattern for using graph structure to guide an agent, not as extraction).
  • Agent pipeline: Selector→Reader→Writer with size-capped working memory — resolved: directly reusable as an architecture for iterative full-text ingestion under context limits in a KB-construction agent.
  • Relationship capture: co-occurrence + citation edges to model inter-paper relations — resolved/partial: co-occurrence (jointly-cited-in-one-sentence) is a concrete, borrowable dependency signal for KB construction.
  • Domain: open-domain academic papers, English — reference only.
